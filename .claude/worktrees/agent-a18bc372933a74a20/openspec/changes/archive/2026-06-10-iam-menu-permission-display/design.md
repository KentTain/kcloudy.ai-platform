## 上下文

### 当前状态

IAM 模块已有基础的权限和角色管理功能，但展示体验不够友好：
- `PermissionList.vue`：简单的 Tab 分组展示，无搜索功能，操作类型纯文本显示
- 无独立的菜单展示页面
- `RoleForm.vue` 中权限分配内嵌于表单，交互不够直观

### 参考实现

Tenant 模块的 `ModuleDetail.vue` 提供了优秀的交互参考：
- 菜单管理：左侧树形展示 + 右侧详情面板
- 权限管理：可搜索表格 + Badge 标签 + 统计信息
- 角色权限分配：独立弹窗 + 全选/取消 + 已选统计

### 约束

- 仅前端 UI 增强，不涉及后端 API 变更
- 复用已有的 `Tree`、`DataTable`、`Badge` 等通用组件
- 保持与现有 IAM 模块风格一致

## 目标 / 非目标

**目标：**
1. 增强权限管理页面的展示体验（搜索、Badge、统计）
2. 新增独立的菜单展示页面（树形展示 + 详情）
3. 优化角色权限分配交互（独立弹窗、全选、统计）

**非目标：**
1. 不修改后端 API
2. 不新增权限编辑/删除功能（只读展示）
3. 不新增菜单编辑功能（只读展示）

## 决策

### 1. 权限管理页面重构

**决策**：采用搜索表格 + 操作类型 Badge 的展示方式

**布局结构**：
```
┌─────────────────────────────────────────────────────────┐
│ 权限管理                                                │
├─────────────────────────────────────────────────────────┤
│ [搜索框：搜索名称/编码/资源]              [刷新按钮]     │
├─────────────────────────────────────────────────────────┤
│ ┌───────────┬───────────────┬────────┬────────┬──────┐ │
│ │权限名称    │权限编码        │资源     │操作类型 │说明   │ │
│ ├───────────┼───────────────┼────────┼────────┼──────┤ │
│ │用户查看    │user:read      │user    │[read]  │...   │ │
│ │用户编辑    │user:write     │user    │[write] │...   │ │
│ │用户删除    │user:delete    │user    │[delete]│...   │ │
│ └───────────┴───────────────┴────────┴────────┴──────┘ │
├─────────────────────────────────────────────────────────┤
│ 共 X 项权限                                              │
└─────────────────────────────────────────────────────────┘
```

**理由**：
- 搜索框提升查找效率
- Badge 直观区分操作类型（read=默认、write=secondary、delete=destructive）
- 统计信息提供数据概览

### 2. 菜单展示页面设计

**决策**：采用左侧树 + 右侧详情的布局

**布局结构**：
```
┌─────────────────────────────────────────────────────────┐
│ 菜单管理                                                │
├─────────────────────────────────────────────────────────┤
│ ┌─────────────────┬───────────────────────────────────┐ │
│ │ 菜单树          │ 菜单详情                          │ │
│ │ ┌─────────────┐ │ ┌─────────────┬─────────────────┐ │ │
│ │ │ ├─系统管理   │ │ │ 菜单名称    │ 用户管理        │ │ │
│ │ │ │ ├─用户管理 │ │ ├─────────────┼─────────────────┤ │ │
│ │ │ │ ├─角色管理 │ │ │ 菜单编码    │ iam:user        │ │ │
│ │ │ │ └─部门管理 │ │ ├─────────────┼─────────────────┤ │ │
│ │ │ ├─租户管理   │ │ │ 路由路径    │ /iam/users      │ │ │
│ │ │ └─...       │ │ ├─────────────┼─────────────────┤ │ │
│ │ └─────────────┘ │ │ 所属模块    │ iam             │ │ │
│ │                 │ ├─────────────┼─────────────────┤ │ │
│ │                 │ │ 图标        │ user            │ │ │
│ │                 │ └─────────────┴─────────────────┘ │ │
│ └─────────────────┴───────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

**数据来源**：
- 调用 `GET /v1/iam/menu` 获取所有菜单树
- 复用 `framework/stores/menu.ts` 中的 `MenuTreeNode` 类型

**理由**：
- 与 Tenant 模块交互风格一致
- 树形结构直观展示菜单层级
- 详情面板提供完整信息

### 3. 角色权限分配弹窗优化

**决策**：从 RoleForm 内嵌改为独立弹窗

**交互流程**：
1. 在 `RoleList.vue` 点击「分配权限」按钮
2. 弹出独立弹窗，显示权限选择列表
3. 提供「全选/取消全选」按钮
4. 显示「已选择 X/Y 项权限」统计
5. 权限项使用 Badge 标识操作类型

**布局结构**：
```
┌─────────────────────────────────────────────────────────┐
│ 分配权限                                          [X]   │
├─────────────────────────────────────────────────────────┤
│ 为角色「管理员」分配权限                                 │
├─────────────────────────────────────────────────────────┤
│ 已选择 5/20 项权限                      [全选] [取消全选]│
├─────────────────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────────────────┐ │
│ │ ☑ 用户管理                              [read]      │ │
│ │ ☐ 用户管理                              [write]     │ │
│ │ ☐ 用户管理                              [delete]    │ │
│ │ ☑ 角色管理                              [read]      │ │
│ │ ...                                                 │ │
│ └─────────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────┤
│                              [取消]  [保存]             │
└─────────────────────────────────────────────────────────┘
```

**理由**：
- 独立弹窗提供更大的操作空间
- 全选/取消提升批量操作效率
- 统计信息直观展示选择进度

## 风险 / 权衡

### 风险 1：菜单 API 响应数据量

**风险**：菜单树较大时，一次性加载可能影响性能

**缓解措施**：
- 当前菜单数量有限，一次性加载可接受
- 后续可按需优化为懒加载

### 风险 2：权限列表搜索性能

**风险**：权限数量较多时，前端搜索可能卡顿

**缓解措施**：
- 使用 computed 进行响应式过滤
- 如需优化可改用后端分页搜索

### 风险 3：与现有 RoleForm 的兼容

**风险**：修改权限分配方式可能影响现有 RoleForm 的使用

**缓解措施**：
- RoleForm 保留现有功能，权限分配弹窗作为补充入口
- 在 RoleList 页面新增「分配权限」按钮触发弹窗
