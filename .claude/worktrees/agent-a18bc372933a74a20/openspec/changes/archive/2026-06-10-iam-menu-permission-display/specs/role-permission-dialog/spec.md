## 新增需求

### 需求:权限分配独立弹窗

系统 SHALL 提供独立的角色权限分配弹窗组件。

#### 场景:打开权限分配弹窗

- **当** 用户在角色列表点击「分配权限」按钮
- **那么** 系统弹出权限分配弹窗，显示该角色当前已分配的权限

#### 场景:关闭弹窗

- **当** 用户点击「取消」或关闭按钮
- **那么** 系统关闭弹窗，不保存修改

### 需求:权限选择统计

系统 SHALL 在权限分配弹窗中显示已选权限统计。

#### 场景:显示已选统计

- **当** 权限分配弹窗打开
- **那么** 系统显示「已选择 X/Y 项权限」

#### 场景:实时更新统计

- **当** 用户勾选或取消勾选权限
- **那么** 系统实时更新已选统计数字

### 需求:全选/取消全选

系统 SHALL 提供全选和取消全选按钮。

#### 场景:全选权限

- **当** 用户点击「全选」按钮
- **那么** 系统勾选所有权限

#### 场景:取消全选

- **当** 用户点击「取消全选」按钮
- **那么** 系统取消所有已勾选的权限

#### 场景:全选按钮状态切换

- **当** 所有权限已全部勾选
- **那么** 「全选」按钮显示为「取消全选」

### 需求:权限项 Badge 展示

系统 SHALL 在权限分配列表中使用 Badge 展示操作类型。

#### 场景:权限项展示

- **当** 权限分配列表渲染时
- **那么** 每个权限项显示：Checkbox + 权限名称 + 操作类型 Badge

### 需求:保存权限分配

系统 SHALL 支持保存权限分配修改。

#### 场景:保存成功

- **当** 用户点击「保存」按钮
- **那么** 系统调用 API 更新角色权限，关闭弹窗，显示成功提示

#### 场景:保存失败

- **当** API 调用失败
- **那么** 系统显示错误提示，保持弹窗打开

### 需求:角色列表入口

系统 SHALL 在角色列表页面提供权限分配入口。

#### 场景:显示分配权限按钮

- **当** 角色列表渲染时
- **那么** 每行角色显示「分配权限」操作按钮

## 修改需求

无。

## 移除需求

无。
