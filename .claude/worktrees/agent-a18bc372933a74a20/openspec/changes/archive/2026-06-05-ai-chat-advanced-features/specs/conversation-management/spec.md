## 新增需求

### 需求:用户可以查看会话列表

系统必须提供会话列表页面，展示用户的所有历史会话。

#### 场景:会话列表加载
- **当** 用户访问会话列表页面
- **那么** 系统请求 `GET /api/v1/conversations`
- **那么** 系统展示会话列表（名称、创建时间、消息数量）
- **那么** 列表按创建时间倒序排列

#### 场景:空会话列表展示
- **当** 用户无任何历史会话
- **那么** 系统展示空状态提示
- **那么** 提供「开始新对话」入口

### 需求:用户可以切换会话

用户必须能够从会话列表切换到指定会话继续对话。

#### 场景:切换会话成功
- **当** 用户点击会话列表中的某个会话
- **那么** 系统跳转到对话页面
- **那么** 系统加载该会话的历史消息
- **那么** 输入框自动聚焦

#### 场景:会话历史消息加载
- **当** 用户切换到已有会话
- **那么** 系统请求会话历史消息
- **那么** 消息按时间顺序展示
- **那么** 支持滚动查看全部历史

### 需求:用户可以删除会话

用户必须能够删除不再需要的会话。

#### 场景:删除会话成功
- **当** 用户点击删除按钮并确认
- **那么** 系统请求 `DELETE /api/v1/conversations/{id}`
- **那么** 系统从列表中移除该会话
- **那么** 系统展示删除成功提示

#### 场景:删除当前会话后跳转
- **当** 用户删除当前正在进行的会话
- **那么** 系统跳转到会话列表或新建会话页面

### 需求:会话数据必须租户隔离

系统必须确保不同租户的会话数据隔离。

#### 场景:租户数据隔离
- **当** 用户请求会话列表
- **那么** 系统仅返回当前租户的会话
- **那么** 禁止访问其他租户的会话
