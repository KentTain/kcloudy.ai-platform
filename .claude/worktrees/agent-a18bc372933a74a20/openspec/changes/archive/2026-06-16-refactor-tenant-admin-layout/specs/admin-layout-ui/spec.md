## 新增需求

### 需求:管理员菜单动态获取
系统必须提供管理后台菜单的动态获取能力，菜单数据来自 Tenant 模块（`module.code='tenant'`）的 `module_menus` 表。

#### 场景:成功获取菜单
- **当** 管理员请求 `GET /tenant/admin/v1/admin/menus`
- **那么** 系统返回 `module.code='tenant'` 模块下所有可见菜单的树形结构

#### 场景:模块不存在时的处理
- **当** 数据库中不存在 `code='tenant'` 的模块
- **那么** 系统返回空数组 `[]`

#### 场景:菜单包含子菜单
- **当** 模块菜单存在父子关系
- **那么** 系统返回嵌套的树形结构，`children` 字段包含子菜单列表

### 需求:侧边栏折叠图标模式
布局侧边栏必须支持 `collapsible="icon"` 模式，允许用户折叠侧边栏为图标状态以释放内容区空间。

#### 场景:展开状态显示完整品牌信息
- **当** 侧边栏处于展开状态
- **那么** 品牌区域显示 Logo 图标、"KCloudy" 和 "管理后台" 文字

#### 场景:折叠状态只显示图标
- **当** 侧边栏处于折叠状态
- **那么** 品牌区域只显示 Logo 图标，文字隐藏

#### 场景:折叠状态菜单只显示图标
- **当** 侧边栏处于折叠状态
- **那么** 菜单项只显示图标，使用 `tooltip` 提示菜单名称

### 需求:布局组件目录规范
布局子组件必须放置在 `layouts/components/` 目录下，使用 `App` 前缀命名。

#### 场景:组件导入路径正确
- **当** `AdminLayout.vue` 引用 `AppNavMain` 组件
- **那么** 导入路径为 `./components/AppNavMain.vue`

#### 场景:组件命名规范
- **当** 创建新的布局子组件
- **那么** 组件文件名使用 `App` 前缀（如 `AppBrandHeader.vue`）

### 需求:菜单样式对齐 Framework
菜单项样式必须与 Framework 的 `AppNavMain` 保持一致。

#### 场景:菜单项高度一致
- **当** 渲染菜单项
- **那么** 菜单项高度为 `h-10`

#### 场景:菜单项激活态
- **当** 当前路由与菜单项 `path` 匹配
- **那么** 菜单项应用 `bg-white! text-primary!` 样式

#### 场景:菜单项悬停效果
- **当** 鼠标悬停在菜单项上
- **那么** 菜单项应用 `hover:bg-accent-foreground/5` 样式

### 需求:用户面板样式对齐
用户面板必须保留在侧边栏底部，但样式与 Framework 的用户面板对齐。

#### 场景:用户面板显示位置
- **当** 侧边栏渲染完成
- **那么** 用户面板位于 `SidebarFooter` 区域

#### 场景:用户面板头像样式
- **当** 渲染用户头像
- **那么** 头像使用 `h-8 w-8 rounded-lg` 尺寸和圆角

#### 场景:用户下拉菜单宽度
- **当** 点击用户面板展开下拉菜单
- **那么** 下拉菜单宽度为 `min-w-56 rounded-lg`
