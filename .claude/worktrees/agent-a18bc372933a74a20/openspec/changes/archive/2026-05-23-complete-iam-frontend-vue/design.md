## Context

IAM 前端模块主体功能已实现，但仍有部分关键功能未完成：
- 权限树组件：角色表单中权限选择使用简单的 checkbox 列表，缺少树形结构
- 部门树组件：部门管理需要独立的树形组件支持节点选择
- 登录历史：个人中心缺少登录历史查看功能
- Framework 集成：侧边栏菜单、租户上下文、权限指令尚未集成
- 测试覆盖：缺少单元测试和性能优化

## Goals / Non-Goals

**Goals:**
- 完成权限树组件，支持按资源分组的树形权限选择
- 完成部门树组件，支持树形展示和节点选择
- 实现登录历史查看功能，展示用户登录记录
- 集成 IAM 模块菜单到 Framework 侧边栏
- 扩展用户状态，支持租户上下文信息
- 集成权限指令，实现按钮级权限控制
- 创建 API、Store、组件的单元测试
- 优化大数据量场景的性能

**Non-Goals:**
- 不修改后端 API 结构
- 不重构现有已实现的功能
- 不实现登录历史的后端存储（假设后端已有接口）

## Decisions

### 权限树组件设计

**选择：使用 Element Plus el-tree 组件**

原因：
- Element Plus 已集成，无需引入新依赖
- el-tree 原生支持多选、展开/折叠、搜索过滤
- 可通过自定义节点内容实现权限名称和描述展示

实现要点：
- 按资源分组权限，资源作为父节点
- 支持全选/取消全选某个资源下的所有权限
- 支持搜索过滤权限名称

### 部门树组件设计

**选择：独立组件 DepartmentTree.vue**

原因：
- 部门树在多处复用（部门管理页面、用户分配部门）
- 需要支持节点选择、展开/折叠、搜索
- 需要支持拖拽排序（可选）

实现要点：
- 使用 el-tree 组件
- 支持单选和多选模式
- 支持节点点击事件和选择事件

### 登录历史实现

**选择：在 Profile.vue 增加 Tab 页签**

原因：
- 登录历史属于个人中心功能
- 使用 Tab 可以与现有功能并存
- 无需新增页面路由

实现要点：
- 新增 `api/auth.ts` 的 `getLoginHistory` 方法
- 使用 el-table 展示登录记录
- 支持分页和按时间范围筛选

### Framework 集成

**侧边栏菜单集成**

选择：修改 AppSidebar.vue，支持动态菜单

实现方式：
- 从路由配置中读取 IAM 模块菜单
- 或通过 props 传入菜单数据
- 支持菜单图标配置

**租户上下文**

选择：扩展 useUserStore

实现方式：
- 在 UserInfo 类型中增加 tenantInfo 字段
- 登录时获取用户租户信息并存储
- 提供 `currentTenant` computed 属性

**权限指令**

选择：使用现有的 permission.ts 指令

实现方式：
- 在 main.ts 中注册指令
- 确保权限数据在用户登录后加载

## Risks / Trade-offs

### 权限树性能
- **风险**：权限数量大时（>500）渲染可能卡顿
- **缓解**：使用虚拟滚动（el-tree-virtual）或分页加载

### 菜单权限过滤
- **风险**：动态菜单可能导致路由闪烁
- **缓解**：在路由守卫中预加载菜单配置

### 租户切换状态丢失
- **风险**：切换租户后部分缓存状态可能不正确
- **缓解**：切换租户时刷新页面或重置相关 Store
