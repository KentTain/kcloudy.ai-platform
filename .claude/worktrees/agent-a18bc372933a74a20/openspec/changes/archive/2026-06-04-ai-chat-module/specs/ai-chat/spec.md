## 新增需求

### 需求:用户可以发送消息并接收流式响应

用户必须能够通过前端界面发送消息，并实时接收 AI 的流式响应。

#### 场景:发送消息成功
- **当** 用户在输入框输入消息并点击发送
- **那么** 系统发送请求到 `POST /api/v1/chat-messages`
- **那么** 系统创建新会话（如果是首次对话）
- **那么** 系统返回 SSE 流式响应

#### 场景:流式响应格式正确
- **当** 后端处理消息请求
- **那么** 系统返回 AI SDK 标准 UIMessageChunk 格式的 SSE 流
- **那么** 流以 `data: {"type":"start","messageId":"..."}` 开始
- **那么** 流以 `data: {"type":"finish","finishReason":"stop"}` 结束
- **那么** 流以 `data: [DONE]` 结束

#### 场景:恢复现有会话
- **当** 用户发送消息时提供 `id`（conversation_id）
- **那么** 系统加载现有会话历史
- **那么** 系统在新消息上下文中包含历史消息

### 需求:前端必须使用 AI SDK 协议

前端必须使用 `@ai-sdk/vue` 的 `useChat` composable 与后端通信。

#### 场景:useChat 配置正确
- **当** 前端初始化 useChat
- **那么** 配置 `api: "/api/v1/chat-messages"`
- **那么** 配置 `body: { model: {...} }` 传递模型参数

#### 场景:消息渲染
- **当** 收到流式响应
- **那么** 消息内容实时渲染在对话界面
- **那么** 支持 Markdown 格式
- **那么** 代码块语法高亮

### 需求:后端必须接受 AI SDK 请求格式

后端 `/chat-messages` 接口必须接受 AI SDK 标准请求格式。

#### 场景:请求格式验证
- **当** 后端收到请求
- **那么** 验证请求体符合 `AIChatRequest` Schema
- **那么** 必须包含 `messages` 数组
- **那么** 每条消息必须包含 `id`、`role`、`parts`

#### 场景:从 messages 提取用户输入
- **当** 后端处理请求
- **那么** 从 `messages` 最后一条消息的 `parts` 中提取文本内容作为用户查询

### 需求:会话必须持久化

系统必须持久化会话和消息记录。

#### 场景:会话创建
- **当** 用户首次发送消息（无 `id`）
- **那么** 系统创建新的 `Conversation` 记录
- **那么** 系统创建用户消息 `Message` 记录
- **那么** 系统创建助手消息 `Message` 记录

#### 场景:消息状态更新
- **当** 流式响应完成
- **那么** 系统更新助手消息的 `content` 字段
- **那么** 系统更新消息状态为 `NORMAL`
- **那么** 系统记录 token 使用量

### 需求:多租户数据隔离

系统必须确保不同租户的会话数据隔离。

#### 场景:租户隔离
- **当** 用户访问会话
- **那么** 系统验证会话的 `tenant_id` 与当前用户匹配
- **那么** 禁止访问其他租户的会话
