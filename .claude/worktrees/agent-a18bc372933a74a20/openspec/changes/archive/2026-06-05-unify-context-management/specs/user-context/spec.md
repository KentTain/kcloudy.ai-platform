## 新增需求

### 需求:用户上下文统一存储

系统必须将用户信息统一存储到 ContextVar，确保在所有层次（Controller、Service、ORM 事件）都可访问。

#### 场景:中间件设置用户上下文
- **当** IAMAuthMiddleware 验证 JWT 成功
- **那么** 用户 ID、用户名、角色、权限被存储到 Context

#### 场景:Service 层获取用户信息
- **当** Service 层调用 get_user_id()
- **那么** 返回当前上下文中的用户 ID

#### 场景:ORM 事件获取用户信息
- **当** before_insert 或 before_update 事件触发
- **且** 调用 get_user_id()
- **那么** 返回当前上下文中的用户 ID

### 需求:用户租户归属验证

系统必须在验证用户身份时，验证用户归属租户与当前租户一致。

#### 场景:租户归属一致
- **当** JWT 中的 tenant_id 与当前租户 ID 一致
- **那么** 用户验证通过
- **且** 用户信息被设置到上下文

#### 场景:租户归属不一致
- **当** JWT 中的 tenant_id 与当前租户 ID 不一致
- **那么** 返回 403 错误
- **且** 错误消息为跨租户访问被拒绝

### 需求:用户上下文清理

系统必须在请求结束时清理用户上下文。

#### 场景:请求成功结束时清理
- **当** 请求处理成功完成
- **那么** 用户上下文被清理

#### 场景:请求异常结束时清理
- **当** 请求处理过程中发生异常
- **那么** 用户上下文仍被清理
