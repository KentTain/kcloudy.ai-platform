# 后台任务管控规范

## 新增需求

### 需求:系统支持任务停止

系统必须提供任务停止功能，支持跨进程取消正在执行的 LLM 任务。

#### 场景:停止指定任务

- **当** 调用 `stop_task_by_id(task_id, task_type)`
- **那么** 系统必须发布停止信号到 Redis PubSub

#### 场景:Worker 接收停止信号

- **当** Worker 订阅的频道收到停止信号
- **那么** 系统必须检查信号中的 `task_id` 是否匹配当前任务

#### 场景:取消 asyncio 任务

- **当** 确认需要停止当前任务
- **那么** 系统必须调用 `asyncio.Task.cancel()` 取消任务

#### 场景:停止后更新消息状态

- **当** 任务被成功停止
- **那么** 系统必须将对应消息状态更新为 `stopped`

---

### 需求:系统支持任务超时清理

系统必须提供任务超时自动清理功能。

#### 场景:启动超时检查

- **当** 任务开始执行
- **那么** 系统必须启动超时计时器

#### 场景:任务超时触发清理

- **当** 任务执行时间超过配置的超时时间
- **那么** 系统必须触发清理操作

#### 场景:清理超时任务

- **当** 触发清理操作
- **那么** 系统必须取消任务并更新消息状态为 `stopped`

#### 场景:配置超时时间

- **当** 查询超时配置
- **那么** 系统必须从 `settings.workflow.task_cleanup_timeout` 读取超时时间（秒）

---

### 需求:系统支持任务类型管理

系统必须支持定义和管理不同类型的后台任务。

#### 场景:定义任务类型常量

- **当** 引用任务类型
- **那么** 系统必须提供 `TASK_TYPE_GENERATE_LLM` 等常量

#### 场景:任务类型用于 PubSub 通道

- **当** 发布/订阅任务信号
- **那么** 系统必须使用任务类型作为通道名称的一部分

---

### 需求:系统支持活跃任务跟踪

系统必须跟踪当前活跃的任务。

#### 场景:记录活跃任务

- **当** 任务开始执行
- **那么** 系统必须将 `task_id` 添加到活跃任务集合

#### 场景:移除活跃任务

- **当** 任务完成或被取消
- **那么** 系统必须将 `task_id` 从活跃任务集合中移除

#### 场景:查询活跃任务数量

- **当** 查询活跃任务统计
- **那么** 系统必须返回当前活跃任务的数量

---

### 需求:系统支持任务取消回调

系统必须支持任务取消时的回调处理。

#### 场景:注册取消回调

- **当** 创建任务
- **那么** 系统必须允许注册取消时的回调函数

#### 场景:执行取消回调

- **当** 任务被取消
- **那么** 系统必须执行已注册的回调函数进行清理

#### 场景:回调处理异常

- **当** 回调函数抛出异常
- **那么** 系统必须捕获异常并记录日志，不影响其他清理操作

---

### 需求:系统复用 PubSub 组件

系统必须复用 `framework/pubsub/` 实现跨进程通信。

#### 场景:使用 PubSub 发布信号

- **当** 发布任务停止信号
- **那么** 系统必须使用 `framework.pubsub.get_pubsub_provider().publish()`

#### 场景:使用 PubSub 订阅信号

- **当** 启动 Worker
- **那么** 系统必须使用 `framework.pubsub.get_pubsub_provider().subscribe()` 订阅任务通道
