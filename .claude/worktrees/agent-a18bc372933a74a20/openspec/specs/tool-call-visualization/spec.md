## 新增需求

### 需求:工具调用必须可视化展示

系统必须在前端对话界面中可视化展示工具调用过程和结果。

#### 场景:工具调用开始展示
- **当** 后端发送 	ool-call 类型 SSE 事件
- **那么** 前端渲染工具调用组件
- **那么** 组件展示工具名称、参数、执行状态（进行中）

#### 场景:工具调用结果展示
- **当** 后端发送 	ool-result 类型 SSE 事件
- **那么** 前端更新工具调用组件状态为已完成
- **那么** 组件展示工具返回结果（可折叠）

#### 场景:多个工具调用顺序展示
- **当** 一次对话中触发多个工具调用
- **那么** 前端按顺序展示所有工具调用
- **那么** 每个工具调用独立渲染

### 需求:工具调用事件格式必须符合 AI SDK 标准

后端发送的工具调用事件必须符合 AI SDK UIMessageChunk 格式。

#### 场景:tool-call 事件格式
- **当** 后端发送工具调用开始事件
- **那么** SSE 数据格式为 `{"type":"tool-call","toolCallId":"xxx","toolName":"search","args":{...}}`

#### 场景:tool-result 事件格式
- **当** 后端发送工具调用结果事件
- **那么** SSE 数据格式为 `{"type":"tool-result","toolCallId":"xxx","result":"..."}`
