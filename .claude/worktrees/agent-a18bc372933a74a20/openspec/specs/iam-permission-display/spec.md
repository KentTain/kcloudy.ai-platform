# iam-permission-display 规范

## 目的
待定 - 由归档变更 iam-menu-permission-display 创建。归档后请更新目的。
## 需求
### 需求:权限列表搜索

系统 SHALL 支持在权限管理页面按名称、编码、资源进行搜索过滤。

#### 场景:按名称搜索

- **当** 用户在搜索框输入权限名称关键词
- **那么** 系统实时过滤显示匹配的权限列表

#### 场景:按编码搜索

- **当** 用户在搜索框输入权限编码关键词
- **那么** 系统实时过滤显示匹配的权限列表

#### 场景:按资源搜索

- **当** 用户在搜索框输入资源名称关键词
- **那么** 系统实时过滤显示匹配的权限列表

#### 场景:清空搜索

- **当** 用户清空搜索框内容
- **那么** 系统显示全部权限列表

### 需求:操作类型 Badge 展示

系统 SHALL 使用不同颜色的 Badge 标签展示权限操作类型。

#### 场景:read 操作类型展示

- **当** 权限操作类型为 read
- **那么** 系统使用 default 变体的 Badge 展示 "read"

#### 场景:write 操作类型展示

- **当** 权限操作类型为 write
- **那么** 系统使用 secondary 变体的 Badge 展示 "write"

#### 场景:delete 操作类型展示

- **当** 权限操作类型为 delete
- **那么** 系统使用 destructive 变体的 Badge 展示 "delete"

### 需求:权限统计信息

系统 SHALL 在权限列表底部显示权限总数和筛选结果统计。

#### 场景:显示权限总数

- **当** 权限列表加载完成
- **那么** 系统显示「共 X 项权限」

#### 场景:显示筛选结果统计

- **当** 用户使用搜索功能过滤权限
- **那么** 系统显示「共 X 项权限，筛选结果 Y 项」

### 需求:权限表格布局

系统 SHALL 使用表格形式展示权限列表，包含以下列：权限名称、权限编码、资源、操作类型、说明。

#### 场景:表格列展示

- **当** 权限列表加载完成
- **那么** 系统按顺序展示：权限名称、权限编码、资源、操作类型（Badge）、说明

#### 场景:空状态展示

- **当** 权限列表为空或搜索无结果
- **那么** 系统显示「暂无权限数据」或「未找到匹配的权限」
