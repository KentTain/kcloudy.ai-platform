## 修改需求

### 需求: 租户详情查询

系统 SHALL 支持查询租户详情。

**变更说明**：优化响应构建性能，使用 `asyncio.gather` 并行查询资源配置。

#### 场景: 查询租户详情
- **当** 管理员请求 `GET /admin/v1/tenants/{id}`
- **那么** 返回租户详细信息

#### 场景: 查询不存在的租户
- **当** 管理员请求 `GET /admin/v1/tenants/nonexistent`
- **那么** 返回 HTTP 404 错误

#### 场景: 并行查询资源配置（新增）
- **当** 构建租户详情响应时
- **那么** 5 个资源配置查询必须使用 `asyncio.gather` 并行执行

### 需求: 租户资源绑定查询

系统 SHALL 支持查询租户的资源绑定情况。

**变更说明**：优化响应构建性能，使用 `asyncio.gather` 并行查询资源配置。

#### 场景: 查询资源绑定
- **当** 管理员请求 `GET /admin/v1/tenants/{id}/resources`
- **那么** 返回租户当前的资源绑定情况

#### 场景: 并行查询资源配置（新增）
- **当** 构建资源绑定响应时
- **那么** 5 个资源配置查询必须使用 `asyncio.gather` 并行执行

### 需求: 租户资源绑定更新

系统 SHALL 支持更新租户的资源绑定。

**变更说明**：优化响应构建性能，使用 `asyncio.gather` 并行查询资源配置。

#### 场景: 资源绑定
- **当** 管理员请求 `PUT /admin/v1/tenants/{id}/resources` 并提供配置 ID
- **那么** 更新租户的资源绑定

#### 场景: 解绑资源
- **当** 管理员将某个配置设为 null
- **那么** 解除该资源的绑定

#### 场景: 并行查询资源配置（新增）
- **当** 构建资源绑定响应时
- **那么** 5 个资源配置查询必须使用 `asyncio.gather` 并行执行

### 需求: 租户列表查询

系统 SHALL 支持跨租户查询租户列表。

**变更说明**：优化响应构建性能，使用 `asyncio.gather` 并行构建租户响应。

#### 场景: 查询租户列表
- **当** 管理员请求 `GET /admin/v1/tenants`
- **那么** 返回所有租户列表（分页）

#### 场景: 搜索租户
- **当** 管理员请求 `GET /admin/v1/tenants?keyword=acme`
- **那么** 返回名称或编码包含 "acme" 的租户列表

#### 场景: 并行构建租户响应（新增）
- **当** 构建租户列表响应时
- **那么** 每个租户的响应构建必须使用 `asyncio.gather` 并行执行
