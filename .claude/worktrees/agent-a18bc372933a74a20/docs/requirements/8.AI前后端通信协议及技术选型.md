# AI 前后端通信协议及技术选型

## 1. 背景与目标

### 1.1 项目现状

**后端技术栈**

- 框架：Python 3.12 + FastAPI
- AI 框架：LangChain + LangGraph
- 现有接口：POST /chat-messages (流式 SSE 响应)
- **现状**：当前接口无前端调用者，可安全改造

**前端技术栈**

- 框架：Vue 3 + TypeScript
- UI 组件：Shadcn-vue
- 待实现：ChatGPT 风格的对话界面

### 1.2 核心目标

1. **后端协议标准化**：改造后端接口以适配 AI SDK 标准协议，保持架构可扩展性
2. **前端快速开发**：利用 AI SDK + AI Elements Vue 快速构建对话功能，避免重复造轮子
3. **长期可维护性**：协议标准化便于与生态工具集成，支持高级特性演进

### 1.3 关键决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| ai-elements-vue 组件位置 | 全局 `components/ai-elements/` | 遵循 shadcn-vue 惯例，便于跨模块复用 |
| 后端改造层次 | 方案 B：扩展 LangChain 回调 | 利用现有扩展点，更贴近 LangChain 生态 |
| 接口路径 | 直接改造 `/chat-messages` | 无前端调用者，可安全改造 |
| 协议适配位置 | 后端完全适配 | 前端无需适配代码，与 AI SDK 生态完全兼容 |
| 实施策略 | 分阶段：阶段一简化实现，阶段二引入 Callback Handler | 快速落地验证，后续技术债清理 |

## 2. 技术选型分析

### 2.1 前端技术栈选型

AI SDK 提供分层架构，各层职责明确，组合使用：

| 包名                | 层次     | 功能                                     | 选型  |
| ----------------- | ------ | -------------------------------------- | --- |
| `ai`              | 核心层    | 流式处理、协议解析、类型定义                         | 必须  |
| `@ai-sdk/vue`     | 框架适配层  | `useChat`、`Chat` class、`ChatTransport` | 必须  |
| `ai-elements-vue` | UI 组件层 | 基于 shadcn-vue 的 AI 对话组件集               | 核心  |
| `chat`（Chat SDK）  | 完整解决方案 | 自带 UI 和后端模板，绑死 Next.js                 | 不适用 |

**最终组合**：

``bash
# 核心依赖
pnpm add ai @ai-sdk/vue

# AI 对话组件（通过 CLI 安装到项目中，源码可控可定制）
npx ai-elements-vue@latest
# 安装位置：web/vue/src/components/ai-elements/（全局）
``

#### 2.1.1 为什么选择 ai-elements-vue

`ai-elements-vue`（[GitHub](https://github.com/vuepont/ai-elements-vue)、[文档](https://ai-elements-vue.com)）是基于 shadcn-vue 构建的 AI 原生组件库，与本项目技术栈完全匹配：

| 维度    | 说明                                                                           |
| ----- | ---------------------------------------------------------------------------- |
| 技术栈匹配 | Vue 3 + shadcn-vue + AI SDK，与本项目一致                                           |
| 组件覆盖  | 40+ 组件，覆盖 Chatbot / Workflow / Vibe-Coding 场景                                |
| 集成方式  | CLI 将组件源码复制到项目（全局 `components/ai-elements/`），完全可控可定制                         |
| 核心依赖  | `vue-stream-markdown`（流式 Markdown）、`vue-stick-to-bottom`（自动滚动）、`shiki`（代码高亮） |

关键 Chatbot 组件一览：

| 组件 | 功能 |
|------|------|
| `conversation` | 对话容器，管理消息列表和滚动 |
| `message` | 消息组件，支持头像、状态、操作 |
| `prompt-input` | 高级输入组件，支持模型选择和附件 |
| `reasoning` | 思考过程展示 |
| `tool` | 工具调用可视化 |
| `sources` | 资源引用展示 |
| `confirmation` | 工具执行确认流程 |
| `checkpoint` | 对话检查点 |
| `model-selector` | AI 模型选择 |
| `suggestion` | 快捷建议操作 |
| `code-block` | 语法高亮代码块 |
| `artifact` | 代码/文档展示 |

#### 2.1.2 为什么不选 Chat SDK

`chat`（`npm install chat`）是 AI SDK 的完整解决方案包，包含后端模板、前端 UI 和数据库集成。但它绑定 Next.js App Router + Prisma + NextAuth，与本项目 Python/FastAPI 后端 + Vue 前端的架构完全不兼容。

## 3. 协议对比与适配方案

### 3.1 当前后端 SSE 格式

`
POST /chat-messages
Request: { model, query, conversation_id, files, search }
Response: SSE 流

事件格式：
data: {"event":"message","data":{"content":"..."}}
data: {"event":"finish","data":{"prompt_tokens":10,"completion_tokens":20}}
data: {"event":"error","data":{"code":"MODEL_ERROR","message":"..."}}
data: {"event":"close"}
`

### 3.2 AI SDK 标准协议格式

`
POST /chat-messages
Request: { id, messages: [...], trigger, messageId, body }
Response: SSE 流

事件格式：
data: {"type":"start","messageId":"msg-xxx"}
data: {"type":"text-start","id":"text-1"}
data: {"type":"text-delta","id":"text-1","delta":"你好"}
data: {"type":"text-delta","id":"text-1","delta":"世界"}
data: {"type":"text-end","id":"text-1"}
data: {"type":"finish","finishReason":"stop","usage":{...}}
data: [DONE]
`

### 3.3 协议差异分析

| 维度 | 当前后端格式 | AI SDK 格式 | 影响 |
|------|-------------|-------------|------|
| 事件结构 | `{event, data}` | `{type, ...payload}` | 结构不兼容 |
| 文本增量 | `content` 字段 | `delta` 字段 + `text-start/end` 包裹 | 需要转换 |
| 消息标识 | 无 | 需要 `id` 字段追踪流 | 需要添加 |
| 工具调用 | 未支持 | `tool-input-*`、`tool-output-*` 系列 | 需要扩展 |
| Reasoning | 未支持 | `reasoning-*` 系列 | 需要扩展 |
| 资源引用 | 未支持 | `source-url`、`source-document` | 需要扩展 |
| 结束标记 | `{"event":"close"}` | `data: [DONE]` | 需要适配 |

### 3.4 适配方案

**方案：直接改造 `/chat-messages` 接口**

`
┌─────────────────────────────────────────────────────────────────────┐
│                    后端架构改造                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Controller 层（薄）                                                │
│  └── POST /chat-messages         ← 直接改造为 AI SDK 标准接口       │
│      ├── 请求：AIChatRequest（替代 LLMChatCompletion）              │
│      └── 响应：AI SDK UIMessageChunk 格式                           │
│                                                                     │
│  阶段一（快速落地）                                                  │
│  └── 直接修改 run_llm_task() 事件处理逻辑                          │
│      └── event_queue.put({"type":"text-delta",...})                │
│                                                                     │
│  阶段二（技术债清理）                                                │
│  └── 引入 UIMessageChunkCallbackHandler                            │
│      └── extended/langchain/callbacks/ui_message_chunk_callback.py  │
│          ├── on_llm_new_token  → text-delta                        │
│          ├── on_tool_start     → tool-input-start                  │
│          ├── on_tool_end       → tool-output-available             │
│          └── on_chain_error    → error                             │
│                                                                     │
│  底层（不变）                                                       │
│  └── Agent / LLMService / AlonChatModel                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
`

**为什么选择直接改造**：
- 现有 `/chat-messages` 接口无前端调用者
- 避免维护两套接口
- 简化架构，减少适配层

## 4. 后端改造方案

### 4.1 改造层次

| 层次 | 文件位置 | 改造内容 | 说明 |
|------|----------|----------|------|
| Controllers | `ai/controllers/v1/chat/llm.py` | 改造 `/chat-messages` 接口 | 接收 AI SDK 标准请求格式 |
| Schemas | `ai/schemas/chat.py` | 新增 AI SDK 请求 Schema | `AIChatRequest`、`UIMessage`、`UIMessagePart` |
| Callbacks | `extended/langchain/callbacks/` | 阶段二新增 Callback Handler | `UIMessageChunkCallbackHandler` |

**阶段一改造**（最小改动）：
- 修改 `chat_messages` 函数接收 `AIChatRequest`
- 修改 `run_llm_task` 输出 AI SDK SSE 格式
- 修改 `_sse_generator` 输出格式

### 4.2 阶段二：UIMessageChunkCallbackHandler

``python
# server/python/src/extended/langchain/callbacks/ui_message_chunk_callback.py

"""LangChain AsyncCallbackHandler，输出 AI SDK UIMessageChunk 格式"""

import json
from typing import Any
from langchain_core.callbacks import AsyncCallbackHandler


class UIMessageChunkCallbackHandler(AsyncCallbackHandler):
    """将 LangChain 流式事件转换为 AI SDK UIMessageChunk 格式"""

    def __init__(self, event_queue: asyncio.Queue, message_id: str):
        self.event_queue = event_queue
        self.message_id = message_id
        self._text_id = f"text-{message_id}"

    async def on_llm_start(self, serialized, prompts, **kwargs):
        """LLM 开始：发送 text-start"""
        await self.event_queue.put({
            "type": "text-start",
            "id": self._text_id,
        })

    async def on_llm_new_token(self, token: str, **kwargs):
        """LLM 新 token：发送 text-delta"""
        await self.event_queue.put({
            "type": "text-delta",
            "id": self._text_id,
            "delta": token,
        })

    async def on_llm_end(self, response, **kwargs):
        """LLM 结束：发送 text-end"""
        await self.event_queue.put({
            "type": "text-end",
            "id": self._text_id,
        })

    async def on_tool_start(self, serialized, input_str, **kwargs):
        """工具开始：发送 tool-input-start"""
        await self.event_queue.put({
            "type": "tool-input-start",
            "toolCallId": kwargs.get("run_id", ""),
            "toolName": serialized.get("name", ""),
        })

    async def on_tool_end(self, output, **kwargs):
        """工具结束：发送 tool-output-available"""
        await self.event_queue.put({
            "type": "tool-output-available",
            "toolCallId": kwargs.get("run_id", ""),
            "output": str(output),
        })
``

import { useChat as useAIChat } from "ai/vue";
import type { AppUIMessage } from "../types";
import { useConversationStore } from "../stores/conversation";

export function useChat(options?: { conversationId?: string }) {
  const conversationStore = useConversationStore();

  return useAIChat<AppUIMessage>({
    api: "/api/v1/chat-messages",  // 代理到后端

## 6. 接口契约

### 6.1 AI SDK 标准请求格式

``typescript
// POST /api/v1/chat-messages
interface AIChatRequest {
  // 会话 ID（可选，不提供则创建新会话）
  id?: string;

  // 消息历史
  messages: Array<{
    id: string;
    role: "user" | "assistant" | "system";
    parts: Array<{
      type: string;
      [key: string]: any;
    }>;
  }>;

  // 触发类型
  trigger: "submit-message" | "regenerate-message";

  // 要重新生成的消息 ID（仅 regenerate-message 时）
  messageId?: string;

  // 扩展字段（后端自定义）
  body?: {
    model?: {
      provider: string;
      name: string;
      completionParams?: Record<string, any>;
    };
    search?: {
      enabled: boolean;
      provider?: string;
    };
    files?: Array<{
      type: string;
      transferMethod: string;
      url?: string;
      uploadFileId?: string;
    }>;
  };
}
``

### 6.2 AI SDK 标准响应格式

`
// SSE 流式响应

// 开始新消息
data: {"type":"start","messageId":"msg-xxx"}

// 文本内容
data: {"type":"text-start","id":"text-1"}
data: {"type":"text-delta","id":"text-1","delta":"你好"}
data: {"type":"text-delta","id":"text-1","delta":"，"}
data: {"type":"text-delta","id":"text-1","delta":"世界"}
data: {"type":"text-end","id":"text-1"}

// 思考过程（可选）
data: {"type":"reasoning-start","id":"reasoning-1"}
data: {"type":"reasoning-delta","id":"reasoning-1","delta":"让我思考..."}
data: {"type":"reasoning-end","id":"reasoning-1"}

// 工具调用（可选）
data: {"type":"tool-input-start","toolCallId":"tc-1","toolName":"search"}
data: {"type":"tool-input-delta","toolCallId":"tc-1","inputTextDelta":"{\"query\":\"天气\"}"}
data: {"type":"tool-input-available","toolCallId":"tc-1","toolName":"search","input":{"query":"天气"}}
data: {"type":"tool-output-available","toolCallId":"tc-1","output":{"result":"晴天"}}

// 资源引用（可选）
data: {"type":"source-url","sourceId":"src-1","url":"https://...","title":"参考文档"}

// 结束
data: {"type":"finish","finishReason":"stop","usage":{"promptTokens":10,"completionTokens":20}}
data: [DONE]
`

## 7. 实施路线

### 7.1 阶段一：基础模块搭建（MVP）

**目标**：创建 AI 模块骨架，实现基础对话功能

**前端任务**

1. 安装核心依赖：`pnpm add ai @ai-sdk/vue`
2. 安装 AI 对话组件：`npx ai-elements-vue@latest`（安装到 `components/ai-elements/`）
3. 创建 `web/vue/src/ai` 目录结构
4. 编写模块入口 `index.ts` 和路由配置
5. 在 `config/modules.ts` 中启用 ai 模块
6. 实现 `useChat.ts` 封装
7. 实现 ChatPage 页面，组装 ai-elements-vue 组件

**后端任务**

1. 新增 `AIChatRequest` Schema（`ai/schemas/chat.py`）
2. 修改 `chat_messages` 接收新请求格式
3. 修改 `run_llm_task` 输出 AI SDK SSE 格式（直接修改，不引入 Callback Handler）
4. 修改 `_sse_generator` 输出格式
5. 标记 `LLMChatCompletion` 为 deprecated

**验证**：发送消息 → 收到流式响应

### 7.2 阶段二：技术债清理（Callback Handler）

**目标**：引入 Callback Handler，职责分离

**后端任务**

1. 实现 `UIMessageChunkCallbackHandler`（`extended/langchain/callbacks/`）
2. 重构 `chat_messages` 使用 Callback 方式
3. 添加工具调用支持（`tool-input-*` / `tool-output-*`）

**前端任务**

1. 安装 ai-elements-vue 的 `tool`、`confirmation` 组件
2. 支持工具调用可视化

### 7.3 阶段三：高级特性

**目标**：支持会话管理、模型选择、多模态

**前端任务**

1. 会话管理页面（列表、切换、删除）
2. 模型选择器组件（动态提供商/模型列表）
3. 联网搜索可视化
4. 图片上传

**后端任务**

1. 会话列表 API
2. 模型列表 API
3. 支持 `reasoning-*`、`source-*` 事件

## 8. 风险与注意事项

### 8.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| ai-elements-vue API 不稳定 | 组件可能需要调整 | 源码复制到项目，锁定版本 |
| 阶段一代码不够优雅 | 技术债 | 阶段二重构，明确技术债 |
| 模型选择需要后端支持动态列表 | 功能延迟 | 阶段三实现，阶段一用静态配置 |
| AI SDK 版本迭代 | API 可能变化 | 锁定版本，跟踪更新日志 |

### 8.2 兼容性考虑

- 现有 `/chat-messages` 接口无前端调用者，可安全改造
- 旧 Schema 标记为 deprecated 但保留

### 8.3 性能考虑

- 流式响应避免一次性加载大消息
- ai-elements-vue 内置 `vue-stick-to-bottom` 自动滚动
- ai-elements-vue 内置 `vue-stream-markdown` 流式 Markdown 渲染
- 长对话需考虑虚拟滚动优化
