# 插件管理功能设计方案

## 1. 背景与目标

### 1.1 设计基础

本文档基于《插件与模型资源管理 Tenant 统一管理设计方案》（docs/designs/3.插件与模型资源管理 Tenant 统一管理设计方案.md）的架构设计，明确 Tenant 模块和 AI 模块的插件管理功能。

### 1.2 职责划分

```
┌─────────────────────────────────────────────────────────────────┐
│  Tenant 模块（管理面）        │  AI 模块（使用面）            │
│  "有什么"                    │  "用什么"                     │
├─────────────────────────────┼───────────────────────────────┤
│  插件定义                   │  插件安装                     │
│  资源注册表                 │  租户配置                     │
│  全局元数据                 │  运行时实例                   │
└─────────────────────────────────────────────────────────────────┘
```

**核心原则**：

- Tenant 模块负责插件定义管理（注册、查询、标记推荐/禁用）
- AI 模块负责插件安装使用（安装、卸载、启动、停止、配置、监控）
- Tenant 模块不负责插件的安装，安装功能都在 AI 模块中

### 1.3 用户角色

| 角色 | 职责 | 操作入口 |
|------|------|----------|
| 平台管理员 | 管理插件定义 | Tenant 管理后台 |
| 租户管理员 | 安装使用插件 | AI 使用侧前台 |

### 2.2 插件隔离策略

插件支持三种安装类型：

| 安装类型 | 说明 | 适用场景 |
|----------|------|----------|
| `shared` | 共享进程（默认） | 资源占用低，适合大多数场景 |
| `.venv` | 虚拟环境隔离 | 需要 Python 依赖隔离的场景 |
| `docker` | Docker 容器隔离 | 需要完整环境隔离的场景 |

**当前阶段**：默认使用 `shared` 模式，后续迭代开放其他方式。

## 3. Tenant 模块管理侧功能

### 3.1 插件定义注册

支持三种注册方式：

#### 3.1.1 服务器目录扫描

```
┌─────────────────────────────────────────────────────────────────┐
│                     服务器目录扫描流程                          │
└─────────────────────────────────────────────────────────────────┘

[server/plugins/*.zip] ─────▶ [解析 manifest] ─────▶ [注册到 plugin_definitions]
                                      │
                                      ├── 验证包格式
                                      ├── 提取元数据
                                      └── 计算校验和
```

**API**：

```python
POST /tenant/admin/v1/plugin-definitions/scan
Request: {
    "directory": "server/plugins",  # 可选，默认 server/plugins
    "recursive": true               # 是否递归扫描子目录
}
Response: {
    "scanned": 8,           # 扫描到的插件包数量
    "registered": 6,        # 成功注册数量
    "skipped": 2,           # 已存在跳过数量
    "failed": 0,            # 失败数量
    "details": [            # 每个插件的处理结果
        {
            "file": "langgenius-ollama_1.0.0.zip",
            "plugin_id": "langgenius/ollama",
            "status": "registered",
            "message": null
        },
        {
            "file": "existing-plugin_1.0.0.zip",
            "plugin_id": "existing/plugin",
            "status": "skipped",
            "message": "插件定义已存在"
        }
    ]
}
```

#### 3.1.2 本地上传 zip 包

```
┌─────────────────────────────────────────────────────────────────┐
│                     本地上传注册流程                            │
└─────────────────────────────────────────────────────────────────┘

[前端上传] ─────▶ [临时存储] ─────▶ [解析验证] ─────▶ [上传到 OSS] ─────▶ [注册]
                                       │
                                       ├── 格式校验
                                       ├── 安全扫描（可选）
                                       └── 存储到 MinIO
```

**API**：

```python
POST /tenant/admin/v1/plugin-definitions/upload
Content-Type: multipart/form-data
Request:
    - plugin_file: <zip file>
    - overwrite: false  # 是否覆盖已存在的定义
Response: {
    "plugin_id": "langgenius/ollama",
    "version": "1.0.0",
    "unique_identifier": "langgenius/ollama:1.0.0@abc123",
    "registered_at": "2026-06-26T10:00:00Z"
}
```

#### 3.1.3 远程 URL 拉取

```
┌─────────────────────────────────────────────────────────────────┐
│                     远程 URL 拉取流程                           │
└─────────────────────────────────────────────────────────────────┘

[输入 URL] ─────▶ [下载插件包] ─────▶ [解析验证] ─────▶ [上传到 OSS] ─────▶ [注册]
                        │
                        ├── HTTPS 校验
                        ├── 大小限制
                        └── 超时控制
```

**API**：

```python
POST /tenant/admin/v1/plugin-definitions/fetch
Request: {
    "url": "https://marketplace.example.com/plugins/langgenius-ollama_1.0.0.zip",
    "overwrite": false
}
Response: {
    "plugin_id": "langgenius/ollama",
    "version": "1.0.0",
    "unique_identifier": "langgenius/ollama:1.0.0@abc123",
    "registered_at": "2026-06-26T10:00:00Z"
}
```

```python
PATCH /tenant/admin/v1/plugin-definitions/{plugin_id}
Request: {
    "is_recommended": true,   # 标记为推荐
    "is_enabled": true        # 启用/禁用
}
Response: {
    "plugin_id": "langgenius/ollama",
    "is_recommended": true,
    "is_enabled": true,
    "updated_at": "..."

```

```python
DELETE /tenant/admin/v1/plugin-definitions/{plugin_id}
Response: {
    "plugin_id": "langgenius/ollama",
    "deleted_at": "..."
}
```

**前置检查**：

- `refers > 0` 时禁止删除（有租户在使用）
- 需先确认所有租户都已卸载

## 4. AI 模块使用侧功能

### 4.1 获取可用插件列表

租户管理员浏览 Tenant 模块已注册的插件定义。

```
┌─────────────────────────────────────────────────────────────────┐
│                     租户管理员浏览可用插件                      │
└─────────────────────────────────────────────────────────────────┘

[AI API] ─────▶ [Tenant Provider] ─────▶ [plugin_definitions]
                    │
                    ├── 过滤已启用的插件
                    ├── 标记是否已安装
                    └── 返回定义列表 + 安装状态
```

**API**：

```python
GET /ai/console/v1/plugins/available
Query:
    - keyword: str
    - type: str         # model, tool, agent
    - is_recommended: bool
    - page: int
    - page_size: int
Response: {
    "items": [
        {
            "plugin_id": "langgenius/ollama",
            "name": "Ollama",
            "version": "1.0.0",
            "type": "model",
            "icon": "https://...",
            "description": "Ollama 模型供应商",
            "author": "langgenius",
            "is_recommended": true,
            "is_installed": true,           # 当前租户是否已安装
            "installation_status": "ACTIVE"  # 如果已安装，显示状态
        }
    ],
    "total": 12,
    "page": 1,
    "page_size": 20
}
```

### 4.2 安装/卸载插件

#### 4.2.1 安装流程（事件驱动）

```
┌─────────────────────────────────────────────────────────────────┐
│                     插件安装流程（事件驱动）                    │
└─────────────────────────────────────────────────────────────────┘

┌─────────┐    ┌──────────────────┐    ┌─────────────────┐
│ 租户    │───▶│ AI API          │───▶│ PluginManager   │
│ 管理员  │    │ install_plugin  │    │                 │
└─────────┘    └──────────────────┘    └────────┬────────┘
                                                  │
                         ┌────────────────────────┴──────┐
                         ▼                               ▼
              ┌─────────────────────┐      ┌─────────────────────┐
              │ Tenant Provider     │      │ AI Schema           │
              │ create_installation │      │ plugin_configs      │
              │ (独立事务)          │      │ plugin_runtime_     │
              │                     │      │ states              │
              └──────────┬──────────┘      └──────────┬──────────┘
                         │                            │
                         │   成功                     │ 失败
                         │                            ▼
                         │              ┌─────────────────────┐
                         │              │ 发布事件            │
                         │              │ PluginInstallation  │
                         │              │ Failed              │
                         │              └──────────┬──────────┘
                         │                         │
                         │                         ▼
                         │              ┌─────────────────────┐
                         │              │ Tenant 监听器       │
                         │              │ 删除安装记录        │
                         │              └─────────────────────┘
                         ▼
              ┌─────────────────────┐
              │ 返回安装结果        │
              └─────────────────────┘
```

#### 4.2.2 安装 API

```python
POST /ai/console/v1/plugins/installations
Request: {
    "plugin_id": "langgenius/ollama",
    "auto_start": true,
    "install_type": "shared",      # shared | .venv | docker（默认 shared）
    "install_config": {            # 可选的安装配置
        "env_vars": {...}
    }
}
Response: {
    "installation_id": "...",
    "plugin_id": "langgenius/ollama",
    "status": "PENDING",
    "task_id": "..."               # 安装任务 ID，可追踪进度
}
```

#### 4.2.3 卸载 API

```python
DELETE /ai/console/v1/plugins/installations/{plugin_id}
Response: {
    "plugin_id": "langgenius/ollama",
    "status": "UNINSTALLED"
}
```

### 4.3 启动/停止插件

#### 4.3.1 状态流转

```
         安装请求
             │
             ▼
      ┌─────────────┐
      │   PENDING   │  ← 安装中
      └──────┬──────┘
             │
    ┌────────┴────────┐
    ▼                 ▼
┌─────────┐     ┌──────────┐
│ ACTIVE  │     │  FAILED  │
└────┬────┘     └──────────┘
     │
     │ 停止
     ▼
┌──────────┐
│ INACTIVE │
└────┬─────┘
     │
     │ 启动
     ▼
┌─────────┐
│ ACTIVE  │
└─────────┘
```

#### 4.3.2 启动 API

```python
POST /ai/console/v1/plugins/installations/{plugin_id}/start
Response: {

```

#### 4.3.3 停止 API

```python

```

## 8. 风险与注意事项

### 8.1 数据一致性

**风险**：安装过程中 AI 侧创建失败，Tenant 侧记录已存在。

**缓解措施**：

- 实现事件处理器立即回滚
- 添加定时任务清理孤儿记录
- 记录详细日志便于手动排查

### 8.2 缓存一致性

`PluginManager.plugins` 是内存缓存，从 Provider 加载。如果其他进程修改了 `tenant.plugin_installations`，当前进程的插件管理器感知不到。

**缓解措施**：缓存 TTL = 60 秒，过期后重新从 Provider 加载。

### 8.3 插件定义删除

**风险**：删除插件定义时，可能还有租户在使用。

**缓解措施**：

- `refers > 0` 时禁止删除
- 需先确认所有租户都已卸载
- 提供"禁用"操作作为软删除
