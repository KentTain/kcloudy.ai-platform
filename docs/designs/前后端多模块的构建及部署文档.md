# 前后端多模块的构建及部署文档

## 概述

本文档描述了 InitProject 平台的多模块构建和部署方案，支持平台版（全模块）和独立模块版两种部署模式。

## 架构设计

### 部署模式

```
部署模式：
├─ 平台版 (platform.kcloudy.com:3000)
│  ├─ backend-platform-api (Web)
│  ├─ backend-platform-task (Task)
│  └─ backend-platform-listener (Listener)
│
└─ 独立模块版
   ├─ Demo 模块 (demo.kcloudy.com:8010)
   │  ├─ backend-demo-api
   │  ├─ backend-demo-task
   │  └─ backend-demo-listener
   ├─ IAM 模块 (iam.kcloudy.com:8020)
   │  └─ backend-iam-api
   └─ Tenant 模块 (tenant.kcloudy.com:8030)
      └─ backend-tenant-api
```

### 技术选型

| 组件 | 前端 | 后端 |
|------|------|------|
| 运行时 | Node.js 22 | Python 3.12 |
| 包管理器 | pnpm | uv |
| 构建工具 | Vite | - |
| 容器基础镜像 | nginx:alpine | python:3.12-slim-bookworm |

## Docker Compose 文件结构

```text
docker/
├── docker-compose.infrastructure.yml  # 基础设施服务
├── docker-compose.backend.yml         # 后端服务
└── docker-compose.frontend.yml        # 前端服务
```

### 文件说明

| 文件 | 说明 |
|------|------|
| docker-compose.infrastructure.yml | 数据库、缓存、对象存储、网关等基础设施 |
| docker-compose.backend.yml | Python FastAPI 后端服务 |
| docker-compose.frontend.yml | Vue 前端服务 |

## 前端构建方案

### Dockerfile 结构

前端采用多阶段构建：

1. **Builder 阶段**：安装依赖、生成模块配置、构建应用
2. **Production 阶段**：使用 Nginx 提供静态文件服务

### 构建参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| BUILD_MODULES | demo,iam,tenant | 要打包的模块列表 |

### 模块配置生成

构建时自动生成 `src/config/modules.ts`：

```typescript
// Auto-generated by Docker build
// BUILD_MODULES: demo,iam,tenant
export const ENABLED_MODULES = ['demo', 'iam', 'tenant'] as const;
export type EnabledModule = typeof ENABLED_MODULES[number];
```

### 构建示例

```bash
# 平台版（全模块）
docker build -t init-project-platform:latest \
  --build-arg BUILD_MODULES=demo,iam,tenant \
  ./web/vue

# Demo 模块独立版
docker build -t init-project-demo:latest \
  --build-arg BUILD_MODULES=demo \
  ./web/vue
```

## 后端构建方案

### Dockerfile 结构

后端采用多阶段构建：

1. **Builder 阶段**：安装依赖、生成模块配置
2. **Production 阶段**：复制虚拟环境和源代码

### 构建参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| BUILD_MODULES | demo,iam,tenant | 要打包的模块列表 |
| APP_ROLE | web | 应用角色（web/task/listener） |

### 模块配置生成

构建时自动生成 `src/config/modules.py`：

```python
# Auto-generated by Docker build
# BUILD_MODULES: demo,iam,tenant
ENABLED_MODULES = ["demo", "iam", "tenant"]
```

### 应用入口修改

三个应用入口（`application_web.py`、`application_task.py`、`application_listener.py`）统一读取配置：

```python
def create_app(module_names: list[str] | None = None) -> FastAPI:
    # 如果未指定模块，尝试从配置文件读取
    if module_names is None:
        try:
            from config.modules import ENABLED_MODULES
            module_names = ENABLED_MODULES
            _logger.info(f"Loaded modules from config: {ENABLED_MODULES}")
        except ImportError:
            _logger.info("No modules config found, loading all modules")
    
    # 加载模块
    src_path = Path(__file__).parent
    modules = load_modules(src_path, module_names)
    ...
```

### 角色分离

通过 `docker-entrypoint.sh` 脚本实现角色分离：

```bash
#!/bin/bash
set -e

ROLE="${APP_ROLE:-web}"
HOST="${HOST:-0.0.0.0}"
PORT="${PORT:-8000}"

case "${ROLE}" in
  web)
    exec uv run uvicorn application_web:app --host ${HOST} --port ${PORT}
    ;;
  task)
    exec uv run runtask
    ;;
  listener)
    exec uv run runlistener
    ;;
  *)
    echo "Error: Unknown APP_ROLE '${APP_ROLE}'. Must be one of: web, task, listener"
    exit 1
    ;;
esac
```

### 构建示例

```bash
# 平台版 Web API
docker build -t init-project-backend-platform:latest \
  --build-arg BUILD_MODULES=demo,iam,tenant \
  --build-arg APP_ROLE=web \
  ./server/python

# Demo 模块 Task Scheduler
docker build -t init-project-backend-demo:latest \
  --build-arg BUILD_MODULES=demo \
  --build-arg APP_ROLE=task \
  ./server/python
```

## Docker Compose 部署

### 服务矩阵

**平台版服务：**

| 服务名 | 模块 | 角色 | 端口 |
|--------|------|------|------|
| platform-app | demo,iam,tenant | - | 3000 |
| backend-platform-api | demo,iam,tenant | web | 8000 |
| backend-platform-task | demo,iam,tenant | task | - |
| backend-platform-listener | demo,iam,tenant | listener | - |

**独立模块版服务（profile: standalone）：**

| 服务名 | 模块 | 角色 | 端口 |
|--------|------|------|------|
| demo-app | demo | - | 3010 |
| backend-demo-api | demo | web | 8010 |
| backend-demo-task | demo | task | - |
| backend-demo-listener | demo | listener | - |
| iam-app | iam | - | 3020 |
| backend-iam-api | iam | web | 8020 |
| tenant-app | tenant | - | 3030 |
| backend-tenant-api | tenant | web | 8030 |

### 启动命令

```bash
# 启动基础设施
docker compose -f docker/docker-compose.infrastructure.yml up -d

# 启动平台版（默认）
docker compose \
  -f docker/docker-compose.infrastructure.yml \
  -f docker/docker-compose.backend.yml \
  -f docker/docker-compose.frontend.yml \
  up -d

# 启动独立模块版
docker compose \
  -f docker/docker-compose.infrastructure.yml \
  -f docker/docker-compose.backend.yml \
  -f docker/docker-compose.frontend.yml \
  --profile standalone \
  up -d demo-app backend-demo-api

# 启动所有服务
docker compose \
  -f docker/docker-compose.infrastructure.yml \
  -f docker/docker-compose.backend.yml \
  -f docker/docker-compose.frontend.yml \
  --profile standalone \
  up -d
```

## 依赖解析

### 模块依赖关系

```
demo → iam → tenant
```

### 自动依赖解析

构建时自动解析依赖：

- `BUILD_MODULES=demo` → 包含 demo, iam, tenant
- `BUILD_MODULES=iam` → 包含 iam, tenant
- `BUILD_MODULES=tenant` → 仅包含 tenant

依赖解析由 `load_modules()` 函数自动完成。

## 故障排查

### 常见问题

1. **模块未加载**
   - 检查 `BUILD_MODULES` 参数格式
   - 查看容器日志确认配置文件生成

2. **角色启动失败**
   - 检查 `APP_ROLE` 值是否正确
   - 查看入口脚本日志输出

3. **数据库连接失败**
   - 确认 PostgreSQL 服务健康
   - 检查 `DATABASE_URL` 环境变量

### 调试命令

```bash
# 查看容器日志
docker logs init-project-backend-platform-api

# 进入容器调试
docker exec -it init-project-backend-platform-api /bin/bash

# 查看生成的配置文件
docker exec init-project-backend-platform-api cat /app/src/config/modules.py
```
