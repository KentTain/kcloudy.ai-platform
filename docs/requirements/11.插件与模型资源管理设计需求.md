# 插件与模型资源管理设计需求

本文档描述 Tenant 模块与 AI 模块之间关于插件和模型资源的协作管理方案。

---

## 1. 背景与问题

### 1.1 当前架构

```
PostgreSQL Schemas:
├── tenant (schema)
│   ├── tenants                    # 租户主表
│   ├── database_configs           # 数据库配置
│   ├── storage_configs            # 存储配置
│   ├── cache_configs              # 缓存配置
│   ├── queue_configs              # 队列配置
│   ├── pubsub_configs             # 发布订阅配置
│   ├── modules                    # 模块定义
│   └── tenant_modules             # 租户-模块分配
│
└── ai (schema)
    ├── plugins                    # 插件（全局，无 tenant_id）
    ├── plugin_installations       # 插件安装记录（租户级，有 tenant_id）
    └── model_providers            # 模型提供商（租户级，有 tenant_id）
```

### 1.2 核心问题

1. **资源管理分散**：PluginInstallation 和 ModelProvider 在 AI 模块，但属于"租户资源"
2. **配置存储混乱**：模型配置（API Key 等）在 AI 模块，应该和资源配置统一管理
3. **配额无法控制**：Tenant 模块对 AI 资源无感知，无法统一分配和限制
4. **模型依赖插件**：模型提供者是插件的一部分，无法独立于插件存在

## 2. 模型与插件的依赖关系

### 2.1 核心结论

**模型依赖于插件，需要先安装插件才能使用模型。**

### 2.2 调用链分析

```
用户请求使用模型
       │
       ▼
ModelProviderFactory.get_providers()
       │
       ▼
ModelClient.fetch_model_providers(tenant_id)
       │
       ▼
┌───────────────────────────────────────────────────────────────┐
│  PluginManager.plugins (遍历租户已安装的所有插件)              │
│                                                               │
│   for plugin_name, plugin_info in plugin_manager.plugins:    │
│       for config in plugin_info.config.models_configuration: │
│           yield PluginModelProviderEntity(...)               │
│                                                               │
└───────────────────────────────────────────────────────────────┘
       │
       ▼
返回可用的模型提供者列表
```

### 2.3 插件清单结构

```yaml
# 插件的 manifest.yaml
plugins:
  models:                    # 声明此插件提供哪些模型
    - models/openai.yaml     # OpenAI 模型提供者
    - models/anthropic.yaml  # Anthropic 模型提供者
```

### 2.4 Provider ID 格式

```
格式：organization/plugin_name/provider_name

例如：langgenius/openai/openai
- langgenius: 组织名
- openai: 插件名
- openai: 提供者名
```

## 3. 管理边界划分

### 3.1 职责划分

| Tenant 模块负责 | AI 模块负责 |
|----------------|-------------|
| 插件安装记录 | 插件包管理 |
| 模型配置存储（credentials） | 插件运行时 |
| 配额和权限 | 进程生命周期 |
| 凭证管理 | 模型调用执行 |
| 租户级资源分配 | 全局插件注册表 |

### 3.2 数据流向

```
Tenant ──────安装/卸载/配置──────► AI
          获取运行时状态◄────────
```

### 3.3 资源分类

| 资源类型 | 管理粒度 | Tenant 模块应该管理什么 |
|----------|----------|------------------------|
| 基础设施资源（database/storage/cache/queue/pubsub） | 配置模板 | 管理：分配给租户使用 |
| 插件 | 租户级安装 | 管理：允许/禁止安装哪些插件、插件数量配额、插件权限控制 |
| 模型提供者 | 插件的子资源 | 间接管理：通过插件安装间接管理、模型调用配额、凭证管理 |

## 4. 数据模型设计

### 4.1 Tenant 模块（新增/移动）

#### 4.1.1 PluginInstallation（从 AI 移动）

```python
# tenant/models/plugin_installation.py
class PluginInstallation(BaseModel, TenantMixin):
    """插件安装记录 - 移动到 Tenant 模块"""
    __tablename__ = "plugin_installations"

    plugin_id: Mapped[str]           # 插件 ID
    status: Mapped[str]              # 安装状态
    auto_start: Mapped[bool]         # 是否自动启动
    installed_at: Mapped[datetime]   # 安装时间

    # 配额相关
    max_calls_per_day: Mapped[int | None]  # 每日调用上限
    max_storage_size: Mapped[int | None]   # 存储上限
```

#### 4.1.2 ModelProviderConfig（新增）

```python
# tenant/models/model_provider_config.py
class ModelProviderConfig(BaseModel, TenantMixin):
    """模型提供者配置 - 新增"""
    __tablename__ = "model_provider_configs"

    provider_name: Mapped[str]       # 如 "openai"
    plugin_installation_id: Mapped[str]  # 关联到插件安装
    credentials: Mapped[dict]        # 加密的凭证
    is_default: Mapped[bool]         # 是否默认
    is_valid: Mapped[bool]           # 是否有效

    # 配额
    max_tokens_per_month: Mapped[int | None]
    max_requests_per_day: Mapped[int | None]
```

### 4.2 AI 模块（保留/简化）

#### 4.2.1 Plugin（保留，全局）

```python
# ai/models/plugin.py
class Plugin(BaseModel):
    """插件元数据 - 全局，无 tenant_id"""
    __tablename__ = "plugins"

    plugin_id: Mapped[str]
    plugin_unique_identifier: Mapped[str]
    refers: Mapped[int]              # 引用计数
    manifest: Mapped[dict]           # 插件清单
```

#### 4.2.2 PluginRuntime（保留，内存）

```python
# ai/components/plugin/engine/core/runtime.py
class PluginRuntime:
    """插件运行时 - 内存中的运行状态"""

    process_id: int | None
    port: int | None
    state: str
    # 不持久化到数据库
```

## 6. 业务流程设计

### 6.1 插件安装流程

```
用户安装插件
       │
       ▼
┌─────────────────────────────────────────────────────────────────┐
│  Tenant 模块                                                    │
│                                                                │
│  1. 检查配额（max_plugins）                                      │
│  2. 创建 PluginInstallation 记录                                │
│  3. 记录 plugin_id, tenant_id, status                           │
│                                                                │
└─────────────────────────────────────────────────────────────────┘
       │
       │ 调用 AI 模块接口
       ▼
┌─────────────────────────────────────────────────────────────────┐
│  AI 模块                                                        │
│                                                                │
│  1. 下载/解压插件包                                             │
│  2. 创建运行时环境（venv）                                      │
│  3. 启动插件进程                                                │
│  4. 运行时状态管理                                              │
│                                                                │
└─────────────────────────────────────────────────────────────────┘
```

### 6.2 模型配置流程

```
用户配置模型 API Key
       │
       ▼
┌─────────────────────────────────────────────────────────────────┐
│  Tenant 模块                                                    │
│                                                                │
│  1. 检查插件是否已安装                                          │
│  2. 创建/更新 ModelProviderConfig                               │
│     - provider_name: "openai"                                   │
│     - plugin_id: "langgenius/openai"                            │

│     - is_default: true                                          │

└─────────────────────────────────────────────────────────────────┘
       │
       │ 调用时获取配置
       ▼
┌─────────────────────────────────────────────────────────────────┐
│  AI 模块                                                        │
│                                                                │
│  1. 从 Tenant 模块获取 credentials                              │
│  2. 调用插件执行模型请求                                        │
│                                                                │
└─────────────────────────────────────────────────────────────────┘
```

## 7. 运行环境设计

### 7.1 隔离级别分析

| 级别 | 描述 | 隔离性 | 复杂度 | 安全性 |
|------|------|--------|--------|--------|
| 无隔离 | 所有插件共享主进程环境 | ★☆☆☆☆ | ★☆☆☆☆ | ★☆☆☆☆ |
| 进程隔离 | 每个插件独立进程，共享 Python | ★★☆☆☆ | ★★☆☆☆ | ★★☆☆☆ |
| venv 隔离 | 每个插件独立虚拟环境 | ★★★☆☆ | ★★★☆☆ | ★★★☆☆ |
| 容器隔离 | 每个插件独立容器 | ★★★★★ | ★★★★★ | ★★★★★ |

### 7.2 推荐方案：分级运行环境

```
                    ┌─────────────────────────────────────┐
                    │       PluginEnvironmentManager      │
                    │                                     │
                    │  管理插件的运行环境生命周期          │
                    └─────────────────────────────────────┘
                                    │
            ┌───────────────────────┼───────────────────────┐
            ▼                       ▼                       ▼
     ┌─────────────┐        ┌─────────────┐        ┌─────────────┐
     │ SharedEnv   │        │ IsolatedEnv │        │ SandboxEnv  │
     │             │        │             │        │             │
     │ 共享环境    │        │ 独立 venv   │        │ 容器隔离    │
     │             │        │             │        │             │
     │ • 内置插件  │        │ • 第三方    │        │ • 不可信    │
     │ • 系统插件  │        │ • 自定义    │        │ • 沙箱测试  │
     │ • 可信插件  │        │ • 高资源    │        │             │
     └─────────────┘        └─────────────┘        └─────────────┘
```

### 7.3 目录结构设计

```text
/plugins/
```

## 8. 数据流与权限设计

### 8.1 统一数据流模式

```
                    ┌─────────────────────────────────────┐
                    │           Tenant 模块               │
                    │                                     │
                    │  Module { code: "ai", ... }        │
                    │  ┌─────────────────────────────┐   │
                    │  │ ModulePermission            │   │
                    │  │ • ai:plugin:install         │   │
                    │  │ • ai:plugin:start           │   │
                    │  │ • ai:model:configure        │   │
                    │  │ • ai:model:invoke           │   │
                    │  └─────────────────────────────┘   │
                    │  ┌─────────────────────────────┐   │
                    │  │ ModuleRole                  │   │
                    │  │ • ai_admin                  │   │
                    │  │ • ai_user                   │   │
                    │  └─────────────────────────────┘   │
                    └─────────────────────────────────────┘
                                    │
                                    │ TenantModule (分配)
                                    │ tenant_modules:
                                    │   tenant_id + module_id="ai"
                                    ▼
┌──────────────────────────────────────────────────────────────────────┐
│                          AI 模块                                      │
│                                                                       │
│   权限校验流程：                                                       │
│                                                                       │
│   1. 请求进入 → get_tenant_id()                                       │
│   2. 检查 TenantModule 是否分配 AI 模块                               │
│   3. 检查用户是否有对应权限 (ai:plugin:install 等)                    │
│   4. 执行业务逻辑                                                     │
│                                                                       │
│   数据隔离：                                                          │
│   • PluginInstallation.tenant_id                                     │
│   • ModelProviderConfig.tenant_id                                    │
│                                                                       │
└──────────────────────────────────────────────────────────────────────┘
```

### 8.2 AI 模块权限定义

```python
# tenant/services/module_service.py 中注册 AI 模块时定义

ai_permissions = [
    {"name": "安装插件", "code": "ai:plugin:install", "resource": "plugin", "action": "write"},
    {"name": "启动插件", "code": "ai:plugin:start", "resource": "plugin", "action": "write"},
    {"name": "停止插件", "code": "ai:plugin:stop", "resource": "plugin", "action": "write"},
    {"name": "配置模型", "code": "ai:model:configure", "resource": "model", "action": "write"},
    {"name": "调用模型", "code": "ai:model:invoke", "resource": "model", "action": "read"},
]

ai_roles = [
    {"name": "AI 管理员", "code": "ai_admin", "permissions": ["ai:plugin:*", "ai:model:*"]},
    {"name": "AI 用户", "code": "ai_user", "permissions": ["ai:model:invoke"]},
]
```

## 10. 预期收益

1. **统一资源管理**：Tenant 模块管理所有租户级资源（包括插件和模型）
2. **职责清晰**：Tenant 管"有什么"，AI 管"怎么用"
3. **最小改动**：主要是数据模型移动，接口层保持不变
4. **配额可控**：可以在 Tenant 层面统一限制插件和模型的使用量
5. **安全隔离**：分级运行环境确保插件间的隔离
