# 租户模块前端功能设计

## 1. 概述

### 1.1 设计目标

基于后端 `tenant-module-system` 变更已完成的 API 实现，配套实现租户管理模块的前端管理界面，提供资源配置、模块管理、租户资源绑定、租户模块分配等功能的前端交互。

### 1.2 设计约束

- 仅实现需求文档描述的功能，不自行扩展
- 复用已迁移的通用组件（DataTable、MessageBox 等）
- 遵循 Vue 3 + TypeScript + shadcn-vue 技术栈
- 参照 Alon 项目系统管理页面的交互风格

## 2. 页面结构

### 2.1 路由结构

```
/admin
├── /tenants                           租户列表（更新）
│   ├── /create                        创建租户
│   └── /:id                           租户详情（扩展 Tab）
│       ├── Tab: 基本信息
│       ├── Tab: 资源绑定
│       └── Tab: 模块分配
│
├── /resource-configs                  资源配置管理（新建）
│   ├── Tab: 数据库
│   ├── Tab: 存储
│   ├── Tab: 缓存
│   ├── Tab: 队列
│   └── Tab: 发布订阅
│
└── /modules                           模块管理（新建）
    ├── /create                        创建模块
    └── /:id                           模块详情
        ├── Tab: 基本信息
        ├── Tab: 菜单管理
        ├── Tab: 权限管理
        └── Tab: 角色管理
```

### 2.2 页面组织决策

| 页面 | 组织方式 | 说明 |
|------|----------|------|
| 资源配置管理 | Tab 切换统一页面 | 5 种资源类型结构相似，Tab 切换减少页面跳转 |
| 模块详情页 | Tab 切换单一页面 | 包含 4 个子功能（基本信息/菜单/权限/角色），Tab 切换保持上下文 |
| 租户详情页 | 扩展为 Tab 结构 | 新增资源绑定和模块分配 Tab |

### 4.3 菜单编辑交互

采用弹窗表单方式编辑菜单：

- 点击「新增菜单」弹出 Dialog
- 表单包含：菜单名称、编码、路径、父菜单（下拉选择）、图标、排序
- 父菜单通过 TreeSelect 选择，过滤当前菜单及其子菜单
- 提交后刷新菜单树

## 8. 统计卡片实现

### 8.1 数据来源

| 统计项 | 数据来源 |
|--------|----------|
| 资源配置总数 | 列表数据长度 |
| 已被引用 | 列表数据中 `tenant_count > 0` 的数量 |
| 未被使用 | 列表数据中 `tenant_count === 0` 的数量 |
| 模块总数 | 列表数据长度 |
| 启用模块 | 列表数据中 `is_active = true` 的数量 |
| 必须模块 | 列表数据中 `is_need = true` 的数量 |
| 已分配次数 | 需要额外统计或遍历模块列表累加 |

### 8.2 实现策略

- 列表查询时使用较大的 page_size（如 100）获取完整数据
- 前端根据列表数据计算统计值
- 如需精确统计，后续可添加后端统计 API

## 9. 错误处理

### 9.1 业务错误

| 场景 | 错误信息 | 处理方式 |
|------|----------|----------|
| 删除已被引用的配置 | "配置已被租户引用，无法删除" | Toast 提示 |
| 创建重复编码模块 | "模块编码已存在" | 表单错误提示 |
| 删除已分配模块 | "模块已被租户分配，无法删除" | Toast 提示 |
| 修改系统内置角色 | "系统内置角色禁止修改" | Toast 提示 |
| 取消必须模块分配 | "必须模块禁止取消" | Toast 提示 |

### 9.2 连通性测试错误

连通性测试失败时，显示具体的错误信息（如：连接超时、认证失败等）。
