# 前端 PC 端 UI 现代化迁移设计需求

本文档描述前端 PC 端 Vue 技术栈从手写布局 + Element Plus 混用，向 shadcn-vue 体系的现代化迁移方案。

**核心原则**：只借鉴 UI 框架层的组件组合模式与布局设计，保留现有的 framework/、demo/、iam/ 模块目录结构、路由架构、store/api 层不动。

## 1. 现状差距

### 1.1 布局壳

| 维度 | 当前实现 | 目标（借鉴 kbhub） |
|------|----------|---------------------|
| 壳入口 | `AdminLayout.vue` 手写 fixed sidebar + navbar | `SidebarProvider` + `Sidebar` + `SidebarInset` 组合 |
| 侧边栏 | `AppSidebar.vue` 320 行手写样式/逻辑/硬编码 SVG | shadcn Sidebar 组件 + `AppNavMain.vue` 分组菜单 |
| 顶部导航 | `AppNavbar.vue` 面包屑 + 两个 action 按钮 | Header：SidebarTrigger + Breadcrumb + 快捷导航 + 命令搜索 + 用户下拉 |
| 标签页 | `AppTagsView.vue` 传统后台标签页 | 保留或弱化（按项目定位决定） |
| 移动端 | `appStore.setDevice()` 手动判断 + `transform: translateX(-100%)` | shadcn Sidebar 内置 Sheet 处理 |

## 3. Change #1: framework-page-skeleton

### 3.1 目标

创建统一的页面骨架组件，让所有业务页面有一致的标题、描述、操作区布局。

### 3.2 借鉴来源

kbhub `KbhubPage.vue`（38 行），提供 title/eyebrow/description/actions/variant 的标准页面头部。

### 3.3 产出

| 文件 | 操作 | 说明 |
|------|------|------|
| `framework/layouts/components/AppPage.vue` | 新增 | 页面骨架组件 |

### 3.4 AppPage 设计

**Props**：

| Prop | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| title | string | 必填 | 页面主标题 |
| eyebrow | string | — | 标题上方小字（如模块名） |
| description | string | — | 页面功能描述 |
| variant | `list \| workbench \| detail \| governance` | `list` | 控制背景色与布局风格 |

**Slots**：

| Slot | 说明 |
|------|------|
| actions | 标题右侧操作按钮区 |
| default | 页面主体内容 |

**结构示意**：

```vue
<template>
  <main :class="containerClass">
    <div class="mx-auto flex min-h-full w-full flex-col gap-5 p-4 md:p-6">
      <header class="flex flex-col gap-3 md:flex-row md:items-end md:justify-between">
        <div class="min-w-0 space-y-1">
          <p v-if="eyebrow" class="text-muted-foreground text-xs font-medium">{{ eyebrow }}</p>
          <h1 class="truncate text-2xl font-semibold tracking-normal">{{ title }}</h1>
          <p v-if="description" class="text-muted-foreground max-w-3xl text-sm">{{ description }}</p>
        </div>
        <div v-if="$slots.actions" class="flex shrink-0 flex-wrap items-center gap-2">
          <slot name="actions" />
        </div>
      </header>
      <slot />
    </div>
  </main>
</template>
```

**variant 对应背景**：

| variant | 背景 | 适用场景 |
|---------|------|----------|
| list | `bg-background` | 列表页（知识库、用户列表等） |
| workbench | `bg-muted/20` | 工作台（问答、编辑器等） |
| detail | `bg-background` | 详情页（知识库配置、用户详情） |
| governance | `bg-background` | 管理页（标签、权限等） |

### 3.5 影响范围

纯加法，不改动任何现有文件。后续页面升级时逐步引入 AppPage。

### 3.6 依赖与风险

- 依赖：无
- 风险：极低

### 7.4 组件替换对照

| Element Plus | shadcn-vue | 说明 |
|--------------|-----------|------|
| `el-card` | `Card` + `AppPage` | 页面骨架由 AppPage 承担，Card 只用于局部区块 |
| `el-table` + `el-table-column` | `Table` + `TableHeader/TableBody/TableRow/TableCell/TableHead` | 无内置排序/筛选，需自写或配合组件 |
| `el-pagination` | 自写 Pagination 或 shadcn Pagination | shadcn Pagination 是导航型，不是数据型；需封装 DataTablePagination |
| `el-form` + `el-form-item` | `vee-validate` `useForm` + `FormField/FormItem/FormLabel/FormControl/FormMessage` | 配合 zod schema |
| `el-input` | `Input` | 直替换 |
| `el-select` | `Select/SelectTrigger/SelectValue/SelectContent/SelectItem` | 直替换 |
| `el-tag` | `Badge` | `variant="outline"` 对应默认标签 |
| `el-button` | `Button` | `type="primary"` → 默认；`link` → `variant="ghost"` |
| `el-tree` | shadcn Tree 或自写 Tree | 项目已有 `components/ui/shadcn-tree/`（需确认是否可用） |
| `v-loading` | `Skeleton` 或自写加载状态 | 替换指令式加载为组件式 |

### 7.5 分页方案

Element Plus 的 `el-pagination` 是数据型分页（total/pageSize/currentPage）。shadcn 的 Pagination 是纯导航 UI。

**方案**：借鉴 kbhub `AlonDataTablePagination.vue`，创建 `framework/components/ui/AppDataTablePagination.vue`，封装 total/pageSize/currentPage 逻辑 + shadcn Pagination UI。

### 7.6 权限树/部门树方案

**方案 A**：使用 shadcn-vue 的 Tree 组件（如果项目已有 `components/ui/shadcn-tree/`）
**方案 B**：自写 TreeSelect 组件，使用 Collapsible + 递归渲染

建议优先检查 shadcn-tree 是否可用，可用则直替换。
