# 前端 PC 端框架设计需求文档

本文档描述前端 PC 端 UI 框架的功能需求，包括设计规范、框架模块和业务演示模块。

---

## 1. 前端 UI 设计规范

### 1.1 设计令牌系统

**需求描述**：建立统一的设计令牌系统，确保视觉一致性。

**功能要点**：
- 基于 Tailwind CSS v4 `@theme` 指令定义设计令牌
- 支持语义化 CSS 类名（如 `bg-primary`、`text-muted`）
- 预留 `data-theme` 属性支持主题切换扩展

**令牌分类**：

| 类型 | 说明 |
|------|------|
| 颜色令牌 | 页面背景、主色、辅色、语义色、文字色、边框色 |
| 字体令牌 | UI 字体、数据/代码字体 |
| 圆角令牌 | UI 圆角、小圆角、大圆角 |
| 阴影令牌 | 小阴影、中阴影、大阴影 |
| 过渡令牌 | 快速、正常、慢速动画 |

**颜色规范**：

| 语义 | 色值 | 用途 |
|------|------|------|
| 主色 | `#1677FF` | 主按钮、链接、菜单选中 |
| 辅色 | `#FF5722` | 次要强调、徽章 |
| 页面背景 | `#F5F7FA` | 内容区、列表底 |
| 抬升面 | `#FFFFFF` | 卡片、侧栏、顶栏 |

### 1.2 响应式设计

**需求描述**：支持不同屏幕尺寸的自适应布局。

**断点规范**：

| 断点 | 宽度 | 行为 |
|------|------|------|
| 桌面 | ≥768px | 完整布局，侧边栏展开 |
| 移动 | <768px | 侧边栏自动折叠 |

## 2. 前端 UI 框架（framework）

### 2.1 布局组件

**需求描述**：提供后台管理系统的整体布局框架。

**组件构成**：

| 组件 | 说明 |
|------|------|
| AdminLayout | 壳布局，整合所有布局子组件 |
| AppSidebar | 可折叠侧边栏（展开 240px / 折叠 64px） |
| AppNavbar | 顶部导航（60px 高度） |
| AppTagsView | 页面标签页，支持多页面切换 |
| AppMain | 内容区，渲染页面内容 |

**侧边栏需求**：

| 功能 | 说明 |
|------|------|
| 展开/折叠 | 点击按钮切换状态 |
| 菜单渲染 | 支持自定义菜单配置 |
| 选中态样式 | 主色文字 + 主色浅底背景 + 左侧指示条 |
| 响应式 | 移动端自动折叠 |

**顶部导航需求**：

| 功能 | 说明 |
|------|------|
| 折叠按钮 | 控制侧边栏展开/折叠 |
| 面包屑 | 显示当前页面路径 |
| 用户菜单 | 个人中心、退出登录 |

**标签页需求**：

| 功能 | 说明 |
|------|------|
| 标签显示 | 访问页面时自动添加标签 |
| 标签关闭 | 支持关闭可关闭的标签 |
| 状态持久化 | 刷新后保留已访问的标签 |

### 2.2 UI 组件库

**需求描述**：提供统一的交互组件，覆盖 80% CRUD 页面需求。

**组件列表**：

| 组件 | 说明 | 优先级 |
|------|------|--------|
| AppButton | 按钮（primary/secondary/outline/ghost/danger 变体） | P0 |
| AppInput | 输入框（默认/聚焦/错误态） | P0 |
| AppCard | 卡片容器 | P0 |
| AppModal | 对话框 | P0 |
| AppLoading | 加载状态 | P0 |
| AppSelect | 下拉选择（单选/多选） | P1 |
| AppTable | 数据表格（排序/分页） | P1 |
| AppForm | 表单（校验） | P1 |

**按钮需求**：

| 变体 | 说明 |
|------|------|
| primary | 主按钮，蓝色背景 |
| secondary | 强调按钮，橙色背景 |
| outline | 描边按钮，白底蓝边 |
| ghost | 幽灵按钮，透明底 |
| danger | 危险按钮，红色背景 |

**输入框需求**：

| 状态 | 说明 |
|------|------|
| 默认态 | 白底灰边框 |
| 聚焦态 | 蓝色边框 + 外圈阴影 |
| 错误态 | 红色边框 + 错误提示文案 |

### 2.3 权限控制

**需求描述**：实现前端权限控制，支持 RBAC 权限模型。

**功能模块**：

| 模块 | 说明 |
|------|------|
| 动态路由 | 根据用户权限动态注册路由 |
| 权限指令 | `v-permission` 控制元素显示 |
| 接口拦截 | 处理 401/403 响应 |
| 路由守卫 | 登录校验、权限校验 |

**动态路由需求**：

| 功能 | 说明 |
|------|------|
| 获取菜单 | 登录后从后端获取用户菜单权限 |
| 路由注册 | 通过 `router.addRoute()` 动态添加 |
| 权限过滤 | 无权限路由跳转 403 页面 |

**权限指令需求**：

| 功能 | 说明 |
|------|------|
| 单权限校验 | `v-permission="'user:add'"` |
| 多权限校验 | `v-permission="['user:add', 'user:edit']"`（满足任一即可） |
| 无权限处理 | 从 DOM 中移除元素 |

**接口拦截需求**：

| 状态码 | 处理 |
|--------|------|
| 401 | 清除 token，跳转登录页 |
| 403 | 跳转 403 无权限页面 |

### 2.4 状态管理

**需求描述**：提供 Pinia Store 管理应用状态。

**Store 列表**：

| Store | 说明 |
|-------|------|
| app | 应用状态（侧边栏、设备类型） |
| user | 用户状态（登录信息、权限码） |
| permission | 权限状态（路由、菜单） |
| tagsview | 标签页状态（已访问标签、缓存组件） |

### 2.5 公共页面

**需求描述**：提供公共页面组件。

| 页面 | 说明 |
|------|------|
| LoginPage | 登录页 |
| ForbiddenPage | 403 无权限页 |
| NotFoundPage | 404 页面不存在 |

## 3. Demo 业务模块

### 3.1 模块结构

**需求描述**：提供业务演示模块，展示框架能力。

**页面列表**：

| 页面 | 路由 | 说明 |
|------|------|------|
| HomePage | `/` | 首页，展示平台介绍 |
| HealthPage | `/health` | 健康检查页面 |
| DatasetsPage | `/datasets` | 知识库列表页面 |

### 3.2 首页需求

**需求描述**：展示平台介绍和功能特性。

**功能要点**：
- 展示平台简介
- 展示核心功能特性（设计令牌、布局、组件、权限）

### 3.3 健康检查需求

**需求描述**：检查后端服务健康状态。

**功能要点**：
- 调用 `/health` API 获取状态
- 显示服务状态和时间戳
- 支持刷新和重试

**API 端点**：

| 端点 | 方法 | 说明 |
|------|------|------|
| `/health` | GET | 获取健康状态 |

### 3.4 知识库管理需求

**需求描述**：展示知识库列表，支持 CRUD 操作。

**功能要点**：
- 显示知识库列表
- 新建知识库
- 查看/删除知识库

**API 端点**：

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/datasets` | GET | 获取知识库列表 |
| `/api/v1/datasets` | POST | 创建知识库 |
| `/api/v1/datasets/{id}` | GET | 获取知识库详情 |
| `/api/v1/datasets/{id}` | PUT | 更新知识库 |
| `/api/v1/datasets/{id}` | DELETE | 删除知识库 |

### 3.5 路由配置需求

**需求描述**：Demo 模块路由使用统一的配置格式。

**元信息要求**：

| 字段 | 说明 |
|------|------|
| title | 页面标题，用于面包屑和标签页 |
| icon | 菜单图标 |
| requiresAuth | 是否需要登录 |

**懒加载要求**：
- 使用动态导入（`() => import(...)`）实现路由懒加载

## 4. 技术约束

### 4.1 技术栈

| 技术 | 版本 |
|------|------|
| Vue | 3.5+ |
| TypeScript | 5.8+ |
| Vite | 6.x |
| Vue Router | 4.x |
| Pinia | 3.x |
| Tailwind CSS | v4 |

### 4.2 不引入的依赖

| 依赖 | 原因 |
|------|------|
| Element Plus | 保持视觉一致性，避免双套视觉语言 |
| ECharts | 暂不需要数据可视化 |

### 4.3 预留扩展

| 扩展点 | 说明 |
|--------|------|
| 暗色主题 | 预留 `data-theme="dark"` 接口 |
| 国际化 | 预留 i18n 接入点 |
