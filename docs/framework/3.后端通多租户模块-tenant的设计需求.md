# 多租户模块（tenant）需求文档

本文档描述多租户模块的功能需求，包括租户上下文管理和多资源数据隔离。

---

## 1. 基础设施

| 组件 | 用途 |
|------|------|
| PostgreSQL + pgvector | 主数据库 + 向量检索 |
| Redis | 缓存 / 队列 / 发布订阅 |
| MinIO | 对象存储 |

---

```text
src/framework/tenant/
```

## 3. 功能需求

### 3.1 租户上下文管理

**需求描述**：请求级别的租户信息存储与获取。

**功能要点**：
- 从请求中解析租户标识（Header: `X-Tenant-Id` 或 Token）
- 线程安全的租户上下文存储
- 请求结束后自动清理上下文

### 3.2 数据隔离

**需求描述**：不同租户的数据相互隔离，业务代码无感知。

| 资源 | 隔离方式 | Key/Path 格式 |
|------|----------|---------------|
| 数据库 | 字段级隔离（tenant_id） | - |
| Redis 缓存 | Key 前缀隔离 | `{tenant_id}:{key}` |
| Redis 队列 | Stream Key 前缀隔离 | `{tenant_id}:queue:{name}` |
| Redis 发布订阅 | Channel 前缀隔离 | `{tenant_id}:channel:{name}` |
| Redis 锁 | Lock Key 前缀隔离 | `{tenant_id}:lock:{key}` |
| MinIO 存储 | 目录隔离 | `{tenant_id}/{path}` |

### 3.3 租户缓存

**需求描述**：租户信息缓存，减少数据库访问。

**功能要点**：
- 两级缓存：L1 本地内存 + L2 Redis
- 缓存自动失效与更新
- 跨实例缓存同步

### 3.4 异步任务支持

**需求描述**：异步任务中保持租户上下文。

**功能要点**：
- 任务入队时自动记录 tenant_id
- 任务执行时自动恢复租户上下文

### 3.5 管理后台

**需求描述**：独立的管理后台，支持跨租户管理。

**功能要点**：
- 独立认证体系（超级管理员）
- 不经过租户中间件
- 支持跨租户查询与操作

### 3.6 默认租户管理员

**需求描述**：系统启动时自动创建默认租户管理员。

**功能要点**：
- 系统启动时自动初始化（不存在则创建）
- 拥有所有租户的管理权限
- 管理后台 API 仅限租户管理员访问

## 4. Web 层需求

### 4.1 租户中间件

**处理流程**：

1. 解析租户标识（优先级：Header > Token > 用户默认租户）
2. 验证租户状态（存在、激活、未过期）
3. 加载租户信息并注入上下文
4. 请求结束后清理上下文

### 4.2 用户-租户关联

用户可属于多个租户，支持租户切换。

### 4.3 API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/tenants` | GET | 当前用户可用租户列表 |
| `/api/v1/tenants/current` | GET | 当前租户信息 |
| `/api/v1/tenants/{id}/switch` | POST | 切换租户 |

### 4.4 管理后台 API

**权限要求**：仅限租户管理员访问。

| 端点 | 方法 | 说明 |
|------|------|------|
| `/admin/api/tenants` | GET/POST | 租户列表/创建 |
| `/admin/api/tenants/{id}` | GET/PUT/DELETE | 租户详情/更新/删除 |
| `/admin/api/tenants/{id}/activate` | POST | 激活租户 |
| `/admin/api/tenants/{id}/deactivate` | POST | 停用租户 |

### 4.5 异常处理

| 场景 | HTTP 状态码 |
|------|-------------|
| 租户不存在 | 404 |
| 租户已停用 | 403 |
| 租户已过期 | 403 |
| 无权访问租户 | 403 |
