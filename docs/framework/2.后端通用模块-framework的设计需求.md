# 2.  通用模块（`framework`）的架构设计需求

## 通用模块（`framework`）的项目结构

```text
server/技术栈/              # 不同技术栈 后端项目根目录，例如：java、netcore、python等
├── src/framework/            # 通用模块
│   ├── config/                # 统一配置模块（基于 YAML 文件的读取）
│   ├── cache/                 # 统一缓存模块，redis实现
│   ├── core/                  # 核心模块，统一接口定义（存储、队列、发布订阅、锁）
│   ├── common/                # 通用模块（异常、上下文、日志等）
│   ├── storage/               # 存储的实现，例如：本地、MinIO、阿里云等等
│   ├── queue/                 # 队列的实现，例如：mysql/pgsql行级锁、redis等等
│   ├── lock/                  # 锁的实现，例如：mysql/pgsql行级锁、redis等等
│   ├── pubsub/                # 发布订阅的实现，例如：redis、rabbitMQ等等
│   ├── database/              # 通用数据库访问的实现，例如：树结构/Tenant实现、event、interceptors、types等
│   ├── tenant/                # 多租户管理（包括每个租户使用的资源配置，例如：数据库、存储、缓存、队列、发布订阅、模型等）
│   ├── utils/                 # 工具函数，包括：字符串、时间日期、列表、枚举
│   ├── CLAUDE.md                # framwork 开发指南
│   └── README.md                # framwork 说明文档
├── tests/framwork/              # framwork 测试文件
```
