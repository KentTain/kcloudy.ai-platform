# 多租户模块（tenant）设计方案

本文档描述多租户模块的详细设计方案，包括：租户信息的上下文统一注入和获取、不同资源的数据隔离实现。

---

## 1. 基础设施及用途

项目的基础设施：数据库（PostgreSQL）、向量库（pgvector）、Redis（cache、queue、pub/sub）、MinIO，采用 MVC 架构。

## 3. 设计决策

### 3.1 数据库隔离策略

**采用字段级隔离（Shared Database, Shared Schema）**

```
┌─────────────────────────────────────────────────────────────┐
│                    数据库隔离策略                            │
├─────────────────────────────────────────────────────────────┤
│  策略：所有租户共享同一数据库，通过 tenant_id 字段隔离数据   │
│                                                             │
│  优点：                                                      │
│  ✅ 实现简单，运维成本低                                     │
│  ✅ 资源共享方便，连接池管理简单                             │
│  ✅ 跨租户查询（管理员场景）容易实现                         │
│                                                             │
│  实现：                                                      │
│  • TenantMixin：模型混入，自动添加 tenant_id 字段            │
│  • TenantEventListener：插入前自动填充 tenant_id             │
│  • TenantQueryInterceptor：查询时自动过滤 tenant_id          │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 Redis 隔离策略

**在工具类层面自动注入租户前缀**

```python
# 调用方式（业务代码无感知）
await RedisUtil.set("user:123", data)

# 实际存储（自动添加租户前缀）
# key: "tenant_001:user:123"
```

实现要点：

- `RedisUtil` 内部从 `TenantContext` 获取当前租户 ID
- 所有 key 操作自动拼接前缀 `{tenant_id}:`
- 提供跳过前缀的方法（管理员场景）：`RedisUtil.set(..., skip_tenant=True)`

### 3.4 租户缓存策略

**采用两级缓存（L1 本地内存 + L2 Redis）**

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         两级缓存架构                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   ┌─────────────────┐     Miss     ┌─────────────────┐                 │
│   │   L1 Cache      │ ──────────▶ │   L2 Cache      │                 │
│   │   (本地内存)    │             │   (Redis)       │                 │
│   ├─────────────────┤             ├─────────────────┤                 │
│   │ • 进程级缓存    │             │ • 跨实例共享    │                 │
│   │ • 热点数据      │             │ • TTL: 5 分钟   │                 │
│   │ • 无网络开销    │             │ • 集群同步      │                 │
│   │ • 最大 1000 条  │             │                 │                 │
│   └─────────────────┘             └─────────────────┘                 │
│          │                                │ Miss                      │
│          │                                ▼                           │
│          │                       ┌─────────────────┐                  │
│          │                       │   Database      │                  │
│          │                       │   (PostgreSQL)  │                  │
│          │                       └─────────────────┘                  │
│          │                                │                           │
│          └────────────────────────────────┘                           │
│                       写入时双向更新                                   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 缓存流程

```
读取流程:
─────────
1. 检查 L1 本地缓存 → 命中则返回
2. 检查 L2 Redis 缓存 → 命中则回填 L1 并返回
3. 查询数据库 → 写入 L1、L2 并返回

写入流程:
─────────
1. 更新数据库
2. 更新 L1 本地缓存
3. 更新 L2 Redis 缓存
4. 发布缓存失效消息（其他实例收到后清除 L1）
```

#### Redis Key 设计

```
# 租户信息缓存
tenant:info:{tenant_id}           → Tenant JSON
tenant:config:{tenant_id}         → TenantConfig JSON

# 缓存失效通知频道
tenant:cache:invalidate           → {"tenant_id": "xxx", "action": "update|delete"}
```

#### 本地缓存实现要点

```python
class TenantCache:
    """租户两级缓存"""

    _local_cache: dict[str, Tenant] = {}  # 进程级缓存
    _max_size: int = 1000

    @classmethod
    async def get(cls, tenant_id: str) -> Tenant | None:
        # L1: 本地缓存
        if tenant_id in cls._local_cache:
            return cls._local_cache[tenant_id]

        # L2: Redis 缓存
        data = await RedisUtil.get(f"tenant:info:{tenant_id}")
        if data:
            tenant = Tenant.from_json(data)
            cls._set_local(tenant_id, tenant)
            return tenant

        # DB: 数据库
        tenant = await TenantRepository.find_by_id(tenant_id)
        if tenant:
            await cls._set_cache(tenant)
        return tenant

    @classmethod
    async def invalidate(cls, tenant_id: str):
        """使缓存失效"""
        # 清除 L1
        cls._local_cache.pop(tenant_id, None)
        # 清除 L2
        await RedisUtil.delete(f"tenant:info:{tenant_id}")
        # 通知其他实例
        await RedisUtil.publish("tenant:cache:invalidate", tenant_id)
```

#### 任务消息格式

```python
@dataclass
class TaskMessage:
    """任务消息（自动携带租户信息）"""

    task_id: str
    task_type: str
    payload: dict[str, Any]
    tenant_id: str | None = None      # 自动填充
    created_at: datetime = field(default_factory=datetime.now)
```

#### 任务入队封装

```python
class TaskQueue:
    """任务队列（自动注入租户上下文）"""

    @classmethod
    async def enqueue(cls, task_type: str, payload: dict) -> str:
        message = TaskMessage(
            task_id=str(uuid.uuid4()),
            task_type=task_type,
            payload=payload,
            tenant_id=get_tenant_id(),  # 自动获取当前租户
        )
        return await RedisUtil.xadd(
            f"queue:{task_type}",
            message.to_dict()
        )
```

#### 任务执行器封装

```python
class TaskExecutor:
    """任务执行器（自动恢复租户上下文）"""

    @classmethod
    async def execute(cls, message: TaskMessage):
        try:
            # 恢复租户上下文
            if message.tenant_id:
                tenant = await TenantCache.get(message.tenant_id)
                if tenant:
                    TenantContext.set_current_tenant(tenant)

            # 执行任务逻辑
            handler = get_handler(message.task_type)
            await handler(message.payload)

        finally:
            # 清理上下文
            TenantContext.clear()
```

```python
class AdminAuthMiddleware:
    """管理后台认证中间件"""

    async def __call__(self, request, call_next):
        # 验证超级管理员权限
        token = self._extract_token(request)
        admin = await AdminService.verify_token(token)

        if not admin or not admin.is_superuser:
            raise AdminAccessDeniedError()

        request.state.admin = admin
        return await call_next(request)
```

### 3.7 默认租户管理员

**需求描述**：系统启动时自动创建默认租户管理员。

**功能要点**：

- 系统启动时自动初始化（不存在则创建）
- 拥有所有租户的管理权限

```python
class TenantAdminInitializer:
    """租户管理员初始化器"""

    @staticmethod
    async def init():
        """系统启动时初始化默认租户管理员"""
        admin = await TenantAdminRepository.find_default()
        if admin:
            return  # 已存在，跳过

        # 创建默认租户管理员
        admin = TenantAdmin(
            id="default",
            username="tenant_admin",
            password=hash_password("default_password"),
            is_default=True,
        )
        await TenantAdminRepository.create(admin)
```

### 5.2 租户信息传递方式

#### 请求头传递

前端通过 HTTP Header 传递租户信息：

```
HTTP Header:
  Authorization: Bearer <token>

```

#### Token 中携带租户信息（可选）

JWT Token Payload 包含租户信息：

```json
{
```

### 5.3 租户中间件设计

#### 中间件职责

1. **解析租户标识**：从 Header 或 Token 中提取 `tenant_id`
2. **验证租户状态**：检查租户是否存在、是否激活
3. **加载租户配置**：从数据库/缓存获取租户完整信息
4. **注入租户上下文**：将租户信息存入 `TenantContext`
5. **清理上下文**：请求结束后清理 `TenantContext`

#### 伪代码示例

```python
class TenantMiddleware:
    async def __call__(self, request, call_next):
        try:
            # 1. 解析租户标识
            tenant_id = self._resolve_tenant_id(request)

            # 2. 加载租户信息（带缓存）
            tenant = await TenantService.get_tenant(tenant_id)

            # 3. 验证租户状态
            if not tenant or tenant.status != "active":
                raise TenantNotFoundOrInactiveError()

            # 4. 注入上下文
            TenantContext.set_current_tenant(tenant)

            # 5. 执行业务逻辑
            return await call_next(request)
        finally:
            # 6. 清理上下文
            TenantContext.clear()
```

### 5.4 租户解析策略

优先级从高到低：

1. **请求头 `X-Tenant-Id`**：用户手动切换租户
2. **Token 中的 `tenant_id`**：用户默认租户
3. **用户主租户**：用户关联的默认租户

```python
def _resolve_tenant_id(request) -> str | None:
    # 优先级1: 请求头
    tenant_id = request.headers.get("X-Tenant-Id")
    if tenant_id:
        return tenant_id

    # 优先级2: Token 中携带
    user = request.state.user
    if user and hasattr(user, "tenant_id"):
        return user.tenant_id

    # 优先级3: 用户主租户（需要查询）
    return None
```

### 5.5 用户-租户关联模型

用户可以属于多个租户，需支持租户切换：

```
┌─────────────────┐       ┌─────────────────────┐       ┌─────────────────┐
│     User        │       │   UserTenant        │       │     Tenant      │
├─────────────────┤       ├─────────────────────┤       ├─────────────────┤
│ id              │───┐   │ user_id             │   ┌───│ id              │
│ username        │   │   │ tenant_id   ◀───────┘────   │ name            │
│ email           │   └──▶│ is_default          │       │ code            │
│ ...             │       │ role                │       │ status          │
└─────────────────┘       │ created_at          │       │ ...             │
                          └─────────────────────┘       └─────────────────┘
```
