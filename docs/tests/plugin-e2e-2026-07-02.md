> 本节记录执行失败的测试用例及其分析。

#### 示例：失败用例分析模板

**用例 ID**：INSTALL-001
**用例名称**：安装 tongyi 插件并验证虚拟环境
**失败时间**：2026-07-02 14:30:00
**错误信息**：

```
AssertionError: 虚拟环境不存在
Expected: /tmp/plugins/tenants/e2e-test-xxx/runtime/langgenius/tongyi/.venv
Actual: 目录不存在
```

**错误分析**：
- 插件安装过程中虚拟环境创建失败
- 可能原因：磁盘空间不足 / uv 安装失败

**修复建议**：
1. 检查磁盘空间是否充足
2. 验证 uv 工具是否正确安装
3. 检查插件包是否完整

## 报告填写说明

### 状态标识

- ✅ **通过**：测试执行成功
- ❌ **失败**：测试执行失败
- ⏭️ **跳过**：测试被跳过（服务不可用、环境变量未配置等）
- ⏳ **待执行**：测试尚未执行

### 填写步骤

1. **执行测试**：运行 `uv run pytest -m e2e tests/ai/e2e/ -v`
2. **收集结果**：记录通过、失败、跳过数量
3. **分析失败**：填写失败用例的错误信息和原因分析
4. **统计性能**：记录执行时间和性能瓶颈
5. **提出建议**：根据测试结果提出改进建议

### 报告生成命令

```bash
# 生成 HTML 报告
uv run pytest -m e2e tests/ai/e2e/ --html=reports/e2e-report.html --self-contained-html

# 生成 JSON 报告
uv run pytest -m e2e tests/ai/e2e/ --json-report --json-report-file=reports/e2e-report.json

# 同时生成多种格式
uv run pytest -m e2e tests/ai/e2e/ \
  --html=reports/e2e-report.html \
  --junitxml=reports/e2e-junit.xml \
  --json-report --json-report-file=reports/e2e-report.json
```
