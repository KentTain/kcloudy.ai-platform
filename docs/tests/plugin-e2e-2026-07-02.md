## 三、失败用例分析

### 3.1 失败用例列表

> 本节记录执行失败的测试用例及其分析。

**用例 ID**：INVOKE-004
**用例名称**：调用 gpustack 模型
**失败时间**：2026-07-02 11:11:34
**错误信息**：

```
PluginCommunicationError: 流式通讯失败: 收到插件错误: {'error_type': 'InvokeError',
'message': '[models] Error: API request failed with status code 401:
{"code":401,"reason":"Unauthorized","message":"Invalid authentication credentials"}'}
```

**错误分析**：
- GPUStack API Key 无效或已过期，导致认证失败（401 Unauthorized）
- 默认配置中的 API Key `gpustack_14d9f2aee5629a0f_...` 无法通过认证

**修复建议**：
1. 更新 GPUStack API Key 环境变量：`export E2E_GPUSTACK_API_KEY="valid-key"`
2. 验证 GPUStack 服务端点 `https://llm-stack.flydiysz.cn` 是否可达
3. 检查 API Key 是否有访问 `qwen3.5-9b` 模型的权限

## 报告填写说明

### 状态标识

- ✅ **通过**：测试执行成功
- ❌ **失败**：测试执行失败
- ⏭️ **跳过**：测试被跳过（服务不可用、环境变量未配置等）
- ⏳ **待执行**：测试尚未执行

### 填写步骤

1. **执行测试**：运行 `uv run pytest -m e2e tests/ai/e2e/ -v`
2. **收集结果**：记录通过、失败、跳过数量
3. **分析失败**：填写失败用例的错误信息和原因分析
4. **统计性能**：记录执行时间和性能瓶颈
5. **提出建议**：根据测试结果提出改进建议

### 报告生成命令

```bash
# 生成 HTML 报告
uv run pytest -m e2e tests/ai/e2e/ --html=reports/e2e-report.html --self-contained-html

# 生成 JSON 报告
uv run pytest -m e2e tests/ai/e2e/ --json-report --json-report-file=reports/e2e-report.json

# 同时生成多种格式
uv run pytest -m e2e tests/ai/e2e/ \
  --html=reports/e2e-report.html \
  --junitxml=reports/e2e-junit.xml \
  --json-report --json-report-file=reports/e2e-report.json
```
