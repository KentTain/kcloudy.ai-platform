## 新增需求

### 需求:凭证加密存储

系统必须使用 AES-256-GCM 算法加密存储插件凭证，确保敏感信息安全。

#### 场景:创建凭证时加密
- **当** 用户创建一个新的插件凭证
- **那么** 系统使用 AES-256-GCM 加密凭证内容
- **且** 加密密钥来自环境变量 TENANT_ENCRYPTION_MASTER_KEY
- **且** 加密后的凭证存储在 plugin_credentials 表中

#### 场景:读取凭证时解密
- **当** 系统需要使用凭证调用插件
- **那么** 系统使用相同密钥解密凭证内容
- **且** 解密后的明文仅在内存中使用，不落盘

### 需求:凭证脱敏显示

系统必须在 API 响应中脱敏显示凭证内容，避免泄露敏感信息。

#### 场景:查询凭证详情时脱敏
- **当** 用户查询凭证详情
- **那么** 系统解密凭证内容
- **且** 系统对敏感字段进行脱敏处理（如显示 `sk-****abcd`）
- **且** 返回脱敏后的凭证内容

#### 场景:更新凭证时保留未修改字段
- **当** 用户更新部分凭证字段（其他字段提交脱敏值）
- **那么** 系统识别未修改的字段（脱敏值与原值匹配）
- **且** 系统保留这些字段的原始明文值

### 需求:凭证格式校验

系统必须在创建和更新凭证时校验凭证格式，确保符合插件定义的凭证架构。

#### 场景:校验必填字段
- **当** 用户提交凭证缺少必填字段
- **那么** 系统拒绝创建并返回字段缺失错误

#### 场景:校验字段类型
- **当** 用户提交的字段类型与架构定义不匹配
- **那么** 系统拒绝创建并返回类型错误信息

### 需求:凭证有效性校验

系统必须支持通过插件运行时校验凭证有效性，确保凭证可用于实际调用。

#### 场景:创建凭证时验证有效性
- **当** 用户创建凭证且格式校验通过
- **那么** 系统调用插件的凭证验证方法
- **且** 如果验证失败，系统拒绝创建并返回错误

#### 场景:更新凭证时验证有效性
- **当** 用户更新凭证内容
- **那么** 系统调用插件的凭证验证方法
- **且** 如果验证失败，系统拒绝更新并返回错误

### 需求:凭证多租户隔离

系统必须确保不同租户的凭证数据完全隔离。

#### 场景:查询凭证列表时租户隔离
- **当** 用户查询凭证列表
- **那么** 系统仅返回当前租户的凭证记录
- **且** 系统不允许跨租户访问凭证

#### 场景:删除凭证时租户校验
- **当** 用户删除凭证
- **那么** 系统验证凭证属于当前租户
- **且** 如果不属于当前租户，系统返回"凭证不存在"错误

### 需求:凭证命名唯一性

系统必须确保同一插件下凭证名称唯一。

#### 场景:创建重名凭证
- **当** 用户创建凭证时使用已存在的名称
- **那么** 系统拒绝创建并返回"凭证名称已存在"错误

#### 场景:更新凭证名称为已存在名称
- **当** 用户更新凭证名称为同一插件下其他凭证的名称
- **那么** 系统拒绝更新并返回"凭证名称已存在"错误
