## 需求
### 需求:人员选择组件提供 Popover 快速搜索功能

人员选择组件必须提供 Popover 快速搜索入口，用户无需打开弹窗即可输入关键词搜索人员。

#### 场景:单选模式搜索并选择人员

- **当** 用户点击人员选择输入框
- **那么** 系统显示 Popover 下拉框，包含搜索输入框
- **当** 用户输入关键词搜索
- **那么** 系统显示匹配的人员列表（分页，每页 20 条）
- **当** 用户点击某个人员
- **那么** 系统选中该人员并关闭 Popover

#### 场景:多选模式搜索并选择多个人员

- **当** 用户点击人员选择输入框
- **那么** 系统显示 Popover 下拉框，包含搜索输入框
- **当** 用户输入关键词搜索并点击多个人员
- **那么** 系统将选中的人员以 Badge 标签形式展示在输入框中
- **当** 选中人员超过容器宽度
- **那么** 系统显示 "N+" 标签折叠溢出人员

### 需求:人员选择组件提供弹窗选择视图

人员选择组件必须提供弹窗选择视图，支持左右布局：左侧组织人员树 + 右侧已选人员列表。

#### 场景:打开人员选择弹窗

- **当** 用户点击人员选择输入框右侧的"选择人员"按钮
- **那么** 系统打开人员选择弹窗
- **当** 弹窗打开时
- **那么** 左侧显示组织人员树，右侧显示已选人员列表

#### 场景:通过组织树选择人员

- **当** 用户展开组织节点
- **那么** 系统显示该组织下的直属人员列表
- **当** 用户点击人员
- **那么** 系统将人员添加到右侧已选列表
- **当** 用户再次点击已选人员
- **那么** 系统从已选列表移除该人员

#### 场景:三态复选框选择组织下所有人员

- **当** 用户勾选组织节点的复选框
- **那么** 系统选中该组织下所有可选人员
- **当** 组织下部分人员已选中
- **那么** 复选框显示半选状态（indeterminate）
- **当** 组织下所有人员已选中
- **那么** 复选框显示全选状态（checked）

### 需求:人员选择组件支持头像展示

人员选择组件必须支持展示用户头像。

#### 场景:显示用户头像

- **当** 系统渲染人员项
- **那么** 系统调用 `/iam/console/v1/users/{user_id}/avatar` 获取头像
- **当** 头像加载成功
- **那么** 显示用户头像图片
- **当** 头像加载失败
- **那么** 显示默认用户图标

### 需求:人员选择组件支持人员详情卡片

人员选择组件必须支持 Popover 形式的人员详情卡片展示。

#### 场景:查看人员详情卡片

- **当** 用户点击已选人员的头像或名称
- **那么** 系统显示 Popover 卡片
- **当** 卡片显示时
- **那么** 卡片包含：头像、姓名、用户名、所属组织、邮箱

### 需求:人员选择组件支持单选和多选模式

人员选择组件必须支持单选和多选模式。

#### 场景:单选模式

- **当** 组件设置 `multiple="false"`
- **那么** 系统只允许选择一个人员
- **当** 用户选择新人员
- **那么** 系统替换之前的人员

#### 场景:多选模式

- **当** 组件设置 `multiple="true"`
- **那么** 系统允许选择多个人员
- **当** 用户选择人员
- **那么** 系统将人员添加到已选列表

### 需求:人员选择组件支持禁用指定人员

人员选择组件必须支持禁用指定人员。

#### 场景:禁用人员不可选择

- **当** 组件设置 `disabled-ids=["user1", "user2"]`
- **那么** 禁用的人员显示为灰色且不可点击
- **当** 用户点击禁用人员
- **那么** 系统不执行任何操作

### 需求:人员选择组件支持清空选择

人员选择组件必须支持清空已选人员。

#### 场景:清空已选人员

- **当** 用户点击清空按钮
- **那么** 系统清除所有已选人员
- **当** 组件触发 `clear` 事件
