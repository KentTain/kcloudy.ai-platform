## 需求
### 需求:管理员认证中间件唯一来源
系统必须确保管理员认证中间件只有一个实现来源，位于 `tenant.middlewares.admin_auth_middleware` 模块。

#### 场景:中间件注册正确
- **当** 应用启动时
- **那么** 只有 `tenant.middlewares.admin_auth_middleware.AdminAuthMiddleware` 被注册

#### 场景:导入来源统一
- **当** 控制器或测试需要使用管理员认证功能
- **那么** 必须从 `tenant.middlewares.admin_auth_middleware` 导入

### 需求:管理员认证路径隔离
系统必须确保只有 `/admin/*` 路径的请求经过管理员认证中间件。

#### 场景:管理员路径需要认证
- **当** 请求路径以 `/admin/` 开头
- **那么** 必须验证管理员 Token

#### 场景:管理员登录路径豁免
- **当** 请求路径为 `/admin/v1/auth/login`
- **那么** 跳过 Token 验证，允许匿名访问

#### 场景:非管理员路径不拦截
- **当** 请求路径不以 `/admin/` 开头
- **那么** 中间件必须放行，由其他中间件处理

### 需求:管理员全局权限
系统必须确保租户管理员（TenantAdmin）是全局管理员，不具有 `tenant_id` 属性。

#### 场景:管理员登录 Token 数据
- **当** 租户管理员成功登录
- **那么** Token 数据必须包含 `admin_id`、`username`、`is_default`、`expires_at`
- **并且** 必须不包含 `tenant_id` 字段

#### 场景:管理员访问跨租户数据
- **当** 租户管理员访问任何租户的数据
- **那么** 系统必须允许访问（租户管理员拥有全局权限）

### 需求:系统设置平台级管理
系统必须确保系统设置是平台级配置，不属于任何特定租户。

#### 场景:管理员创建系统设置
- **当** 租户管理员创建系统设置
- **那么** 设置必须不关联到任何租户
- **并且** 设置必须对所有租户可见

#### 场景:管理员查询系统设置
- **当** 租户管理员查询系统设置列表
- **那么** 必须返回所有平台级设置
- **并且** 不按租户过滤
