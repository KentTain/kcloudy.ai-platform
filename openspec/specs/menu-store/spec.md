## 新增需求

### 需求:菜单数据获取

系统必须提供菜单 Store，从后端 API 获取用户可见菜单。

菜单 Store 必须实现：
- `menus`: 响应式菜单树数组
- `fetchMenus()`: 从 `/api/v1/menus/user` 获取菜单数据
- `loading`: 加载状态
- `error`: 错误信息

#### 场景:成功获取菜单
- **当** 用户已登录且调用 fetchMenus()
- **那么** menus 更新为后端返回的菜单树

#### 场景:获取菜单失败
- **当** API 请求失败
- **那么** error 设置为错误信息，menus 保持不变

#### 场景:未登录用户
- **当** 用户未登录时调用 fetchMenus()
- **那么** 请求被拦截，重定向到登录页

### 需求:跨子域名菜单处理

系统必须正确处理跨子域名的菜单项。

菜单 Store 必须：
- 检测菜单项的 `deployment_base_url` 字段
- 计算菜单项的 `isExternal` 属性（deployment_base_url 与当前域名不同）
- 为跨域菜单标记特殊属性

#### 场景:同域菜单
- **当** 菜单项 deployment_base_url 为空或与当前域名相同
- **那么** isExternal 为 false，使用 Vue Router 导航

#### 场景:跨域菜单
- **当** 菜单项 deployment_base_url 为其他子域名
- **那么** isExternal 为 true，使用 window.location.href 跳转

### 需求:菜单缓存

系统必须支持菜单数据本地缓存，提升加载性能。

缓存策略：
- 使用 localStorage 存储菜单数据
- 设置缓存过期时间（默认 5 分钟）
- 提供强制刷新参数

#### 场景:使用缓存菜单
- **当** 缓存存在且未过期
- **那么** fetchMenus() 优先返回缓存数据，后台静默更新

#### 场景:强制刷新
- **当** 调用 fetchMenus({ force: true })
- **那么** 忽略缓存，直接请求后端 API

### 需求:菜单权限过滤

系统必须根据用户权限过滤菜单可见性。

菜单 Store 必须：
- 接收后端返回的已过滤菜单（后端根据权限过滤）
- 不在前端进行二次权限过滤

#### 场景:显示用户有权限的菜单
- **当** 用户有部分菜单权限
- **那么** 仅显示有权限的菜单项
