## ADDED Requirements

### 需求:数据库配置管理

系统必须支持数据库配置的 CRUD，配置独立于租户存在，多个租户可共享同一配置。

#### 场景:创建数据库配置
- **当** 管理员请求 `POST /api/v1/tenant/resource-configs/databases` 并提供 name、type、host、port、database、username、password
- **那么** 创建配置记录，密码 AES 加密存储

#### 场景:查询数据库配置列表
- **当** 管理员请求 `GET /api/v1/tenant/resource-configs/databases`
- **那么** 返回配置列表，密码字段脱敏返回

#### 场景:获取单个数据库配置
- **当** 管理员请求 `GET /api/v1/tenant/resource-configs/databases/{id}`
- **那么** 返回配置详情，密码字段脱敏返回

#### 场景:更新数据库配置
- **当** 管理员请求 `PUT /api/v1/tenant/resource-configs/databases/{id}`
- **那么** 更新配置信息；密码更新时重新加密

#### 场景:删除数据库配置
- **当** 管理员请求 `DELETE /api/v1/tenant/resource-configs/databases/{id}` 且配置未被租户引用
- **那么** 删除配置

#### 场景:删除已引用的数据库配置
- **当** 管理员尝试删除已被租户引用的配置
- **那么** 返回 HTTP 400，错误消息为"配置已被租户使用，禁止删除"

### 需求:数据库连通性测试

系统必须支持数据库配置的连通性测试。

#### 场景:测试连接成功
- **当** 管理员请求 `POST /api/v1/tenant/resource-configs/databases/{id}/test-connection` 且连接参数正确
- **那么** 返回连接成功，执行 `SELECT 1` 验证

#### 场景:测试连接失败
- **当** 管理员请求测试连接且连接参数错误
- **那么** 返回连接失败及具体错误信息

#### 场景:测试连接超时
- **当** 连接测试超过 5 秒
- **那么** 返回超时错误

### 需求:密码加密与脱敏

系统必须对资源配置中的密码字段进行加密存储和脱敏返回。

#### 场景:密码加密存储
- **当** 创建或更新资源配置包含密码字段
- **那么** 密码使用 AES 加密后存储

#### 场景:密码脱敏返回
- **当** 查询资源配置
- **那么** 密码字段返回 `"******"`，不暴露明文

### 需求:资源配置支持默认标记

系统必须支持为每种资源配置类型（数据库、缓存、存储、队列、发布订阅）标记一个默认配置。

#### 场景:查看默认配置标记
- **当** 管理员查看资源配置列表
- **那么** 系统显示每个配置的 `is_default` 字段，默认配置标记为 `true`

#### 场景:默认配置排在第一位
- **当** 管理员查询资源配置列表
- **那么** `is_default=true` 的配置排在列表第一位

### 需求:默认配置唯一性保证

系统必须保证每种资源配置类型只能有一个默认配置。

#### 场景:创建新的默认配置
- **当** 管理员创建资源配置并标记为默认
- **那么** 系统将其他同类型配置的 `is_default` 设置为 `false`，新配置标记为 `true`

#### 场景:更新配置为默认
- **当** 管理员将现有配置更新为默认配置
- **那么** 系统将其他同类型配置的 `is_default` 设置为 `false`，当前配置标记为 `true`

#### 场景:并发创建默认配置
- **当** 多个请求同时尝试创建默认配置
- **那么** 数据库唯一索引阻止重复，只有一个请求成功

### 需求:应用启动时自动初始化默认配置

系统必须在应用启动时自动创建默认资源配置（如果不存在）。

#### 场景:首次启动创建默认配置
- **当** 应用首次启动且不存在默认配置
- **那么** 系统从 `application.yml` 读取配置并创建默认资源配置，标记 `is_default=true`

#### 场景:默认配置已存在
- **当** 应用启动且默认配置已存在
- **那么** 系统跳过创建，保持现有配置不变

#### 场景:配置文件包含敏感信息
- **当** 创建默认配置时涉及密码、密钥等敏感字段
- **那么** 系统使用 AES-256-GCM 加密后存储

### 需求:数据库连接字符串解析

系统必须支持从数据库连接 URL 解析连接参数。

#### 场景:解析 PostgreSQL 连接字符串
- **当** 系统从 `application.yml` 读取 `sqlalchemy.url`
- **那么** 系统解析 URL 并提取数据库类型、主机、端口、数据库名、用户名、密码

#### 场景:URL 包含特殊字符
- **当** 数据库密码包含特殊字符（如 `@`, `:`, `/`）
- **那么** 系统正确解析 URL 编码的字符

### 需求:租户创建时自动关联默认配置

系统必须在创建租户时，如果未指定资源配置，自动关联默认配置。

#### 场景:未指定配置时使用默认
- **当** 管理员创建租户时未传入 `db_config_id`
- **那么** 系统查询 `DatabaseConfig` 中 `is_default=true` 的配置并关联

#### 场景:指定配置时使用指定
- **当** 管理员创建租户时传入 `db_config_id`
- **那么** 系统使用指定的配置 ID，不使用默认配置

#### 场景:默认配置不存在
- **当** 创建租户时未指定配置且默认配置不存在
- **那么** 系统创建租户但不关联配置（配置 ID 为 `null`）

### 需求:前端创建租户显示配置选择器

前端必须显示资源配置选择器，默认配置排在第一位且默认选中。

#### 场景:显示配置选择器
- **当** 管理员访问创建租户页面
- **那么** 页面显示数据库、缓存、存储、队列、发布订阅配置选择器

#### 场景:默认配置默认选中
- **当** 配置选择器加载完成
- **那么** `is_default=true` 的配置排在第一位且处于选中状态

#### 场景:选择其他配置
- **当** 管理员从选择器中选择非默认配置
- **那么** 选中的配置 ID 传递给后端

#### 场景:创建新配置
- **当** 管理员点击"+ 创建新配置"按钮
- **那么** 页面跳转到资源配置页面，创建完成后返回

### 需求:API 响应包含 is_default 字段

资源配置 API 响应必须包含 `is_default` 字段。

#### 场景:查询配置列表
- **当** 调用 `GET /tenant/admin/v1/database-configs`
- **那么** 响应中每个配置包含 `is_default` 字段

#### 场景:查询单个配置
- **当** 调用 `GET /tenant/admin/v1/database-configs/{id}`
- **那么** 响应包含 `is_default` 字段

### 需求:默认配置不应轻易删除

系统必须提示用户删除默认配置的风险。

#### 场景:尝试删除默认配置
- **当** 管理员尝试删除 `is_default=true` 的配置
- **那么** 系统显示警告提示"此配置是默认配置，删除后新租户将无法自动关联配置"

#### 场景:默认配置被租户引用
- **当** 尝试删除的默认配置被租户引用
- **那么** 系统禁止删除并提示"此配置被租户使用，无法删除"
