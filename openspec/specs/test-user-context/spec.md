# test-user-context Specification

## Purpose
测试用户上下文管理规范，支持在非生产环境下通过请求头注入测试用户信息。

## Requirements
### 需求:测试用户上下文注入

系统必须支持在非生产环境下通过请求头注入测试用户信息。

#### 场景:开发环境注入测试用户
- **当** 环境为非生产环境
- **且** 请求头包含 X-Test-User-Id
- **那么** 系统将该用户 ID 设置到当前上下文
- **且** 记录警告日志

#### 场景:生产环境拒绝测试用户
- **当** 环境为生产环境
- **且** 请求头包含 X-Test-User-Id
- **那么** 系统忽略该请求头
- **且** 不记录日志

#### 场景:未启用测试用户功能
- **当** TestUserMiddleware 未启用
- **那么** 系统忽略所有 X-Test-User-Id 请求头

### 需求:测试用户日志警告

系统必须在注入测试用户时记录警告日志，提醒开发者此功能仅用于测试。

#### 场景:注入测试用户时记录日志
- **当** 成功注入测试用户
- **那么** 系统记录警告日志，内容包含用户 ID 和用途说明
