## 新增需求

### 需求:Tenant 模块登录页面测试

系统必须包含管理员登录页面的 UI 测试，验证登录流程的正确性。

#### 场景:UI 登录成功
- **当** 访问 `/admin/login` 页面
- **当** 输入正确的用户名 `admin` 和密码 `admin123`
- **当** 点击登录按钮
- **那么** 页面跳转到 `/admin/tenants`
- **那么** localStorage 包含有效的 `admin_token`

#### 场景:UI 登录失败
- **当** 输入错误的用户名或密码
- **当** 点击登录按钮
- **那么** 页面显示错误提示信息
- **那么** 页面保持在登录页

### 需求:租户管理 CRUD 测试

系统必须包含租户管理模块的完整 CRUD 测试。

#### 场景:租户列表加载
- **当** 访问 `/admin/tenants` 页面
- **那么** 页面显示租户列表表格
- **那么** 统计卡片显示正确的数值（租户总数、未激活数、过期数）

#### 场景:创建租户
- **当** 点击"新建租户"按钮
- **当** 填写租户名称、编码等必填字段
- **当** 点击保存按钮
- **那么** 创建成功提示出现
- **那么** 新租户出现在列表中

#### 场景:编辑租户
- **当** 点击租户行的"编辑"按钮
- **当** 修改租户信息
- **当** 点击保存按钮
- **那么** 更新成功提示出现
- **那么** 列表显示更新后的信息

#### 场景:删除租户
- **当** 点击租户行的"删除"按钮
- **当** 确认删除操作
- **那么** 删除成功提示出现
- **那么** 租户从列表中消失

#### 场景:搜索租户
- **当** 在搜索框输入租户名称或编码
- **当** 点击搜索按钮
- **那么** 列表显示匹配的租户
