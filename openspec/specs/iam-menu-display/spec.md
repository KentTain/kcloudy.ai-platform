# iam-menu-display 规范

## 目的
待定 - 由归档变更 iam-menu-permission-display 创建。归档后请更新目的。
## 需求
### 需求:菜单树展示

系统 SHALL 在菜单管理页面左侧以树形结构展示所有菜单。

#### 场景:加载菜单树

- **当** 用户访问菜单管理页面
- **那么** 系统调用 `GET /v1/iam/menu` API 获取菜单树并展示

#### 场景:树形结构展示

- **当** 菜单数据加载完成
- **那么** 系统按层级缩进展示菜单树，支持展开/折叠

#### 场景:选中菜单节点

- **当** 用户点击树中的菜单节点
- **那么** 系统高亮选中该节点并在右侧展示详情

#### 场景:空状态展示

- **当** 菜单树为空
- **那么** 系统显示「暂无菜单数据」

### 需求:菜单详情展示

系统 SHALL 在页面右侧展示选中菜单的详细信息。

#### 场景:显示菜单详情

- **当** 用户选中某个菜单节点
- **那么** 系统在右侧展示：菜单名称、菜单编码、路由路径、所属模块、图标、是否可见

#### 场景:未选中状态

- **当** 用户未选中任何菜单节点
- **那么** 系统显示「请选择菜单」提示

### 需求:菜单按模块标识

系统 SHALL 在菜单树中标识每个菜单所属的模块。

#### 场景:模块标识展示

- **当** 菜单树渲染时
- **那么** 每个菜单节点关联显示其所属模块（如 iam、tenant、demo）

### 需求:菜单页面路由

系统 SHALL 提供菜单管理页面的路由配置。

#### 场景:访问菜单页面

- **当** 用户访问 `/iam/menus` 路径
- **那么** 系统渲染菜单管理页面
