## 需求
### 需求:使用 DataTable 组件展示列表数据

IAM 模块的列表页面必须使用 `@/components/common/DataTable` 组件替代手动组装的 Table 组件，以提供统一的表格交互体验。

#### 场景:用户列表页面使用 DataTable
- **当** 用户访问 `/iam/users` 页面
- **那么** 用户列表必须使用 DataTable 组件展示，包含分页、加载状态、空状态

#### 场景:角色成员表格使用 DataTable
- **当** 用户在角色管理页面查看角色成员
- **那么** 成员列表必须使用 DataTable 组件展示

#### 场景:权限管理角色列表使用 DataTable
- **当** 用户在权限管理页面查看角色列表
- **那么** 角色列表必须使用 DataTable 组件展示

#### 场景:部门管理使用 DataTable
- **当** 用户在部门管理页面查看下级组织或直属成员
- **那么** 下级组织表格和直属成员表格必须使用 DataTable 组件展示

#### 场景:登录历史使用 DataTable
- **当** 用户在个人中心查看登录历史
- **那么** 登录历史列表必须使用 DataTable 组件展示

### 需求:使用 useDataTable composable 管理表格状态

所有 DataTable 必须通过 `useDataTable` composable 初始化，实现远程数据获取和状态管理。

#### 场景:正确初始化 DataTable
- **当** 页面需要展示分页数据
- **那么** 必须使用 `useDataTable` 初始化，传入 `columns` 和 `remoteFetchFn`

#### 场景:刷新表格数据
- **当** 用户执行搜索、筛选或操作后需要刷新数据
- **那么** 必须调用 `dataTable.refresh()` 方法重新获取数据

### 需求:列定义使用 ColumnDef 类型

表格列必须使用 `ColumnDef<T>[]` 类型定义，使用 `accessorKey` 映射数据字段，使用 `h()` 函数渲染自定义单元格。

#### 场景:基础列定义
- **当** 定义简单数据列
- **那么** 必须使用 `accessorKey` 映射字段，使用 `cell` 函数自定义渲染

#### 场景:操作列定义
- **当** 定义操作按钮列
- **那么** 必须使用 `id: "actions"` 标识，使用 `h()` 函数渲染按钮组件

### 需求:保持现有功能不变

迁移后必须保持所有现有功能正常工作，包括但不限于：分页、筛选、排序、行内操作、状态展示。

#### 场景:分页功能保持不变
- **当** 用户切换页码或每页条数
- **那么** 必须正确加载对应页的数据

#### 场景:筛选功能保持不变
- **当** 用户执行搜索或筛选操作
- **那么** 必须正确过滤数据并重置到第一页

#### 场景:行内操作保持不变
- **当** 用户点击表格中的操作按钮
- **那么** 必须正确触发对应的处理函数
