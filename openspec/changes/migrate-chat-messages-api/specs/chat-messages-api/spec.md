# LLM 对话接口规范

## 新增需求

### 需求:系统支持流式对话接口

系统必须提供 `POST /api/v1/chat-messages` 接口，支持流式对话。

#### 场景:发起对话请求

- **当** 客户端发送 `POST /api/v1/chat-messages` 请求
- **那么** 系统必须返回 `StreamingResponse`，媒体类型为 `text/event-stream`

#### 场景:流式返回对话内容

- **当** 对话生成内容
- **那么** 系统必须通过 SSE 流式返回内容块

#### 场景:对话完成返回统计

- **当** 对话生成完成
- **那么** 系统必须返回 finish 事件，包含 token 统计信息

---

### 需求:系统支持会话创建

系统必须支持创建新会话。

#### 场景:创建新会话

- **当** 请求不包含 `conversation_id`
- **那么** 系统必须创建新的 Conversation 记录

#### 场景:返回会话 ID

- **当** 新会话创建成功
- **那么** 系统必须在响应中返回新生成的 `conversation_id`

#### 场景:会话关联租户

- **当** 创建新会话
- **那么** 系统必须将会话关联到当前租户

---

### 需求:系统支持会话恢复

系统必须支持恢复已有会话继续对话。

#### 场景:恢复已有会话

- **当** 请求包含有效的 `conversation_id`
- **那么** 系统必须在该会话上下文中继续对话

#### 场景:验证会话归属

- **当** 尝试恢复会话
- **那么** 系统必须验证会话属于当前租户

#### 场景:会话不存在

- **当** 请求的 `conversation_id` 不存在
- **那么** 系统必须返回 404 错误

#### 场景:跨租户访问会话

- **当** 尝试访问其他租户的会话
- **那么** 系统必须拒绝访问并返回 403 错误

---

### 需求:系统支持模型配置

系统必须支持在请求中配置模型参数。

#### 场景:指定模型

- **当** 请求包含 `model.name` 和 `model.provider`
- **那么** 系统必须使用指定的模型进行对话

#### 场景:配置模型参数

- **当** 请求包含 `model.completion_params`
- **那么** 系统必须将参数传递给 LLM 调用

#### 场景:默认模型参数

- **当** 请求未指定模型参数
- **那么** 系统必须使用默认参数（temperature=0.7 等）

---

### 需求:系统支持任务停止

系统必须提供 `POST /api/v1/chat-messages/:id/stop` 接口停止正在进行的对话。

#### 场景:停止对话

- **当** 调用停止接口
- **那么** 系统必须发送停止信号取消正在进行的任务

#### 场景:更新消息状态

- **当** 对话被停止
- **那么** 系统必须将对应消息状态更新为 `stopped`

#### 场景:停止不存在的会话

- **当** 尝试停止不存在的会话
- **那么** 系统必须返回 404 错误

---

### 需求:系统支持错误处理

系统必须正确处理对话过程中的错误。

#### 场景:模型调用失败

- **当** LLM 调用失败
- **那么** 系统必须发送 error 事件并结束流

#### 场景:错误事件格式

- **当** 发送错误事件
- **那么** 事件必须包含 `code` 和 `message` 字段

#### 场景:更新消息错误状态

- **当** 对话失败
- **那么** 系统必须将消息状态更新为 `error` 并记录错误信息

---

### 需求:系统支持联网搜索（可选）

系统可选支持联网搜索增强对话。

#### 场景:启用联网搜索

- **当** 请求包含 `search.enabled=true`
- **那么** 系统必须在对话中使用搜索工具

#### 场景:返回搜索关键词

- **当** 搜索工具被调用
- **那么** 系统必须发送 `search_keywords` 事件

---

### 需求:系统支持租户隔离

系统必须确保不同租户的数据隔离。

#### 场景:请求携带租户上下文

- **当** 请求进入
- **那么** 系统必须从认证信息中提取租户 ID

#### 场景:自动关联租户

- **当** 创建会话或消息
- **那么** 系统必须自动关联到当前租户，无需客户端指定
