# SSE 事件格式规范

## 新增需求

### 需求:系统定义 SSE 事件格式

系统必须使用标准 SSE（Server-Sent Events）格式返回流式响应。

#### 场景:SSE 响应头

- **当** 返回流式响应
- **那么** 响应头必须包含 `Content-Type: text/event-stream`

#### 场景:SSE 事件格式

- **当** 发送 SSE 事件
- **那么** 每个事件必须以 `data: ` 开头，以 `\n\n` 结尾

---

### 需求:系统支持 message 事件

系统必须支持 `message` 事件类型，用于返回对话内容块。

#### 场景:发送内容块

- **当** 模型生成内容块
- **那么** 系统必须发送 message 事件

#### 场景:message 事件格式

- **当** 发送 message 事件
- **那么** 数据格式必须为：`{"event": "message", "data": {"content": "内容"}}`

#### 场景:内容块逐个发送

- **当** 流式生成多个内容块
- **那么** 系统必须为每个内容块发送独立的 message 事件

---

### 需求:系统支持 finish 事件

系统必须支持 `finish` 事件类型，用于标记对话完成。

#### 场景:对话完成发送 finish

- **当** 对话生成完成
- **那么** 系统必须发送 finish 事件

#### 场景:finish 事件格式

- **当** 发送 finish 事件
- **那么** 数据格式必须为：`{"event": "finish", "data": {"prompt_tokens": N, "completion_tokens": M}}`

#### 场景:finish 事件包含统计

- **当** 发送 finish 事件
- **那么** 数据必须包含 `prompt_tokens` 和 `completion_tokens` 字段

---

### 需求:系统支持 search_keywords 事件

系统必须支持 `search_keywords` 事件类型，用于返回联网搜索关键词。

#### 场景:搜索触发发送事件

- **当** 联网搜索工具被调用
- **那么** 系统必须发送 search_keywords 事件

#### 场景:search_keywords 事件格式

- **当** 发送 search_keywords 事件
- **那么** 数据格式必须为：`{"event": "search_keywords", "data": {"keywords": ["关键词1", "关键词2"]}}`

---

### 需求:系统支持 error 事件

系统必须支持 `error` 事件类型，用于返回错误信息。

#### 场景:错误发送 error 事件

- **当** 对话过程发生错误
- **那么** 系统必须发送 error 事件并结束流

#### 场景:error 事件格式

- **当** 发送 error 事件
- **那么** 数据格式必须为：`{"event": "error", "data": {"code": "ERROR_CODE", "message": "错误描述"}}`

#### 场景:错误代码定义

- **当** 发送 error 事件
- **那么** `code` 字段必须使用预定义的错误代码（如 `MODEL_ERROR`、`PROVIDER_ERROR`、`TIMEOUT_ERROR`）

---

### 需求:系统保证事件顺序

系统必须保证 SSE 事件的正确顺序。

#### 场景:事件顺序

- **当** 对话正常完成
- **那么** 事件顺序必须为：多个 message 事件 → finish 事件

#### 场景:搜索场景事件顺序

- **当** 启用联网搜索
- **那么** 事件顺序可能为：search_keywords 事件 → message 事件 → finish 事件

#### 场景:错误中断事件顺序

- **当** 发生错误
- **那么** 系统必须在当前事件后发送 error 事件并结束流，不再发送后续事件

---

### 需求:系统支持连接管理

系统必须正确管理 SSE 连接的生命周期。

#### 场景:客户端断开连接

- **当** 客户端断开连接
- **那么** 系统必须停止生成并清理资源

#### 场景:超时断开连接

- **当** 连接超过配置的超时时间
- **那么** 系统必须发送 error 事件（如适用）并关闭连接

#### 场景:正常结束连接

- **当** 发送完 finish 或 error 事件
- **那么** 系统必须关闭连接
