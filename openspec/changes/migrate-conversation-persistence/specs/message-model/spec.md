# 消息模型规范

## 新增需求

### 需求:系统支持消息持久化

系统必须提供 `Message` ORM 模型，支持消息数据的持久化存储。

#### 场景:创建消息

- **当** 创建新的 `Message` 记录
- **那么** 系统必须生成 UUID 作为主键，并关联到指定会话（`conversation_id`）

#### 场景:消息关联会话

- **当** 创建消息时
- **那么** 系统必须将消息关联到有效的会话记录

#### 场景:消息包含角色信息

- **当** 创建消息时
- **那么** 系统必须记录消息角色（`role`：user/assistant/system/tool）

---

### 需求:系统支持消息状态管理

系统必须支持消息状态枚举和状态变更。

#### 场景:消息状态枚举

- **当** 查询消息状态
- **那么** 系统必须支持 `pending`、`completed`、`error`、`stopped` 四种状态

#### 场景:消息初始状态

- **当** 创建新消息
- **那么** 系统必须将初始状态设为 `pending`

#### 场景:消息完成状态

- **当** 消息生成完成
- **那么** 系统必须将状态更新为 `completed` 并记录 `answer` 内容

#### 场景:消息错误状态

- **当** 消息生成失败
- **那么** 系统必须将状态更新为 `error` 并记录错误信息

#### 场景:消息停止状态

- **当** 消息被用户停止
- **那么** 系统必须将状态更新为 `stopped`

---

### 需求:系统支持消息内容存储

系统必须支持消息内容的完整存储。

#### 场景:存储用户问题

- **当** 创建用户消息
- **那么** 系统必须将用户问题存储在 `query` 字段

#### 场景:存储助手回复

- **当** 消息完成
- **那么** 系统必须将助手回复存储在 `answer` 字段

#### 场景:存储流式内容

- **当** 流式生成完成
- **那么** 系统必须将完整内容拼接后存储在 `content` 字段

---

### 需求:系统支持 token 统计

系统必须支持消息的 token 使用统计。

#### 场景:记录 token 数量

- **当** 消息完成
- **那么** 系统必须记录 `token_count` 字段

#### 场景:token 统计包含输入输出

- **当** 记录 token 统计
- **那么** `token_count` 必须包含 prompt tokens 和 completion tokens 的总和

---

### 需求:系统支持消息查询

系统必须支持按条件查询消息列表。

#### 场景:按会话查询消息

- **当** 指定 `conversation_id` 查询
- **那么** 系统必须返回该会话的所有消息，按 `created_at` 排序

#### 场景:分页查询消息

- **当** 使用分页参数查询消息
- **那么** 系统必须返回指定页的消息列表

#### 场景:按状态过滤消息

- **当** 指定 `status=completed` 过滤
- **那么** 系统必须只返回状态为 `completed` 的消息

---

### 需求:系统支持扩展元数据

系统必须支持消息的扩展元数据存储。

#### 场景:存储元数据

- **当** 创建消息时传入 `message_metadata`
- **那么** 系统必须将元数据以 JSONB 格式存储

#### 场景:查询元数据

- **当** 查询消息
- **那么** 系统必须正确解析并返回 `message_metadata` 字段
