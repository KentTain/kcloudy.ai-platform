## 新增需求

### 需求:角色权限关联同步

系统必须在租户分配模块时同步角色权限关联。

#### 场景:模块分配时同步角色权限
- **当** 租户分配模块
- **那么** 同步该模块所有 ModuleRolePermission 到 RolePermission

#### 场景:同步映射规则
- **当** 同步 ModuleRolePermission 到 RolePermission
- **那么** 通过 ref_id 查找租户实例层的 Role 和 Permission ID

#### 场景:新增角色权限同步
- **当** 模块角色新增权限关联（已分配模块）
- **那么** 为所有已分配该模块的租户创建 RolePermission 记录

#### 场景:删除角色权限同步
- **当** 模块角色删除权限关联
- **那么** 删除所有租户对应的 RolePermission 记录

### 需求:权限缓存失效触发

系统必须在角色权限变更时触发权限缓存失效。

#### 场景:角色权限同步后缓存失效
- **当** 角色权限同步完成
- **那么** 调用 `invalidate_tenant_permission_cache(tenant_id)`

#### 场景:角色权限分配后缓存失效
- **当** 管理员为角色分配新权限
- **那么** 清除该租户所有用户的权限缓存

#### 场景:用户角色变更后缓存失效
- **当** 用户的角色被修改
- **那么** 清除该用户的权限缓存

### 需求:同步幂等性

系统必须保证同步操作的幂等性。

#### 场景:重复同步幂等
- **当** 同一 ModuleRolePermission 多次同步
- **那么** 只创建一条 RolePermission 记录（通过唯一约束保证）

#### 场景:同步失败回滚
- **当** 同步过程中发生错误
- **那么** 回滚本次同步的所有变更

### 需求:同步事件监听

系统必须监听模块定义层的变更事件并触发同步。

#### 场景:监听 ModuleRolePermission 变更
- **当** ModuleRolePermission 发生增删改
- **那么** 触发同步处理函数

#### 场景:监听 ModuleMenuPermission 变更
- **当** ModuleMenuPermission 发生增删改
- **那么** 触发同步处理函数
