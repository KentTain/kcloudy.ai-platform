## 新增需求

### 需求:租户角色自动创建

系统必须在创建租户时自动创建 owner、admin、member 三个租户级角色。

#### 场景:创建租户时自动创建角色
- **当** 创建新租户
- **那么** 自动创建三个角色：
  - `{tenant_id}:owner` - 租户所有者
  - `{tenant_id}:admin` - 租户管理员
  - `{tenant_id}:member` - 租户成员

#### 场景:角色编码规范
- **当** 创建租户角色
- **那么** 角色编码格式为 `{tenant_id}:{role_type}`，确保唯一性

#### 场景:租户删除时角色级联删除
- **当** 租户被删除
- **那么** 该租户的所有角色自动删除（通过 tenant_id 外键级联）

### 需求:租户所有者角色

系统必须在租户创建时将创建者分配为 owner 角色。

#### 场景:创建者自动分配 owner 角色
- **当** 创建新租户
- **那么** 创建者自动获得 `{tenant_id}:owner` 角色

#### 场景:owner 角色权限
- **当** 用户拥有 owner 角色
- **那么** 用户拥有该租户下所有模块的所有权限

### 需求:租户角色权限模板

系统必须为租户角色分配默认权限。

#### 场景:owner 角色权限
- **当** 创建 owner 角色
- **那么** 分配所有已分配模块的所有权限

#### 场景:admin 角色权限
- **当** 创建 admin 角色
- **那么** 分配已分配模块的管理权限（不含删除）

#### 场景:member 角色权限
- **当** 创建 member 角色
- **那么** 分配已分配模块的基础访问权限

### 需求:用户加入租户角色分配

系统必须在用户加入租户时分配默认角色。

#### 场景:用户加入租户分配 member 角色
- **当** 用户加入租户（非创建者）
- **那么** 自动分配 `{tenant_id}:member` 角色

#### 场景:邀请用户时指定角色
- **当** 租户管理员邀请用户并指定角色
- **那么** 用户接受邀请后获得指定角色
