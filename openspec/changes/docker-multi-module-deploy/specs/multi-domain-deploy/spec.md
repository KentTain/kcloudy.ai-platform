## 新增需求

### 需求:多域名部署配置

系统必须支持将不同模块部署到不同域名。

每个模块必须能够配置独立的外部域名。

模块域名配置必须通过环境变量或配置文件指定。

#### 场景:配置模块域名
- **当** 配置 demo 模块域名为 demo.example.com
- **那么** demo 模块部署到 demo.example.com

#### 场景:模块使用内部路由
- **当** 模块未配置外部域名
- **那么** 模块使用 Vue Router 内部路由

### 需求:Nginx 反向代理配置

系统必须提供 Nginx 配置，支持多域名反向代理。

Nginx 必须将不同域名请求代理到对应的模块服务。

Nginx 必须配置 SSL 终止和 CORS 头。

#### 场景:域名代理到模块服务
- **当** 请求 demo.example.com
- **那么** Nginx 将请求代理到 demo-service

#### 场景:SSL 终止
- **当** 请求使用 HTTPS
- **那么** Nginx 处理 SSL 并转发 HTTP 到后端服务

#### 场景:CORS 配置
- **当** 跨域请求
- **那么** Nginx 返回正确的 CORS 头

### 需求:统一 API 网关

系统必须提供统一 API 网关，所有前端模块通过网关调用后端 API。

API 网关必须统一处理认证、权限和请求转发。

模块禁止直接调用后端 API，必须通过网关。

#### 场景:通过网关调用 API
- **当** 前端调用 /api/v1/*
- **那么** 请求通过 API 网关转发到后端

#### 场景:网关认证
- **当** 请求未携带认证 Token
- **那么** 网关返回 401 未授权

### 需求:菜单跨域跳转

系统必须支持点击菜单项跳转到外部域名。

当菜单项配置了外部域名时，必须使用 window.open 或 location.href 跳转。

#### 场景:点击跨域菜单项
- **当** 点击配置了 domain: 'https://iam.example.com' 的菜单项
- **那么** 浏览器跳转到 https://iam.example.com

#### 场景:点击内部菜单项
- **当** 点击未配置 domain 的菜单项
- **那么** 使用 Vue Router 导航
