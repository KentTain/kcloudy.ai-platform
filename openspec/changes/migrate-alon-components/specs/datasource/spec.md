## 新增需求

### 需求:系统支持多数据库连接

系统必须提供统一的数据源连接器接口，支持 MySQL、PostgreSQL、Oracle、MSSQL、SQLite、ClickHouse、达梦等多种数据库。

#### 场景:MySQL 连接成功

- **当** 使用正确的 MySQL 连接参数创建连接器
- **那么** 系统建立数据库连接并可执行查询

#### 场景:PostgreSQL 连接成功

- **当** 使用正确的 PostgreSQL 连接参数创建连接器
- **那么** 系统建立数据库连接并可执行查询

#### 场景:Oracle 连接成功

- **当** 使用正确的 Oracle 连接参数创建连接器
- **那么** 系统建立数据库连接并可执行查询

#### 场景:连接参数无效

- **当** 使用无效的连接参数（如错误的主机、端口、凭据）
- **那么** 系统抛出 ConnectionError 异常

### 需求:系统提供数据库方言适配

系统必须为不同数据库提供方言适配，处理 SQL 语法差异。

#### 场景:达梦数据库方言适配

- **当** 使用达梦数据库连接执行查询
- **那么** 系统自动应用达梦方言的 SQL 适配

### 需求:系统支持连接池管理

系统必须支持数据库连接池配置，包括连接数、超时、空闲连接回收等参数。

#### 场景:配置连接池参数

- **当** 在配置中指定连接池参数
- **那么** 系统使用指定参数创建连接池

### 需求:系统提供统一查询接口

系统必须提供统一的查询接口，屏蔽底层数据库差异。

#### 场景:执行 SELECT 查询

- **当** 调用连接器的 execute_query 方法
- **那么** 系统返回标准化的查询结果集

#### 场景:执行 INSERT/UPDATE/DELETE

- **当** 调用连接器的 execute_update 方法
- **那么** 系统返回受影响的行数

### 需求:系统支持事务管理

系统必须支持数据库事务，包括开启、提交、回滚操作。

#### 场景:事务提交成功

- **当** 在事务中执行多个操作后调用 commit
- **那么** 所有操作持久化到数据库

#### 场景:事务回滚

- **当** 在事务中发生异常后调用 rollback
- **那么** 所有操作撤销，数据库状态恢复
