## 新增需求

### 需求:模块注册功能

系统必须提供 ModuleRegistry 类，管理模块的注册和查询。

ModuleRegistry 必须提供以下方法：
- `register(module: unknown)`: 注册模块，执行类型和格式验证
- `getRoutes()`: 返回所有已注册模块的路由数组
- `getMenuItems()`: 返回所有已注册模块的菜单项数组
- `getModule(name: string)`: 根据名称获取模块描述符

#### 场景:注册有效模块
- **当** 调用 register() 传入有效的 ModuleDescriptor
- **那么** 模块被添加到注册表，控制台输出注册成功日志

#### 场景:拒绝无效模块
- **当** 调用 register() 传入无效的对象
- **那么** 抛出 TypeError 异常，模块不被注册

#### 场景:拒绝重复注册
- **当** 调用 register() 传入已存在的模块名称
- **那么** 抛出 Error 异常，提示模块已注册

### 需求:路由收集功能

系统必须能够从所有已注册模块收集路由配置。

getRoutes() 方法必须：
- 遍历所有已注册模块
- 调用每个模块的 getRoutes() 方法
- 返回合并后的路由数组

#### 场景:收集单个模块路由
- **当** 注册一个模块后调用 getRoutes()
- **那么** 返回该模块的所有路由

#### 场景:收集多个模块路由
- **当** 注册多个模块后调用 getRoutes()
- **那么** 返回所有模块路由的合并数组

### 需求:依赖验证功能

系统必须在注册时验证模块依赖关系。

依赖验证必须：
- 检查 dependencies 中声明的模块是否已注册
- 未满足依赖时抛出明确的错误信息

#### 场景:依赖满足
- **当** 模块 A 依赖模块 B，且 B 已注册
- **那么** 模块 A 注册成功

#### 场景:依赖不满足
- **当** 模块 A 依赖模块 B，但 B 未注册
- **那么** 抛出 Error，提示缺失依赖
