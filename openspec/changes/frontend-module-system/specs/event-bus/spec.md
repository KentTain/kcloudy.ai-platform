## 新增需求

### 需求:事件总线接口

系统必须提供事件总线，支持发布/订阅模式的模块间通信。

事件总线必须提供以下方法：
- `emit(event, payload)`: 发布事件
- `on(event, handler)`: 订阅事件
- `off(event, handler)`: 取消订阅

事件总线必须支持 TypeScript 类型安全的事件类型定义。

#### 场景:发布和订阅事件
- **当** 模块 A 订阅 `menu:updated` 事件，模块 B 发布该事件
- **那么** 模块 A 的处理函数被调用并收到 payload

#### 场景:取消订阅
- **当** 模块调用 `off(event, handler)`
- **那么** 后续发布该事件不再触发该处理函数

#### 场景:订阅不存在的事件
- **当** 发布一个无订阅者的事件
- **那么** 系统正常运行，无副作用

### 需求:内置事件类型

系统必须定义以下内置事件类型：

- `module:loaded`: 模块加载完成，payload 为 `{ name: string }`
- `menu:updated`: 菜单更新，payload 为 `MenuItem[]`
- `route:added`: 路由添加，payload 为 `RouteRecordRaw[]`

#### 场景:模块加载完成事件
- **当** 模块加载成功
- **那么** 系统必须发布 `module:loaded` 事件

#### 场景:菜单更新事件
- **当** 菜单 Store 数据变更
- **那么** 系统必须发布 `menu:updated` 事件

### 需求:事件总线内存管理

事件总线必须防止内存泄漏。

系统必须在组件卸载时自动清理事件订阅（通过 Vue `onUnmounted` 钩子）。

#### 场景:组件卸载自动清理
- **当** 组件在 `setup()` 中订阅事件后卸载
- **那么** 订阅自动取消，不再接收事件
