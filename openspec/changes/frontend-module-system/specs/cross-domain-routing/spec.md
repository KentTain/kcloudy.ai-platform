## 新增需求

### 需求:跨域路由配置

系统必须支持模块配置外部域名，实现跨域路由跳转。

ModuleDescriptor 必须支持可选的 `domain` 字段，指定模块的外部域名。

当模块配置了外部域名时，菜单点击必须跳转到外部 URL，而非 Vue Router 路由。

#### 场景:模块配置外部域名
- **当** 模块的 MenuItem 包含 `domain: 'https://iam.example.com'`
- **那么** 点击菜单项时打开外部链接

#### 场景:模块使用内部路由
- **当** 模块的 MenuItem 不包含 domain 字段
- **那么** 点击菜单项时使用 Vue Router 导航

### 需求:跨域菜单合并

系统必须支持合并来自不同域名的菜单。

当部署为多域名架构时，主域名必须能够聚合各模块域名的菜单。

#### 场景:多域名菜单聚合
- **当** 系统部署为 demo.example.com 和 iam.example.com
- **那么** 主域名能够显示所有模块的菜单项

### 需求:跨域路由守卫

系统必须提供路由守卫，处理跨域导航。

当用户尝试访问无权限的跨域模块时，必须重定向到登录页或 403 页。

#### 场景:访问无权限跨域模块
- **当** 用户点击跨域菜单项但无权限
- **那么** 显示 403 页面或重定向到登录页
