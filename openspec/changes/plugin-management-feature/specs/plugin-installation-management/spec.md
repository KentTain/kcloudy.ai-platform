# 插件安装使用规范

本文档定义插件安装使用功能的需求规范，包括可用插件浏览、安装卸载、启停控制、配置管理、运行时监控和统计。

## 新增需求

### 需求:租户管理员可以浏览可用插件列表

系统必须提供可用插件浏览接口，从 Tenant 模块获取已注册的插件定义。

#### 场景:浏览已启用的插件
- **当** 租户管理员请求可用插件列表
- **那么** 系统返回 `is_enabled=true` 的插件定义
- **并且** 标记每个插件是否已安装
- **并且** 显示已安装插件的状态

#### 场景:按类型过滤
- **当** 租户管理员指定类型为 "model"
- **那么** 系统只返回类型为 "model" 的可用插件

#### 场景:只显示推荐插件
- **当** 租户管理员指定 `is_recommended=true`
- **那么** 系统只返回标记为推荐的插件

### 需求:租户管理员可以安装插件

系统必须支持租户管理员从可用插件列表中选择并安装插件。

#### 场景:安装成功
- **当** 租户管理员请求安装插件
- **那么** 系统在 Tenant 侧创建安装记录（status=PENDING）
- **并且** 在 AI 侧创建插件配置和运行时状态
- **并且** 更新状态为 ACTIVE
- **并且** 更新插件定义的 refers 计数
- **并且** 返回安装任务 ID

#### 场景:安装失败回滚
- **当** AI 侧创建配置失败
- **那么** 系统发布 `PluginInstallationFailed` 事件
- **并且** Tenant 监听器删除安装记录
- **并且** 返回错误信息

#### 场景:重复安装拒绝
- **当** 租户已安装同一 plugin_id 的插件
- **那么** 系统拒绝安装
- **并且** 返回错误信息"插件已安装"

#### 场景:指定安装类型
- **当** 租户管理员指定 `install_type="shared"`
- **那么** 系统使用共享进程模式安装插件
- **并且** 记录安装类型到 `plugin_installations.install_type`

### 需求:租户管理员可以卸载插件

系统必须支持租户管理员卸载已安装的插件。

#### 场景:卸载成功
- **当** 租户管理员请求卸载插件
- **那么** 系统删除 AI 侧配置和运行时状态
- **并且** 通过 Provider 删除 Tenant 侧安装记录
- **并且** 更新插件定义的 refers 计数（减 1）

#### 场景:卸载失败保留记录
- **当** AI 侧删除失败
- **那么** Tenant 侧安装记录保留
- **并且** 记录错误日志
- **并且** 可通过重试恢复

### 需求:租户管理员可以启动插件

系统必须支持租户管理员启动处于 INACTIVE 状态的插件。

#### 场景:启动成功
- **当** 租户管理员请求启动插件
- **那么** 系统启动插件进程
- **并且** 更新状态为 ACTIVE
- **并且** 记录进程 ID 和端口
- **并且** 更新 `last_started_at` 时间戳

#### 场景:启动失败
- **当** 插件进程启动失败
- **那么** 系统更新状态为 FAILED
- **并且** 记录错误信息
- **并且** 返回错误详情

### 需求:租户管理员可以停止插件

系统必须支持租户管理员停止处于 ACTIVE 状态的插件。

#### 场景:停止成功
- **当** 租户管理员请求停止插件
- **那么** 系统停止插件进程
- **并且** 更新状态为 INACTIVE
- **并且** 更新 `last_stopped_at` 时间戳

#### 场景:强制停止
- **当** 插件进程无响应
- **那么** 系统强制终止进程
- **并且** 记录强制终止日志

### 需求:租户管理员可以查看插件配置

系统必须提供插件配置查询接口。

#### 场景:查看配置成功
- **当** 租户管理员请求插件配置
- **那么** 系统返回 `plugin_config`（能力配置）
- **并且** 返回 `runtime_config`（运行时配置）

### 需求:租户管理员可以更新插件配置

系统必须支持租户管理员更新插件的运行时配置。

#### 场景:更新配置成功
- **当** 租户管理员更新 `runtime_config`
- **那么** 系统更新 `ai.plugin_configs` 表
- **并且** 返回更新时间

#### 场景:更新后重启生效
- **当** 配置更新后插件正在运行
- **那么** 系统提示"配置将在重启后生效"
- **或者** 系统自动重启插件（可配置）

### 需求:租户管理员可以查看插件运行时状态

系统必须提供插件运行时状态监控接口。

#### 场景:查看单个插件状态
- **当** 租户管理员请求插件运行时状态
- **那么** 系统返回进程信息（PID、端口、工作目录）
- **并且** 返回调用统计（调用次数、错误次数、成功率）
- **并且** 返回时间戳（启动时间、最后访问时间、冻结时间）
- **并且** 返回健康检查状态

#### 场景:批量查看运行时状态
- **当** 租户管理员请求所有插件运行时状态
- **那么** 系统返回每个插件的状态摘要
- **并且** 返回汇总统计（运行中进程数、总内存占用、总 CPU 使用率）

### 需求:租户管理员可以查看安装任务

系统必须提供安装任务追踪接口。

#### 场景:查看任务列表
- **当** 租户管理员请求安装任务列表
- **那么** 系统返回任务列表
- **并且** 每个任务包含状态、进度、开始时间、完成时间

#### 场景:查看任务详情
- **当** 租户管理员请求任务详情
- **那么** 系统返回任务步骤列表
- **并且** 返回每个步骤的状态和进度
- **并且** 返回执行日志

### 需求:系统提供插件使用统计数据

系统必须提供插件使用的统计数据。

#### 场景:获取状态统计
- **当** 请求统计数据
- **那么** 返回各状态插件数量（ACTIVE、INACTIVE、PENDING、FAILED）

#### 场景:获取使用统计
- **当** 请求统计数据
- **那么** 返回今日调用次数
- **并且** 返回总调用次数和错误次数
- **并且** 返回成功率和平均响应时间

#### 场景:获取运行时统计
- **当** 请求统计数据
- **那么** 返回运行中进程数
- **并且** 返回已冻结进程数
- **并且** 返回总内存占用和 CPU 使用率
- **并且** 返回活跃端点数

### 需求:不同租户的插件安装数据隔离

系统必须保证插件安装数据的租户隔离。

#### 场景:查询已安装插件只返回本租户数据
- **当** 租户 A 查询已安装插件列表
- **那么** 系统只返回租户 A 安装的插件
- **并且** 不包含其他租户的安装记录

#### 场景:安装记录包含租户标识
- **当** 创建安装记录
- **那么** 记录必须包含 `tenant_id` 字段
- **并且** 通过唯一约束保证同一租户同一插件只能有一条记录

### 需求:插件状态正确流转

系统必须保证插件状态的正确流转。

#### 场景:安装成功状态流转
- **当** 插件安装成功
- **那么** 状态从 PENDING 变为 ACTIVE

#### 场景:安装失败状态流转
- **当** 插件安装失败
- **那么** 状态从 PENDING 变为 FAILED
- **并且** 记录错误信息

#### 场景:停止插件状态流转
- **当** 插件被停止
- **那么** 状态从 ACTIVE 变为 INACTIVE

#### 场景:启动插件状态流转
- **当** 插件被启动
- **那么** 状态从 INACTIVE 变为 ACTIVE

### 需求:插件自动冻结

系统必须支持插件的自动冻结机制。

#### 场景:长时间未访问自动冻结
- **当** 插件超过 `freeze_threshold_hours` 小时未被访问
- **那么** 系统自动冻结插件进程
- **并且** 记录 `frozen_at` 时间戳

#### 场景:访问时自动解冻
- **当** 插件被冻结后有新的访问请求
- **那么** 系统自动解冻插件
- **并且** 清空 `frozen_at` 时间戳
