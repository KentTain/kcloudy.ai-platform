# 插件定义管理规范

本文档定义插件定义管理功能的需求规范，包括插件定义的注册、查询、标记和统计。

## 新增需求

### 需求:平台管理员可以通过服务器目录扫描注册插件定义

系统必须支持平台管理员从服务器指定目录扫描插件包（.zip 格式）并自动注册到插件定义表。

#### 场景:扫描成功注册新插件
- **当** 平台管理员请求扫描 `server/plugins` 目录
- **那么** 系统扫描目录下所有 .zip 文件
- **并且** 解析每个插件包的 manifest
- **并且** 将新的插件定义注册到 `tenant.plugin_definitions` 表
- **并且** 返回扫描结果（成功数、跳过数、失败数）

#### 场景:扫描跳过已存在的插件
- **当** 插件包的 `plugin_unique_identifier` 已存在于数据库
- **那么** 系统跳过该插件
- **并且** 返回结果中标记为 `skipped`

#### 场景:扫描失败记录错误
- **当** 插件包格式无效或 manifest 缺失
- **那么** 系统记录错误信息
- **并且** 返回结果中标记为 `failed`
- **并且** 不影响其他插件的处理

### 需求:平台管理员可以通过本地上传注册插件定义

系统必须支持平台管理员通过 Web 界面上传插件包并注册。

#### 场景:上传成功注册插件
- **当** 平台管理员上传有效的 .zip 插件包
- **那么** 系统解析插件包
- **并且** 上传到对象存储（MinIO）
- **并且** 注册插件定义
- **并且** 返回插件 ID 和版本信息

#### 场景:上传覆盖已存在的插件
- **当** 平台管理员上传已存在的插件包且指定 `overwrite=true`
- **那么** 系统更新现有插件定义
- **并且** 更新对象存储中的文件

#### 场景:上传格式错误拒绝
- **当** 上传的文件不是 .zip 格式
- **那么** 系统拒绝请求
- **并且** 返回错误信息"插件文件必须是.zip格式"

### 需求:平台管理员可以通过远程 URL 拉取注册插件定义

系统必须支持平台管理员通过远程 URL 下载插件包并注册。

#### 场景:远程拉取成功注册
- **当** 平台管理员提供有效的插件包 URL
- **那么** 系统下载插件包
- **并且** 解析验证
- **并且** 上传到对象存储
- **并且** 注册插件定义

#### 场景:远程拉取超时失败
- **当** 远程 URL 响应超过 30 秒
- **那么** 系统终止下载
- **并且** 返回错误信息"下载超时"

#### 场景:远程拉取大小限制
- **当** 插件包大小超过 100MB
- **那么** 系统拒绝下载
- **并且** 返回错误信息"插件包大小超过限制"

### 需求:平台管理员可以查询插件定义列表

系统必须提供插件定义的分页查询接口，支持关键词搜索和类型过滤。

#### 场景:查询全部插件定义
- **当** 平台管理员请求插件定义列表
- **那么** 系统返回所有插件定义
- **并且** 包含插件基本信息（ID、名称、版本、类型、图标、描述）
- **并且** 包含安装次数（refers）
- **并且** 包含推荐和启用状态

#### 场景:按关键词搜索
- **当** 平台管理员提供关键词 "ollama"
- **那么** 系统返回名称或 ID 包含 "ollama" 的插件

#### 场景:按类型过滤
- **当** 平台管理员指定类型为 "model"
- **那么** 系统只返回类型为 "model" 的插件定义

### 需求:平台管理员可以查看插件定义详情

系统必须提供插件定义详情查询接口，返回完整的 manifest 信息。

#### 场景:查看详情成功
- **当** 平台管理员请求插件定义详情
- **那么** 系统返回完整的 declaration（manifest）
- **并且** 返回安装统计信息
- **并且** 返回元数据（创建时间、更新时间）

#### 场景:插件不存在返回 404
- **当** 请求的 plugin_id 不存在
- **那么** 系统返回 404 错误

### 需求:平台管理员可以标记插件为推荐

系统必须支持平台管理员将插件标记为推荐或取消推荐。

#### 场景:标记推荐成功
- **当** 平台管理员将插件标记为推荐
- **那么** 系统更新 `is_recommended` 字段为 true
- **并且** 返回更新后的状态

#### 场景:取消推荐成功
- **当** 平台管理员取消插件的推荐标记
- **那么** 系统更新 `is_recommended` 字段为 false

### 需求:平台管理员可以启用或禁用插件定义

系统必须支持平台管理员启用或禁用插件定义。

#### 场景:禁用插件定义
- **当** 平台管理员禁用插件定义
- **那么** 系统更新 `is_enabled` 字段为 false
- **并且** 该插件在可用插件列表中不再显示
- **并且** 已安装该插件的租户不受影响

#### 场景:启用插件定义
- **当** 平台管理员启用插件定义
- **那么** 系统更新 `is_enabled` 字段为 true
- **并且** 该插件在可用插件列表中重新显示

### 需求:平台管理员可以删除插件定义

系统必须支持平台管理员删除不再需要的插件定义。

#### 场景:删除无安装记录的插件
- **当** 插件的 `refers = 0`（无租户安装）
- **那么** 系统删除插件定义记录
- **并且** 删除对象存储中的插件包

#### 场景:拒绝删除有安装记录的插件
- **当** 插件的 `refers > 0`（有租户安装）
- **那么** 系统拒绝删除
- **并且** 返回错误信息"插件正在被使用，无法删除"

### 需求:系统提供插件定义统计数据

系统必须提供定义侧和安装侧的统计数据。

#### 场景:获取定义侧统计
- **当** 请求统计数据
- **那么** 返回已注册插件总数
- **并且** 返回按类型分布（model、tool、agent）
- **并且** 返回推荐插件数

#### 场景:获取安装侧统计
- **当** 请求统计数据
- **那么** 返回全平台总安装次数
- **并且** 返回活跃安装数（ACTIVE 状态）
- **并且** 返回本周新增安装数

### 需求:不同租户的插件定义数据隔离

系统必须保证插件定义数据的正确隔离。

#### 场景:插件定义为全局共享
- **当** 插件定义注册到 `tenant.plugin_definitions`
- **那么** 该定义对所有租户可见
- **并且** 无 tenant_id 字段

#### 场景:安装记录按租户隔离
- **当** 租户 A 安装插件
- **那么** 安装记录包含 tenant_id
- **并且** 租户 B 无法看到租户 A 的安装记录
