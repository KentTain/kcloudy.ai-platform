## 新增需求

### 需求:租户管理员可以启动已安装的插件

系统必须支持租户管理员启动状态为 `INACTIVE` 的插件。启动过程必须创建插件进程并更新运行时状态。

#### 场景:成功启动插件
- **当** 租户管理员请求 `POST /ai/console/v1/plugins/installations/{plugin_id}/start` 且插件状态为 `INACTIVE`
- **那么** 系统创建插件进程，更新 `plugin_installations` 状态为 `ACTIVE`，初始化 `plugin_runtime_states` 记录，返回进程信息（pid、port）

#### 场景:启动已在运行的插件
- **当** 租户管理员请求启动状态为 `ACTIVE` 的插件
- **那么** 系统返回 400 错误"插件已在运行"

#### 场景:启动插件时插件未安装
- **当** 租户管理员请求启动未安装的插件
- **那么** 系统返回 404 错误"插件未安装"

### 需求:租户管理员可以停止运行中的插件

系统必须支持租户管理员停止状态为 `ACTIVE` 的插件。停止过程必须终止插件进程并更新运行时状态。

#### 场景:成功停止插件
- **当** 租户管理员请求 `POST /ai/console/v1/plugins/installations/{plugin_id}/stop` 且插件状态为 `ACTIVE`
- **那么** 系统终止插件进程，更新 `plugin_installations` 状态为 `INACTIVE`，更新 `plugin_runtime_states` 的停止时间

#### 场景:停止未运行的插件
- **当** 租户管理员请求停止状态为 `INACTIVE` 的插件
- **那么** 系统返回 400 错误"插件未在运行"

### 需求:租户管理员可以查看插件配置

系统必须支持租户管理员查看插件的配置信息，包括插件能力配置和运行时配置。

#### 场景:成功查看插件配置
- **当** 租户管理员请求 `GET /ai/console/v1/plugins/installations/{plugin_id}/config`
- **那么** 系统返回插件配置，包含 `plugin_config`（能力配置）和 `runtime_config`（运行时配置）

### 需求:租户管理员可以更新插件配置

系统必须支持租户管理员更新插件的运行时配置。

#### 场景:成功更新插件配置
- **当** 租户管理员请求 `PATCH /ai/console/v1/plugins/installations/{plugin_id}/config` 且提供配置更新
- **那么** 系统更新 `plugin_configs` 表的配置字段，返回更新后的配置

#### 场景:更新配置时插件未安装
- **当** 租户管理员请求更新未安装插件的配置
- **那么** 系统返回 404 错误"插件未安装"

### 需求:租户管理员可以查看单个插件的运行时状态

系统必须支持租户管理员查看单个插件的运行时状态，包括进程信息、统计信息、健康状态。

#### 场景:成功查看插件运行时状态
- **当** 租户管理员请求 `GET /ai/console/v1/plugins/installations/{plugin_id}/runtime-state`
- **那么** 系统返回运行时状态，包含进程信息（pid、port、work_directory）、统计信息（调用次数、错误次数、成功率）、健康状态

#### 场景:查看运行时状态时插件未运行
- **当** 租户管理员请求查看状态为 `INACTIVE` 插件的运行时状态
- **那么** 系统返回运行时状态，但进程信息和统计信息为空或初始值

### 需求:租户管理员可以批量查看运行时状态

系统必须支持租户管理员批量查看所有已安装插件的运行时状态摘要。

#### 场景:成功批量查看运行时状态
- **当** 租户管理员请求 `GET /ai/console/v1/plugins/runtime-states`
- **那么** 系统返回所有已安装插件的运行时状态列表，包含汇总统计（运行中进程数、总内存占用、总 CPU 使用率）

### 需求:运行时管理必须有权限控制

系统必须对运行时管理 API 进行权限校验，要求 `ai:plugin:read` 或 `ai:plugin:write` 权限。

#### 场景:无权限启动插件
- **当** 用户请求启动插件且没有 `ai:plugin:write` 权限
- **那么** 系统返回 403 错误"权限不足"

#### 场景:无权限查看运行时状态
- **当** 用户请求查看运行时状态且没有 `ai:plugin:read` 权限
- **那么** 系统返回 403 错误"权限不足"

### 需求:插件进程必须支持健康检查

系统必须定期检查运行中的插件进程健康状态。

#### 场景:健康检查通过
- **当** 插件进程正常运行
- **那么** 系统标记健康状态为 `healthy`

#### 场景:健康检查失败
- **当** 插件进程异常退出或无响应
- **那么** 系统标记健康状态为 `unhealthy`，记录错误信息
