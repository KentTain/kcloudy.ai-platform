## 新增需求

### 需求:平台管理员可以查看插件定义统计数据

系统必须支持平台管理员查看全局插件定义统计数据，包括已注册插件总数、按类型分布、推荐插件数、总安装次数。

#### 场景:成功获取插件定义统计数据
- **当** 平台管理员请求 `GET /tenant/admin/v1/plugin-definitions/statistics`
- **那么** 系统返回统计数据，包含 `definition_stats`（总数、按类型分布、推荐数、启用数）和 `installation_stats`（总安装数、活跃安装数、本周新增安装数）

### 需求:租户管理员可以查看插件使用统计数据

系统必须支持租户管理员查看当前租户的插件使用统计数据，包括状态统计、使用统计、运行时统计。

#### 场景:成功获取插件使用统计数据
- **当** 租户管理员请求 `GET /ai/console/v1/plugins/statistics`
- **那么** 系统返回统计数据，包含 `status_stats`（活跃数、停用数、失败数）、`usage_stats`（今日调用数、总调用数、错误数、成功率、平均响应时间）、`runtime_stats`（运行进程数、冻结进程数、总内存占用、总 CPU 使用率、活跃端点数）

### 需求:统计数据必须实时或准实时计算

系统必须确保统计数据反映最新状态，或使用缓存时明确标注缓存时间。

#### 场景:统计数据实时计算
- **当** 用户请求统计数据
- **那么** 系统从数据库查询最新数据并计算统计结果

#### 场景:统计数据使用缓存
- **当** 系统配置使用缓存
- **那么** 统计数据使用缓存，缓存 TTL 不超过 5 分钟，响应中包含 `cached_at` 字段

### 需求:统计 API 必须有权限控制

系统必须对统计 API 进行权限校验。

#### 场景:无权限查看 Tenant 统计数据
- **当** 用户请求 `GET /tenant/admin/v1/plugin-definitions/statistics` 且没有 `tenant:plugin:read` 权限
- **那么** 系统返回 403 错误"权限不足"

#### 场景:无权限查看 AI 统计数据
- **当** 用户请求 `GET /ai/console/v1/plugins/statistics` 且没有 `ai:plugin:read` 权限
- **那么** 系统返回 403 错误"权限不足"

### 需求:统计数据必须支持多租户隔离

系统必须确保租户只能查看自己租户的统计数据。

#### 场景:租户 A 无法查看租户 B 的统计数据
- **当** 租户 A 请求 AI 模块统计 API
- **那么** 系统仅返回租户 A 的统计数据，不包含其他租户数据
