## ADDED Requirements

### 需求:用户菜单 API 端点

系统必须提供用户菜单 API，返回当前用户有权限的菜单树。

#### 场景:获取用户菜单成功
- **当** 已登录用户请求 `GET /api/v1/user/menus`
- **那么** 返回 HTTP 200，包含用户有权限的菜单树

#### 场景:未登录用户请求菜单
- **当** 未登录用户请求 `GET /api/v1/user/menus`
- **那么** 返回 HTTP 401 Unauthorized

### 需求:菜单权限过滤

系统必须根据用户权限过滤菜单。

#### 场景:用户有菜单权限
- **当** 用户拥有菜单 A 的权限
- **那么** 响应中包含菜单 A

#### 场景:用户无菜单权限
- **当** 用户不拥有菜单 A 的权限
- **那么** 响应中不包含菜单 A

#### 场景:父菜单权限覆盖子菜单
- **当** 用户拥有父菜单权限但不拥有子菜单权限
- **那么** 响应中包含父菜单但不包含子菜单

#### 场景:子菜单权限包含父菜单
- **当** 用户拥有子菜单权限但不拥有父菜单权限
- **那么** 响应中同时包含父菜单和子菜单（父菜单为展示需要）

### 需求:菜单树结构

系统必须返回树形结构的菜单数据。

#### 场景:菜单树响应格式
- **当** 用户请求菜单
- **那么** 响应格式为：
```json
{
  "data": [
    {
      "id": "uuid",
      "code": "iam",
      "name": "IAM",
      "icon": "Settings",
      "path": null,
      "sort_order": 1,
      "children": [
        {
          "id": "uuid",
          "code": "iam.users",
          "name": "用户管理",
          "path": "/iam/users",
          "sort_order": 1,
          "children": []
        }
      ]
    }
  ]
}
```

#### 场景:空菜单列表
- **当** 用户无任何菜单权限
- **那么** 返回 HTTP 200，data 为空数组

### 需求:只返回启用模块的菜单

系统必须只返回已启用模块的菜单。

#### 场景:模块已启用
- **当** 模块 is_active=true
- **那么** 该模块的菜单参与权限过滤

#### 场景:模块已禁用
- **当** 模块 is_active=false
- **那么** 该模块的菜单不返回给用户

### 需求:多租户菜单隔离

系统必须保证租户间的菜单隔离。

#### 场景:租户只能看到已分配模块的菜单
- **当** 租户 A 只分配了 IAM 模块
- **那么** 租户 A 的用户只能看到 IAM 模块的菜单

#### 场景:租户未分配任何模块
- **当** 租户未分配任何模块
- **那么** 该租户用户获取菜单返回空数组
