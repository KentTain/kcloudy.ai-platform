## 新增需求

### 需求:系统必须提供全局共享角色

系统必须在 tenant 模块的模块定义层提供两个全局共享角色：`sysAdmin`（系统管理员）和 `normalUser`（普通用户），所有模块共享这两个角色，而不是每个模块独立定义角色。

#### 场景:全局角色定义
- **当** 系统启动时
- **那么** `ModuleDefinitionSyncService` 必须将全局角色同步到 `tenant.module_roles` 表
- **且** 全局角色的 `module_id` 必须为 `NULL`，表示不属于任何特定模块

#### 场景:全局角色权限通配符
- **当** 定义全局角色权限时
- **那么** `sysAdmin` 必须使用通配符 `*:*:*` 匹配所有权限
- **且** `normalUser` 必须使用通配符 `*:*:read` 匹配所有只读权限
- **且** 系统必须使用 `fnmatch` 展开通配符为具体权限编码

### 需求:模块必须移除独立的角色定义

所有业务模块（iam、tenant、demo、ai）必须移除 `get_module_definition()` 返回的 `default_roles` 列表，改为空列表。

#### 场景:模块定义更新
- **当** 各模块的 `module.py` 定义模块元数据时
- **那么** `default_roles` 必须返回空列表 `[]`
- **且** 菜单和权限定义保持不变

### 需求:租户分配模块时必须同步全局角色

当租户分配模块时，`ModuleSyncService` 必须同步全局角色到租户实例层。

#### 场景:模块分配同步全局角色
- **当** 租户分配模块时
- **那么** 系统必须查询 `module_roles` 表中 `module_id IS NULL` 的全局角色
- **且** 为租户创建 `iam.roles` 记录，`ref_id` 指向全局角色
- **且** 通过 `code` 字段匹配全局角色（`sysAdmin`、`normalUser`）

#### 场景:全局角色权限关联
- **当** 同步全局角色到租户实例层时
- **那么** 系统必须根据通配符展开的权限编码，创建 `role_permissions` 关联记录
- **且** 关联到租户实例层的所有权限（通过 `permission.code` 匹配）

### 需求:全局角色禁止编辑和删除

全局角色是系统内置角色，禁止通过管理界面编辑或删除。

#### 场景:全局角色保护
- **当** 用户尝试编辑或删除全局角色时
- **那么** 系统必须拒绝操作
- **且** 返回错误提示"系统内置角色禁止修改"

### 需求:用户分配角色时可通过 code 查找

当为用户分配角色时，系统必须支持通过角色 `code` 查找角色，而不是仅通过 `id`。

#### 场景:通过 code 查找角色
- **当** 初始化默认用户时
- **那么** 系统可以通过 `SELECT id FROM iam.roles WHERE code = 'sysAdmin'` 查找角色
- **且** 无需预先知道角色的 UUID
