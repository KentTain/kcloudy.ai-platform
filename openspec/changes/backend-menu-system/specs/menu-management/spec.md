## 新增需求

### 需求:菜单数据模型

系统必须提供菜单数据模型，支持树形结构和权限关联。

菜单模型必须包含以下字段：
- 名称（name）：必填，最大 100 字符
- 路由路径（path）：必填，最大 255 字符
- 图标（icon）：可选，最大 100 字符
- 组件路径（component）：可选，最大 255 字符
- 是否显示（is_visible）：默认 true
- 排序（order）：默认 0
- 父菜单 ID（parent_id）：可选，用于构建树形结构

菜单必须支持多对多关联权限。

#### 场景:创建根菜单
- **当** 创建一个不指定 parent_id 的菜单
- **那么** 该菜单作为根菜单存储，level 为 0

#### 场景:创建子菜单
- **当** 创建一个指定 parent_id 的菜单
- **那么** 该菜单作为子菜单存储，level 为父菜单 level + 1

#### 场景:菜单关联权限
- **当** 为菜单分配多个权限
- **那么** 菜单-权限关联表中存在对应记录

#### 场景:菜单层级限制
- **当** 创建菜单时 level 超过 3
- **那么** 系统必须拒绝并返回错误

### 需求:菜单 CRUD API

系统必须提供菜单的增删改查 API，端点位于 `/admin/v1/menus`。

所有端点必须通过 JWT 认证，且调用者必须具备菜单管理权限。

#### 场景:获取菜单列表
- **当** GET `/admin/v1/menus`
- **那么** 返回所有菜单的扁平列表，按 order 升序排序

#### 场景:创建菜单
- **当** POST `/admin/v1/menus` 携带合法菜单数据
- **那么** 系统创建菜单并返回菜单详情，HTTP 状态码 201

#### 场景:更新菜单
- **当** PUT `/admin/v1/menus/{id}` 携带更新数据
- **那么** 系统更新菜单字段并返回更新后的菜单详情

#### 场景:删除菜单
- **当** DELETE `/admin/v1/menus/{id}` 且该菜单无子菜单
- **那么** 系统软删除菜单（设置 deleted_at）

#### 场景:删除有子菜单的菜单
- **当** DELETE `/admin/v1/menus/{id}` 且该菜单存在子菜单
- **那么** 系统必须拒绝并返回错误，HTTP 状态码 400

#### 场景:未认证访问
- **当** 未携带 JWT 调用任何菜单端点
- **那么** 系统必须返回 HTTP 401

### 需求:菜单树查询 API

系统必须提供菜单树查询 API，返回嵌套的树形结构。

#### 场景:获取完整菜单树
- **当** GET `/admin/v1/menus/tree`
- **那么** 返回所有菜单的嵌套树形结构，每个节点包含 children 字段

#### 场景:获取用户菜单树
- **当** GET `/admin/v1/menus/user`
- **那么** 返回当前用户权限范围内可访问的菜单树

#### 场景:用户无权限时返回空树
- **当** 当前用户没有任何菜单权限
- **那么** GET `/admin/v1/menus/user` 返回空数组

#### 场景:菜单按 is_visible 过滤
- **当** 查询菜单树
- **那么** is_visible 为 false 的菜单不出现在结果中

### 需求:菜单种子数据

系统必须提供初始菜单种子数据，覆盖 demo、iam、tenant 三个模块。

种子数据必须通过 Alembic 数据迁移脚本注入，且必须支持回滚。

#### 场景:首次部署初始化
- **当** 运行数据库迁移到最新版本
- **那么** `iam_menu` 表中存在 demo、iam、tenant 三个根菜单

#### 场景:种子数据回滚
- **当** 运行数据库迁移降级到种子数据之前的版本
- **那么** 种子菜单数据被完整删除

### 需求:菜单数据库迁移

系统必须通过 Alembic 迁移脚本创建菜单相关表。

迁移必须包含：
- 创建 `iam_menu` 表
- 创建 `iam_menu_permission` 关联表
- 必要的索引（parent_id、path）

#### 场景:升级迁移
- **当** 运行 `alembic upgrade head`
- **那么** `iam_menu` 和 `iam_menu_permission` 表被创建

#### 场景:降级迁移
- **当** 运行 `alembic downgrade`
- **那么** `iam_menu` 和 `iam_menu_permission` 表被删除
