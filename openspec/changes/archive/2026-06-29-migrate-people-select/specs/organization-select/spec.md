## 新增需求

### 需求:组织选择组件提供 Popover 选择入口

组织选择组件必须提供选择入口，支持单选和多选模式。

#### 场景:单选模式选择组织

- **当** 用户点击组织选择输入框
- **那么** 系统打开组织选择弹窗
- **当** 用户在组织树中选择一个组织
- **那么** 系统选中该组织并关闭弹窗
- **当** 弹窗关闭后
- **那么** 输入框显示选中组织的名称

#### 场景:多选模式选择多个组织

- **当** 组件设置 `multiple="true"`
- **那么** 系统允许选择多个组织
- **当** 用户选择多个组织
- **那么** 系统将组织以 Badge 标签形式展示在输入框中
- **当** 选中组织超过容器宽度
- **那么** 系统显示 "N+" 标签折叠溢出组织

### 需求:组织选择组件提供树形选择视图

组织选择组件必须提供树形选择视图。

#### 场景:显示组织树

- **当** 组织选择弹窗打开
- **那么** 左侧显示组织树
- **当** 组织树加载完成
- **那么** 根节点默认展开

#### 场景:搜索组织

- **当** 用户在搜索框输入关键词
- **那么** 系统过滤组织树，只显示匹配的组织
- **当** 清空搜索关键词
- **那么** 系统恢复显示完整组织树

#### 场景:展开和折叠组织节点

- **当** 用户点击组织的展开按钮
- **那么** 系统展开该组织，显示子组织
- **当** 用户再次点击展开按钮
- **那么** 系统折叠该组织，隐藏子组织

### 需求:组织选择组件显示已选组织

组织选择组件必须在右侧面板显示已选组织列表。

#### 场景:显示已选组织

- **当** 用户选择了组织
- **那么** 右侧面板显示已选组织卡片
- **当** 卡片包含组织名称和组织路径

#### 场景:移除已选组织

- **当** 用户点击组织卡片的移除按钮
- **那么** 系统从已选列表移除该组织
- **当** 用户点击"清空"按钮
- **那么** 系统清除所有已选组织

### 需求:组织选择组件支持禁用状态

组织选择组件必须支持禁用状态。

#### 场景:禁用状态

- **当** 组件设置 `disabled="true"`
- **那么** 输入框显示为灰色且不可点击
- **当** 用户点击禁用的输入框
- **那么** 系统不打开弹窗

### 需求:组织选择组件支持清空选择

组织选择组件必须支持清空已选组织。

#### 场景:清空已选组织

- **当** 用户点击清空按钮
- **那么** 系统清除所有已选组织
- **当** 组件触发 `clear` 事件
