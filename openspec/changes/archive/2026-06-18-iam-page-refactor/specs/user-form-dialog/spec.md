## 新增需求

### 需求:用户创建编辑弹窗
系统必须提供 UserFormDialog 组件，用于创建和编辑用户。

#### 场景:创建用户
- **当** 点击人员管理页面的"新增人员"按钮
- **那么** 弹窗打开，表单包含姓名、用户名、所属部门、状态、手机号、邮箱字段，以及角色分配 Checkbox 列表

#### 场景:编辑用户
- **当** 点击用户行的"编辑"按钮
- **那么** 弹窗打开，表单填充用户的当前信息，用户名字段为只读

#### 场景:角色分配
- **当** 弹窗显示角色列表
- **那么** 每个角色显示为 Checkbox，附带角色名称和描述，已分配的角色默认勾选

#### 场景:选择部门
- **当** 弹窗中的"所属部门"字段
- **那么** 使用 TreeSelect 组件展示部门树，支持搜索和选择

#### 场景:保存用户
- **当** 提交创建表单，姓名和用户名为必填
- **那么** 调用创建用户 API，成功后关闭弹窗并刷新列表

#### 场景:保存编辑
- **当** 提交编辑表单
- **那么** 调用更新用户 API，成功后关闭弹窗并刷新列表
