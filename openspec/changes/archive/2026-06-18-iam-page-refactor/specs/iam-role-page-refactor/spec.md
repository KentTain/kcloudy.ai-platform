## 新增需求

### 需求:角色管理页面布局改造
角色管理页面必须采用左侧角色列表 + 右侧详情面板的布局，右侧包含 Tabs（角色成员、权限列表）。

#### 场景:左侧角色列表
- **当** 页面加载完成
- **那么** 左侧显示角色列表，每个角色显示名称和编码，点击角色后右侧显示该角色详情

#### 场景:角色详情头部
- **当** 选中某个角色
- **那么** 右侧顶部显示角色名称、编码、描述信息和操作按钮（编辑、删除、新增角色）

### 需求:角色成员 Tab
角色详情面板必须包含"角色成员"Tab，显示属于该角色的用户列表。

#### 场景:查看角色成员
- **当** 切换到"角色成员"Tab
- **那么** 以表格显示该角色的用户列表：姓名、账号、联系方式、状态

#### 场景:添加成员
- **当** 点击"添加成员"按钮
- **那么** 打开 PeopleSelectDialog，选择人员后调用角色成员分配 API

#### 场景:移除成员
- **当** 点击某个成员的"移除"按钮
- **那么** 弹出确认对话框，确认后调用移除角色成员 API

### 需求:权限列表 Tab
角色详情面板必须包含"权限列表"Tab，显示该角色已分配的权限。

#### 场景:查看权限列表
- **当** 切换到"权限列表"Tab
- **那么** 按资源分组显示该角色的权限列表

#### 场景:分配权限
- **当** 点击"分配权限"按钮
- **那么** 打开权限分配对话框，以 Checkbox 树形式展示所有权限
