## 新增需求

### 需求:部门管理页面布局改造
部门管理页面必须采用左右布局：左侧 300px 固定宽度显示组织树，右侧自适应宽度显示组织详情和 Tabs。

#### 场景:初始加载骨架屏
- **当** 页面首次加载，组织树数据未返回
- **那么** 左侧树区域和右侧详情区域显示 Skeleton 骨架屏占位

#### 场景:页面头部操作按钮
- **当** 页面头部显示标题"部门管理"和描述文字
- **那么** 右侧显示操作按钮组（新增同级、新增子组织、编辑、删除），按钮根据选中状态动态禁用

#### 场景:选中组织节点
- **当** 用户点击组织树中的某个节点
- **那么** 右侧显示该组织详情区域，包含组织路径、直属成员数、下级累计数等统计信息

### 需求:组织信息 Tab
右侧详情区域必须包含"组织信息"Tab，显示选中组织的基本信息。

#### 场景:查看组织信息
- **当** 切换到"组织信息"Tab
- **那么** 以网格布局（2列）显示组织名称、编码、负责人、联系电话、状态、排序号、说明等字段

### 需求:下级组织 Tab
右侧详情区域必须包含"下级组织"Tab，显示选中组织的直接子组织列表。

#### 场景:查看下级组织
- **当** 切换到"下级组织"Tab
- **那么** 以表格显示下级组织的名称、编码、负责人、成员数、状态，每行有"查看"链接可跳转到该组织

#### 场景:无下级组织
- **当** 切换到"下级组织"Tab 且当前组织无子组织
- **那么** 显示"当前组织暂无下级组织"空状态提示

### 需求:直属成员 Tab
右侧详情区域必须包含"直属成员"Tab，显示选中组织的直属成员列表。

#### 场景:查看直属成员
- **当** 切换到"直属成员"Tab
- **那么** 以表格显示成员的姓名、账号、联系方式、状态，每行有"启用/禁用"和"删除"操作

#### 场景:添加成员
- **当** 点击"添加成员"按钮
- **那么** 打开 PeopleSelectDialog 人员选择器弹窗，选择人员后调用批量添加成员 API

#### 场景:移除成员
- **当** 点击某个成员的"删除"按钮
- **那么** 弹出确认对话框，确认后调用移除成员 API，刷新成员列表

#### 场景:切换成员状态
- **当** 点击启用成员的"禁用"按钮
- **那么** 弹出确认对话框，确认后调用停用 API，刷新成员列表

### 需求:组织树搜索高亮
组织树必须支持关键词搜索，匹配的文字高亮显示。

#### 场景:搜索过滤
- **当** 在搜索框输入关键字
- **那么** 组织树只显示名称或编码包含关键字的组织及其祖先路径，匹配文字高亮显示
