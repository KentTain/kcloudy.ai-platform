## 新增需求

### 需求:人员管理页面布局改造
人员管理页面必须采用顶部统计卡片 + 左侧部门树筛选 + 右侧人员列表的布局。

#### 场景:统计卡片显示
- **当** 页面加载完成
- **那么** 顶部显示 4 个统计卡片：人员总数、启用账号数、当前部门人数、多角色成员数，每个卡片附带辅助说明文字

#### 场景:部门树筛选
- **当** 用户点击左侧部门树中的某个节点
- **那么** 右侧人员列表自动刷新，仅显示该部门及其下级部门的人员；卡片"当前部门"显示选中部门名称

#### 场景:清除部门筛选
- **当** 用户点击部门树面板的"查看全部"按钮
- **那么** 清除部门筛选，显示所有人员

### 需求:人员列表多条件筛选
人员列表必须支持关键词搜索、状态筛选、角色筛选三个条件同时生效。

#### 场景:关键词搜索
- **当** 在搜索框输入关键词并回车
- **那么** 人员列表按姓名、用户名、手机号模糊匹配进行筛选

#### 场景:状态筛选
- **当** 在状态下拉框选择"启用"或"停用"
- **那么** 人员列表自动按状态筛选

#### 场景:角色筛选
- **当** 在角色下拉框选择某个角色
- **那么** 人员列表自动按角色筛选

#### 场景:重置筛选
- **当** 点击"重置筛选"按钮
- **那么** 清空所有筛选条件，恢复默认视图

### 需求:人员列表行内操作
人员列表每行必须提供编辑、启用/禁用、重置密码、删除操作。

#### 场景:编辑用户
- **当** 点击用户行的"编辑"链接
- **那么** 打开 UserFormDialog 弹窗，填充用户当前信息

#### 场景:启用/禁用用户
- **当** 点击启用用户的"禁用"链接或停用用户的"启用"链接
- **那么** 直接调用对应 API 切换状态，刷新列表

#### 场景:重置密码
- **当** 点击用户行的"重置密码"链接
- **那么** 弹出确认对话框，确认后调用重置密码 API，成功后 toast 提示

#### 场景:删除用户
- **当** 点击用户行的"删除"链接
- **那么** 弹出 AlertDialog 确认框，确认后调用删除 API
