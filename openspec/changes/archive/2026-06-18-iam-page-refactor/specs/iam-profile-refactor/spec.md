## 新增需求

### 需求:账户管理页面布局改造
账户管理页面必须采用头像区 + Tabs（个人信息、安全设置）的横向 Tab 布局。

#### 场景:页面头部
- **当** 页面加载完成
- **那么** 顶部显示用户头像（圆形，可点击上传）和用户名、描述文字

### 需求:个人信息 Tab
"个人信息"Tab 必须以表单形式展示，支持昵称、邮箱、手机号的编辑。

#### 场景:查看个人信息
- **当** 切换到"个人信息"Tab
- **那么** 显示 Card 容器包含表单字段：用户名（只读）、昵称、邮箱、手机号，底部有"保存修改"按钮

#### 场景:保存个人信息
- **当** 修改表单字段后点击"保存修改"按钮
- **那么** 调用 updateCurrentUser API，成功后 toast 提示并更新全局 userStore

### 需求:安全设置 Tab
"安全设置"Tab 必须包含修改密码功能。

#### 场景:查看安全设置
- **当** 切换到"安全设置"Tab
- **那么** 显示 Card 容器包含"密码"项，附带"修改密码"按钮

#### 场景:修改密码弹窗
- **当** 点击"修改密码"按钮
- **那么** 打开 Dialog 弹窗，包含原密码、新密码、确认密码三个字段，每个密码字段有显示/隐藏切换按钮

#### 场景:密码验证
- **当** 提交修改密码表单
- **那么** zod schema 验证：原密码必填、新密码 8-32 位、确认密码必须与新密码一致
