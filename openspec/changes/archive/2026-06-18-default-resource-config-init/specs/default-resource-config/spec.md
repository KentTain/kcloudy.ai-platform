# default-resource-config Specification

## Purpose

定义默认资源配置的管理规范，包括配置初始化、默认标记、自动关联等行为。

## Requirements

### Requirement: 资源配置支持默认标记

系统必须支持为每种资源配置类型（数据库、缓存、存储、队列、发布订阅）标记一个默认配置。

#### Scenario: 查看默认配置标记

- **当** 管理员查看资源配置列表
- **那么** 系统显示每个配置的 `is_default` 字段，默认配置标记为 `true`

#### Scenario: 默认配置排在第一位

- **当** 管理员查询资源配置列表
- **那么** `is_default=true` 的配置排在列表第一位

### Requirement: 默认配置唯一性保证

系统必须保证每种资源配置类型只能有一个默认配置。

#### Scenario: 创建新的默认配置

- **当** 管理员创建资源配置并标记为默认
- **那么** 系统将其他同类型配置的 `is_default` 设置为 `false`，新配置标记为 `true`

#### Scenario: 更新配置为默认

- **当** 管理员将现有配置更新为默认配置
- **那么** 系统将其他同类型配置的 `is_default` 设置为 `false`，当前配置标记为 `true`

#### Scenario: 并发创建默认配置

- **当** 多个请求同时尝试创建默认配置
- **那么** 数据库唯一索引阻止重复，只有一个请求成功

### Requirement: 应用启动时自动初始化默认配置

系统必须在应用启动时自动创建默认资源配置（如果不存在）。

#### Scenario: 首次启动创建默认配置

- **当** 应用首次启动且不存在默认配置
- **那么** 系统从 `application.yml` 读取配置并创建默认资源配置，标记 `is_default=true`

#### Scenario: 默认配置已存在

- **当** 应用启动且默认配置已存在
- **那么** 系统跳过创建，保持现有配置不变

#### Scenario: 配置文件包含敏感信息

- **当** 创建默认配置时涉及密码、密钥等敏感字段
- **那么** 系统使用 AES-256-GCM 加密后存储

### Requirement: 数据库连接字符串解析

系统必须支持从数据库连接 URL 解析连接参数。

#### Scenario: 解析 PostgreSQL 连接字符串

- **当** 系统从 `application.yml` 读取 `sqlalchemy.url`
- **那么** 系统解析 URL 并提取数据库类型、主机、端口、数据库名、用户名、密码

#### Scenario: URL 包含特殊字符

- **当** 数据库密码包含特殊字符（如 `@`, `:`, `/`）
- **那么** 系统正确解析 URL 编码的字符

### Requirement: 租户创建时自动关联默认配置

系统必须在创建租户时，如果未指定资源配置，自动关联默认配置。

#### Scenario: 未指定配置时使用默认

- **当** 管理员创建租户时未传入 `db_config_id`
- **那么** 系统查询 `DatabaseConfig` 中 `is_default=true` 的配置并关联

#### Scenario: 指定配置时使用指定

- **当** 管理员创建租户时传入 `db_config_id`
- **那么** 系统使用指定的配置 ID，不使用默认配置

#### Scenario: 默认配置不存在

- **当** 创建租户时未指定配置且默认配置不存在
- **那么** 系统创建租户但不关联配置（配置 ID 为 `null`）

### Requirement: 前端创建租户显示配置选择器

前端必须显示资源配置选择器，默认配置排在第一位且默认选中。

#### Scenario: 显示配置选择器

- **当** 管理员访问创建租户页面
- **那么** 页面显示数据库、缓存、存储、队列、发布订阅配置选择器

#### Scenario: 默认配置默认选中

- **当** 配置选择器加载完成
- **那么** `is_default=true` 的配置排在第一位且处于选中状态

#### Scenario: 选择其他配置

- **当** 管理员从选择器中选择非默认配置
- **那么** 选中的配置 ID 传递给后端

#### Scenario: 创建新配置

- **当** 管理员点击"+ 创建新配置"按钮
- **那么** 页面跳转到资源配置页面，创建完成后返回

### Requirement: API 响应包含 is_default 字段

资源配置 API 响应必须包含 `is_default` 字段。

#### Scenario: 查询配置列表

- **当** 调用 `GET /tenant/admin/v1/database-configs`
- **那么** 响应中每个配置包含 `is_default` 字段

#### Scenario: 查询单个配置

- **当** 调用 `GET /tenant/admin/v1/database-configs/{id}`
- **那么** 响应包含 `is_default` 字段

### Requirement: 默认配置不应轻易删除

系统必须提示用户删除默认配置的风险。

#### Scenario: 尝试删除默认配置

- **当** 管理员尝试删除 `is_default=true` 的配置
- **那么** 系统显示警告提示"此配置是默认配置，删除后新租户将无法自动关联配置"

#### Scenario: 默认配置被租户引用

- **当** 尝试删除的默认配置被租户引用
- **那么** 系统禁止删除并提示"此配置被租户使用，无法删除"
