## 新增需求

### 需求:IAM 模块登录页面测试

系统必须包含用户登录页面的 UI 测试，验证登录流程的正确性。

#### 场景:UI 登录成功
- **当** 访问 `/login` 页面
- **当** 输入正确的账号 `admin` 和密码 `admin123`
- **当** 点击登录按钮
- **那么** 页面跳转到首页或默认页面
- **那么** localStorage 包含有效的 `token` 和 `tenant_id`

#### 场景:UI 登录失败
- **当** 输入错误的账号或密码
- **当** 点击登录按钮
- **那么** 页面显示错误提示信息

### 需求:组织管理测试

系统必须包含组织管理模块的完整测试。

#### 场景:组织树渲染
- **当** 访问 `/iam/organizations` 页面
- **那么** 左侧组织树正确渲染
- **那么** 支持展开/折叠节点

#### 场景:组织详情展示

- **那么** 组织详情信息完整展示

- **当** 填写组织名称等必填字段
- **当** 点击保存
- **那么** 创建成功，组织树刷新显示新节点

#### 场景:编辑组织
- **当** 选中组织并点击"编辑"
- **当** 修改组织信息
- **当** 点击保存
- **那么** 更新成功，组织树刷新

#### 场景:删除组织
- **当** 选中组织并点击"删除"
- **当** 确认删除
- **那么** 删除成功，组织从树中移除
