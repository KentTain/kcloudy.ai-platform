## 新增需求

### 需求:插件进程隔离运行

系统必须为每个插件创建独立的进程运行环境，确保插件之间的资源隔离和故障隔离。

#### 场景:启动插件进程
- **当** 管理员启动一个已安装的插件
- **那么** 系统创建独立的子进程运行插件代码
- **且** 插件进程拥有独立的工作目录和虚拟环境

#### 场景:插件进程崩溃不影响主服务
- **当** 插件进程因异常崩溃
- **那么** 主服务进程继续运行
- **且** 系统记录崩溃日志和错误信息
- **且** 系统更新插件状态为 INACTIVE

### 需求:插件虚拟环境管理

系统必须使用 uv 为每个插件创建独立的 Python 虚拟环境，确保插件依赖与主应用隔离。

#### 场景:安装插件时创建虚拟环境
- **当** 安装一个包含 Python 依赖的插件
- **那么** 系统使用 uv 创建虚拟环境
- **且** 系统安装插件声明的依赖包
- **且** 虚拟环境路径记录在插件安装记录中

#### 场景:卸载插件时清理虚拟环境
- **当** 卸载一个插件
- **那么** 系统删除该插件的虚拟环境目录
- **且** 系统释放磁盘空间

### 需求:插件通信协议

系统必须使用 JSON 协议与插件进程通信，支持请求-响应和流式两种模式。

#### 场景:同步调用插件方法
- **当** 系统调用插件的某个方法
- **那么** 系统通过 stdin 发送 JSON 格式的请求
- **且** 插件通过 stdout 返回 JSON 格式的响应
- **且** 请求包含唯一的 session_id 用于匹配响应

#### 场景:流式调用插件方法
- **当** 系统以流式模式调用插件
- **那么** 插件可以返回多个增量响应
- **且** 每个响应包含相同的 session_id
- **且** 最后一个响应标记为完成

### 需求:插件生命周期管理

系统必须支持插件的启动、停止、重启操作，并正确管理进程状态。

#### 场景:启动已安装的插件
- **当** 管理员请求启动插件
- **那么** 系统检查插件是否已在运行
- **且** 如果未运行，系统创建并启动插件进程
- **且** 系统更新插件的 last_started_at 时间戳

#### 场景:停止运行中的插件
- **当** 管理员请求停止插件
- **那么** 系统发送终止信号给插件进程
- **且** 系统等待进程正常退出（最多 30 秒）
- **且** 如果超时，系统强制终止进程
- **且** 系统更新插件的 last_stopped_at 时间戳

### 需求:插件懒加载与预热

系统必须支持插件懒加载模式，并提供预热机制减少首次调用延迟。

#### 场景:懒加载模式首次调用
- **当** 首次调用一个未启动的插件
- **那么** 系统自动启动插件进程
- **且** 系统等待插件就绪后执行调用

#### 场景:启动时预热指定插件
- **当** 系统启动时配置了插件预热
- **那么** 系统在后台启动预热列表中的插件
- **且** 预热结果记录在日志中

#### 场景:自动冻结长期未使用的插件
- **当** 插件超过冻结阈值时间未被访问
- **那么** 系统自动停止该插件以释放资源
- **且** 系统更新插件状态为 INACTIVE
