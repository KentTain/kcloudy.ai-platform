## 新增需求

### 需求:插件资源文件存储

系统必须将插件包中的 _assets 目录下的资源文件上传到对象存储，并生成访问 URL。

#### 场景:安装插件时上传资源文件
- **当** 安装包含 _assets 目录的插件
- **那么** 系统扫描 _assets 目录下的所有文件
- **且** 系统将每个文件上传到对象存储
- **且** 系统记录文件路径与 OSS 路径的映射关系

#### 场景:插件无资源文件
- **当** 安装不包含 _assets 目录的插件
- **那么** 系统跳过资源上传步骤
- **且** 安装流程正常完成

### 需求:插件资源文件访问

系统必须提供插件资源文件的访问 URL，支持图标、图片等静态资源的展示。

#### 场景:获取插件图标 URL
- **当** 查询插件详情时包含图标配置
- **那么** 系统生成图标的完整访问 URL
- **且** URL 指向对象存储中的资源文件

#### 场景:访问不存在的资源文件
- **当** 请求访问插件不存在的资源文件
- **那么** 系统返回 404 错误

### 需求:插件包存储

系统必须将插件包上传到对象存储，支持版本管理和后续下载。

#### 场景:上传插件包到 OSS
- **当** 安装插件时
- **那么** 系统将插件包上传到 `plugins/{tenant_id}/{plugin_id}/{version}/plugin.zip`
- **且** 系统记录 OSS 路径到安装配置中

#### 场景:从 OSS 下载插件包
- **当** 需要重新安装或恢复插件时
- **那么** 系统从 OSS 下载插件包
- **且** 系统验证下载的文件完整性

### 需求:资源文件缓存

系统必须支持插件资源文件的缓存，减少重复下载和网络延迟。

#### 场景:首次访问资源文件
- **当** 首次访问某个资源文件
- **那么** 系统从 OSS 下载并缓存到本地
- **且** 后续访问优先使用本地缓存

#### 场景:缓存过期后重新下载
- **当** 缓存的资源文件过期
- **那么** 系统重新从 OSS 下载最新版本
- **且** 系统更新本地缓存
