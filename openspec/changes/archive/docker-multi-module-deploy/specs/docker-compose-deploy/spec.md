## 新增需求

### 需求:平台版部署服务

系统必须提供平台版部署配置，包含所有模块。

平台版服务必须：
- 构建时包含 demo, iam, tenant 所有模块
- 监听端口 3000（可配置）
- 设置 MODULE_BASE_PATH 环境变量

#### 场景:启动平台版容器
- **当** 执行 `docker-compose up platform-app`
- **那么** 容器在端口 3000 提供完整平台服务

#### 场景:访问平台版菜单
- **当** 访问平台版
- **那么** 显示所有模块的菜单项

### 需求:独立模块部署服务

系统必须支持各模块独立部署。

独立模块服务必须：
- 各模块使用不同端口
- 构建时仅包含指定模块
- 配置子域名

#### 场景:启动 Demo 模块独立部署
- **当** 执行 `docker-compose up demo-app`
- **那么** 容器在端口 3001 提供 Demo 模块服务

#### 场景:启动 IAM 模块独立部署
- **当** 执行 `docker-compose up iam-app`
- **那么** 容器在端口 3002 提供 IAM 模块服务

#### 场景:启动 Tenant 模块独立部署
- **当** 执行 `docker-compose up tenant-app`
- **那么** 容器在端口 3003 提供 Tenant 模块服务

### 需求:后端服务依赖

系统必须配置前端容器对后端服务的依赖。

依赖配置必须：
- 前端容器等待后端容器启动
- 配置 API 代理地址
- 设置健康检查依赖

#### 场景:后端未启动时前端等待
- **当** 启动 docker-compose 且后端未就绪
- **那么** 前端容器等待后端健康检查通过

### 需求:网络配置

系统必须配置 Docker 网络支持容器间通信。

网络配置必须：
- 创建统一的 overlay 网络
- 支持服务发现
- 配置子域名解析

#### 场景:容器间服务发现
- **当** 前端容器请求后端 API
- **那么** 通过服务名 `backend` 解析后端地址
