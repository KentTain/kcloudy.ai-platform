## 新增需求

### 需求:系统支持知识图谱索引构建

系统必须提供 GraphRAG 索引构建能力，从文档集合中提取实体、关系，构建知识图谱。

#### 场景:创建索引构建任务

- **当** 调用 create_index_build_task 方法，传入文档列表
- **那么** 系统创建异步索引构建任务并返回任务 ID

#### 场景:实体提取成功

- **当** 索引构建过程中提取实体
- **那么** 系统识别文档中的实体并存储

#### 场景:关系提取成功

- **当** 索引构建过程中提取关系
- **那么** 系统识别实体间的关系并存储

#### 场景:社区报告生成

- **当** 实体和关系提取完成
- **那么** 系统生成社区摘要报告

### 需求:系统支持知识图谱查询

系统必须提供基于知识图谱的结构化查询能力，支持全局搜索和局部搜索。

#### 场景:全局搜索

- **当** 调用全局搜索方法，传入查询问题
- **那么** 系统基于社区报告生成回答

#### 场景:局部搜索

- **当** 调用局部搜索方法，传入查询问题和实体
- **那么** 系统基于相关实体和关系生成回答

#### 场景:查询无结果

- **当** 知识图谱中不存在相关内容
- **那么** 系统返回空结果或提示信息

### 需求:系统支持 LLM 集成

系统必须集成 LLM 服务，用于实体提取、关系提取、社区报告生成等任务。

#### 场景:OpenAI LLM 调用

- **当** 使用 OpenAI 配置初始化 LLM 客户端
- **那么** 系统可调用 OpenAI API 完成文本生成

#### 场景:LLM 调用限流

- **当** LLM 调用频率超过限制
- **那么** 系统自动限流等待

#### 场景:LLM 调用缓存

- **当** 相同请求重复调用 LLM
- **那么** 系统返回缓存结果

### 需求:系统支持多种存储后端

系统必须支持多种存储后端，包括文件系统、MinIO、内存存储。

#### 场景:文件系统存储

- **当** 配置使用文件系统存储
- **那么** 系统将索引数据存储到本地文件

#### 场景:MinIO 存储

- **当** 配置使用 MinIO 存储
- **那么** 系统将索引数据存储到 MinIO 对象存储

### 需求:系统支持提示词调优

系统必须提供提示词调优工具，根据领域数据优化实体提取、社区报告生成的提示词。

#### 场景:生成实体提取提示词

- **当** 调用提示词调优工具，传入领域文档样本
- **那么** 系统生成优化的实体提取提示词

#### 场景:生成社区报告提示词

- **当** 调用提示词调优工具
- **那么** 系统生成优化的社区报告生成提示词

### 需求:系统支持中文提示词

系统必须提供中文提示词模板，支持中文知识图谱构建。

#### 场景:使用中文实体提取提示词

- **当** 配置使用中文语言
- **那么** 实体提取使用中文提示词模板

### 需求:系统提供 GraphRAG 客户端

系统必须提供 GraphRAG 客户端（GraphRAGClient），封装索引构建、查询、实体管理等操作。

#### 场景:获取知识图谱数据

- **当** 调用客户端获取图谱数据
- **那么** 系统返回实体和关系的列表

#### 场景:更新实体

- **当** 调用客户端更新实体属性
- **那么** 系统更新知识图谱中的实体

#### 场景:删除实体

- **当** 调用客户端删除实体
- **那么** 系统从知识图谱中移除实体及相关关系
