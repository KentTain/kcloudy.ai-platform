## 新增需求

### 需求:插件包解析导入测试

系统必须验证真实插件包的解析导入流程正确性。

#### 场景:解析 tongyi 插件包

- **当** 上传 tongyi 插件包到系统
- **那么** 系统成功解析 manifest.yaml
- **且** 提取正确的插件元数据（author、name、version、type）
- **且** 插件 ID 格式为 `langgenius/tongyi`

#### 场景:解析 gpustack 插件包

- **当** 上传 gpustack 插件包到系统
- **那么** 系统成功解析 manifest.yaml
- **且** 提取正确的插件元数据

#### 场景:解析缺少 manifest 的无效插件包

- **当** 上传缺少 manifest.yaml 的插件包
- **那么** 系统拒绝安装并返回错误信息
- **且** 错误信息包含"缺少 manifest 文件"

### 需求:插件安装流程测试

系统必须验证插件安装流程的正确性，包括包解压、虚拟环境创建、依赖安装和 OSS 上传。

#### 场景:安装 tongyi 插件

- **当** 调用安装 API 安装 tongyi 插件包
- **那么** 系统创建 PluginInstallation 记录
- **且** 系统创建虚拟环境并安装依赖
- **且** 系统上传插件包到 OSS
- **且** 安装状态变为 ACTIVE

#### 场景:安装已存在的插件

- **当** 尝试安装已安装的 tongyi 插件
- **那么** 系统拒绝安装并返回"插件已安装"错误

#### 场景:卸载已安装的插件

- **当** 调用卸载 API 卸载 tongyi 插件
- **那么** 系统停止插件进程
- **且** 系统删除虚拟环境
- **且** 系统删除 PluginInstallation 记录

### 需求:插件配置验证测试

系统必须验证插件凭证配置和有效性校验的正确性。

#### 场景:配置 tongyi 凭证

- **当** 创建 tongyi 插件的凭证配置
- **且** 提供有效的 API Key
- **那么** 系统加密存储凭证
- **且** 凭证状态为 ACTIVE

#### 场景:配置无效凭证

- **当** 创建插件凭证时提供无效的 API Key
- **那么** 系统拒绝创建并返回验证失败错误

#### 场景:更新凭证

- **当** 更新已存在凭证的 API Key
- **那么** 系统更新加密存储的凭证
- **且** 保留未修改的字段

#### 场景:脱敏显示凭证

- **当** 查询凭证详情
- **那么** 敏感字段被脱敏显示
- **且** 格式为 `sk-****xxxx`

### 需求:插件运行时测试

系统必须验证插件进程启动、停止和状态管理的正确性。

#### 场景:启动 tongyi 插件

- **当** 调用启动 API 启动已安装的 tongyi 插件
- **那么** 系统创建独立的子进程运行插件
- **且** 插件状态变为 ACTIVE
- **且** 进程 ID 记录到运行时状态

#### 场景:停止运行中的插件

- **当** 调用停止 API 停止运行中的 tongyi 插件
- **那么** 系统发送终止信号给插件进程
- **且** 插件状态变为 INACTIVE
- **且** 进程退出

#### 场景:插件进程崩溃不影响主服务

- **当** 插件进程因异常崩溃
- **那么** 主服务进程继续运行
- **且** 系统更新插件状态为 INACTIVE
- **且** 系统记录崩溃日志

### 需求:模型调用测试

系统必须验证通过插件调用真实 AI 模型的正确性。

#### 场景:调用 tongyi 模型

- **当** 配置有效的 tongyi API Key
- **且** 调用模型生成接口
- **那么** 系统成功调用 tongyi API
- **且** 返回有效的模型响应

#### 场景:调用 gpustack 模型

- **当** 配置有效的 gpustack API Key 和 Endpoint
- **且** 调用模型生成接口
- **那么** 系统成功调用 gpustack API
- **且** 返回有效的模型响应

#### 场景:流式调用模型

- **当** 以流式模式调用 tongyi 模型
- **那么** 系统返回多个增量响应
- **且** 最后一个响应标记为完成

#### 场景:调用失败时返回错误

- **当** 使用无效 API Key 调用模型
- **那么** 系统返回认证失败错误
- **且** 错误包含详细错误信息

### 需求:完整生命周期测试

系统必须验证插件从安装到卸载的完整生命周期。

#### 场景:tongyi 插件完整生命周期

- **当** 执行以下步骤：
  1. 上传并安装 tongyi 插件包
  2. 配置 tongyi API Key 凭证
  3. 启动插件
  4. 调用模型生成
  5. 停止插件
  6. 卸载插件
- **那么** 每个步骤都成功完成
- **且** 最终资源完全清理

#### 场景:gpustack 插件完整生命周期

- **当** 执行以下步骤：
  1. 上传并安装 gpustack 插件包
  2. 配置 gpustack API Key 和 Endpoint
  3. 启动插件
  4. 调用模型生成
  5. 停止插件
  6. 卸载插件
- **那么** 每个步骤都成功完成
- **且** 最终资源完全清理
