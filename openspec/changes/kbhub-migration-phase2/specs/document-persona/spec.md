# 人设管理功能规范

## 场景

### 创建人设

- **Given** 用户已登录且具备人设管理权限
- **When** 用户创建人设，指定名称、角色和指令
- **Then** 人设创建成功，返回人设 ID

### 创建人设——名称租户内唯一

- **Given** 同一租户下已存在同名的人设
- **When** 用户创建同名人设
- **Then** 系统拒绝创建并返回"人设名称已存在"的错误提示

### 查询人设

- **Given** 租户下存在多个人设
- **When** 用户查询人设列表
- **Then** 返回该租户下所有人设

### 编辑人设

- **Given** 用户具备人设管理权限，且人设存在
- **When** 用户修改人设的名称、角色或指令
- **Then** 人设更新成功，更新时间刷新

### 编辑人设——名称租户内唯一

- **Given** 同一租户下已存在名称为"助手B"的人设
- **When** 用户将另一个人设的名称修改为"助手B"
- **Then** 系统拒绝修改并返回"人设名称已存在"的错误提示

### 删除人设——未被标签引用

- **Given** 人设未被任何标签引用
- **When** 用户删除该人设
- **Then** 人设从系统中删除

### 删除人设——已被标签引用

- **Given** 人设已被标签引用
- **When** 用户尝试删除该人设
- **Then** 系统拒绝删除并返回"人设已被标签引用，不可删除"的错误提示

### 编辑人设后已引用标签自动使用最新内容

- **Given** 标签 T 引用了人设 P，人设 P 的指令为"你是一个翻译助手"
- **When** 用户将人设 P 的指令修改为"你是一个代码审查助手"
- **Then** 标签 T 关联的人设内容自动更新为"你是一个代码审查助手"，无需标签侧额外操作

### 人设选项列表——供标签管理选择

- **Given** 租户下存在多个人设
- **When** 标签管理模块请求人设选项列表
- **Then** 返回所有可用人设的 ID 和名称列表（不含完整指令内容），供下拉选择使用

### 人设选项列表——空列表

- **Given** 租户下不存在任何人设
- **When** 标签管理模块请求人设选项列表
- **Then** 返回空列表

### 多租户隔离——数据不可见

- **Given** 租户 A 和租户 B 各自拥有独立的人设
- **When** 租户 A 的用户查询人设列表
- **Then** 仅返回租户 A 的人设，租户 B 的人设不可见

### 多租户隔离——操作不可达

- **Given** 租户 A 的用户获知租户 B 的人设 ID
- **When** 租户 A 的用户尝试编辑或删除租户 B 的人设
- **Then** 系统拒绝操作并返回"资源不存在"的错误提示（不泄露跨租户存在性）

### 多租户隔离——名称唯一性仅限租户内

- **Given** 租户 A 已存在名称为"助手"的人设，租户 B 不存在同名人设
- **When** 租户 B 的用户创建名称为"助手"的人设
- **Then** 创建成功，不同租户间人设名称允许重复
