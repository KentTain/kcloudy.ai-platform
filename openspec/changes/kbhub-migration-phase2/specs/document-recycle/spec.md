# 回收站管理

## 场景

### 查看回收站

- **Given** 文档库中存在已软删除的文件或文件夹
- **When** 用户查看回收站
- **Then** 返回所有已软删除的文件和文件夹列表，按删除时间倒序排列

### 查看回收站——权限校验

- **Given** 用户拥有文档库的成员身份
- **When** 用户查看回收站
- **Then** 返回该文档库的回收站内容

### 查看回收站——筛选

- **Given** 回收站中存在多种类型的资源
- **When** 用户按资源类型（file/folder）筛选
- **Then** 仅返回指定类型的已删除资源

### 恢复文件

- **Given** 回收站中存在已软删除的文件，且其原始父文件夹仍存在
- **When** 用户恢复该文件
- **Then** 文件的 deleted_at 置空，恢复到原始父文件夹下

### 恢复文件——原始路径不存在

- **Given** 已软删除的文件的原始父文件夹已被永久删除
- **When** 用户恢复该文件
- **Then** 文件恢复到文档库根目录下

### 恢复文件夹

- **Given** 回收站中存在已软删除的文件夹
- **When** 用户恢复该文件夹
- **Then** 文件夹及其下所有子文件夹和文件同时恢复，保持原有树形结构

### 恢复文件夹——名称冲突

- **Given** 恢复路径下已存在同名文件夹
- **When** 用户恢复文件夹
- **Then** 系统自动重命名（如"文件夹名(1)"）后恢复

### 批量恢复

- **Given** 回收站中存在多个已删除资源
- **When** 用户批量选择并恢复
- **Then** 所有选中资源恢复到原路径（或根目录）

### 永久删除文件

- **Given** 用户是文档库的 owner 或 admin
- **When** 用户永久删除回收站中的文件
- **Then** 文件从系统中彻底删除（物理删除），不可恢复，写入审计日志

### 永久删除文件夹

- **Given** 用户是文档库的 owner 或 admin
- **When** 用户永久删除回收站中的文件夹
- **Then** 文件夹及其下所有子文件夹和文件彻底删除，写入审计日志

### 永久删除——无权限

- **Given** 用户不是文档库的 owner 或 admin
- **When** 用户尝试永久删除资源
- **Then** 系统拒绝操作并返回权限不足的错误提示

### 批量永久删除

- **Given** 用户是文档库的 owner 或 admin
- **When** 用户批量选择并永久删除
- **Then** 所有选中资源彻底删除，写入审计日志

### 清空回收站

- **Given** 用户是文档库的 owner 或 admin
- **When** 用户清空回收站
- **Then** 回收站中所有资源彻底删除，写入审计日志

### 自动清理——超过保留期限

- **Given** 系统配置了回收站保留期限（如 30 天）
- **When** 定时任务扫描回收站
- **Then** 超过保留期限的资源自动永久删除

### 多租户隔离

- **Given** 不同租户的回收站数据
- **When** 查询回收站
- **Then** 仅返回当前租户当前文档库的已删除资源
