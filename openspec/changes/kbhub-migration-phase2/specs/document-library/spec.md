# 文档库管理功能规范

## 场景

### 创建个人文档库

- **Given** 用户已登录且尚未拥有个人文档库
- **When** 用户请求创建个人文档库
- **Then** 系统创建个人文档库，该用户自动成为 owner，且该用户此后不可再创建第二个个人文档库

### 创建个人文档库——已达上限

- **Given** 用户已登录且已拥有一个个人文档库
- **When** 用户再次请求创建个人文档库
- **Then** 系统拒绝创建并返回"每个用户最多一个个人文档库"的错误提示

### 创建团队文档库

- **Given** 用户已登录且具备团队文档库创建权限
- **When** 用户请求创建团队文档库，指定名称与描述
- **Then** 系统创建团队文档库，创建者自动成为 owner

### 创建团队文档库——名称重复

- **Given** 同一租户下已存在同名团队文档库
- **When** 用户请求创建同名团队文档库
- **Then** 系统拒绝创建并返回"文档库名称已存在"的错误提示

### 文档库列表查询——分页

- **Given** 租户下存在 25 个文档库
- **When** 用户以 page=1, pageSize=10 查询文档库列表
- **Then** 返回第 1-10 条记录，total=25

### 文档库列表查询——关键词筛选

- **Given** 租户下存在名称含"产品"和"研发"的文档库
- **When** 用户以 keyword="产品" 查询文档库列表
- **Then** 仅返回名称包含"产品"的文档库

### 文档库列表查询——类型筛选

- **Given** 租户下存在个人文档库和团队文档库
- **When** 用户以 type="personal" 查询文档库列表
- **Then** 仅返回个人文档库

### 编辑文档库——名称

- **Given** 用户是文档库的 owner 或 admin
- **When** 用户修改文档库名称为"新名称"
- **Then** 文档库名称更新为"新名称"，更新时间刷新

### 编辑文档库——描述/图标/设置

- **Given** 用户是文档库的 owner 或 admin
- **When** 用户修改文档库的描述、图标或设置项
- **Then** 对应字段更新，更新时间刷新

### 编辑文档库——无权限

- **Given** 用户仅拥有 readonly 权限
- **When** 用户尝试编辑文档库
- **Then** 系统拒绝操作并返回权限不足的错误提示

### 删除文档库——软删除

- **Given** 用户是文档库的 owner
- **When** 用户删除该文档库
- **Then** 文档库标记为已删除（软删除），不再出现在正常列表中，但仍在回收站中可恢复

### 删除文档库——无权限

- **Given** 用户不是文档库的 owner
- **When** 用户尝试删除该文档库
- **Then** 系统拒绝操作并返回权限不足的错误提示

### 成员管理——添加成员

- **Given** 用户是文档库的 admin 或 owner，且目标用户属于同一租户
- **When** 用户将目标用户添加为文档库成员并分配角色
- **Then** 目标用户成为该文档库成员，拥有对应角色权限

### 成员管理——添加成员——跨租户

- **Given** 目标用户属于不同租户
- **When** 用户尝试将目标用户添加为文档库成员
- **Then** 系统拒绝操作并返回"跨租户不可添加成员"的错误提示

### 成员管理——移除成员

- **Given** 用户是文档库的 admin 或 owner，且目标成员不是 owner
- **When** 用户移除目标成员
- **Then** 目标成员从文档库成员列表中移除，其权限立即失效

### 成员管理——角色分配

- **Given** 用户是文档库的 owner
- **When** 用户将某成员的角色从 editor 变更为 admin
- **Then** 该成员角色更新为 admin，权限立即生效

### 成员管理——不可移除 owner

- **Given** 目标成员是文档库的 owner
- **When** 用户尝试移除该 owner
- **Then** 系统拒绝操作，owner 不可被移除

### 文件夹树管理——创建文件夹

- **Given** 用户拥有文档库的编辑权限，且父文件夹存在
- **When** 用户在父文件夹下创建子文件夹
- **Then** 子文件夹创建成功，出现在父文件夹的子节点中

### 文件夹树管理——重命名文件夹

- **Given** 用户拥有编辑权限，且目标文件夹存在
- **When** 用户将文件夹重命名为"新文件夹名"
- **Then** 文件夹名称更新，更新时间刷新

### 文件夹树管理——移动文件夹

- **Given** 用户拥有源文件夹和目标文件夹的编辑权限，且移动不会形成循环引用
- **When** 用户将文件夹 A 移动到文件夹 B 下
- **Then** 文件夹 A 成为文件夹 B 的子节点，其下所有子文件夹和文件路径同步更新

### 文件夹树管理——移动文件夹——循环引用

- **Given** 文件夹 B 是文件夹 A 的子孙节点
- **When** 用户尝试将文件夹 A 移动到文件夹 B 下
- **Then** 系统拒绝操作并返回"不允许形成循环引用"的错误提示

### 文件夹树管理——删除文件夹

- **Given** 用户拥有编辑权限，且目标文件夹下无锁定文件
- **When** 用户删除该文件夹
- **Then** 文件夹及其下所有子文件夹和文件进入回收站（软删除）

### 文件上传

- **Given** 用户拥有文档库的编辑权限，且目标文件夹存在
- **When** 用户上传文件到目标文件夹
- **Then** 文件保存成功，出现在目标文件夹的文件列表中，文件大小和类型符合限制

### 文件上传——超出大小限制

- **Given** 用户上传的文件大小超过系统限制
- **When** 用户尝试上传该文件
- **Then** 系统拒绝上传并返回"文件大小超出限制"的错误提示

### 文件预览

- **Given** 用户拥有文档库的只读或编辑权限，且文件格式支持在线预览
- **When** 用户请求预览该文件
- **Then** 系统返回文件预览内容

### 文件下载

- **Given** 用户拥有文档库的只读或编辑权限
- **When** 用户请求下载该文件
- **Then** 系统返回文件下载流

### 回收站——查看

- **Given** 文档库中存在已软删除的文件或文件夹
- **When** 用户查看回收站
- **Then** 返回所有已软删除的文件和文件夹列表

### 回收站——恢复

- **Given** 回收站中存在已软删除的文件，且其原始父文件夹仍存在
- **When** 用户恢复该文件
- **Then** 文件从回收站移出，恢复到原始父文件夹下

### 回收站——恢复——原始路径不存在

- **Given** 已软删除的文件的原始父文件夹已被永久删除
- **When** 用户恢复该文件
- **Then** 文件恢复到文档库根目录下

### 回收站——永久删除

- **Given** 用户是文档库的 owner 或 admin
- **When** 用户永久删除回收站中的文件
- **Then** 文件从系统中彻底删除，不可恢复

### 元数据管理——设置元数据

- **Given** 用户拥有文档库的编辑权限
- **When** 用户为文件设置元数据键值对（如 author="张三", version="2.0"）
- **Then** 元数据保存成功，与文件关联

### 元数据管理——查询元数据

- **Given** 文件已关联元数据
- **When** 用户查询该文件的元数据
- **Then** 返回该文件的所有元数据键值对

### 元数据管理——按元数据搜索

- **Given** 多个文件已关联元数据
- **When** 用户按元数据条件（如 author="张三"）搜索文件
- **Then** 返回满足条件的文件列表

### 多租户隔离——数据不可见

- **Given** 租户 A 和租户 B 各自拥有独立的文档库
- **When** 租户 A 的用户查询文档库列表
- **Then** 仅返回租户 A 的文档库，租户 B 的数据不可见

### 多租户隔离——操作不可达

- **Given** 租户 A 的用户获知租户 B 的文档库 ID
- **When** 租户 A 的用户尝试访问租户 B 的文档库
- **Then** 系统拒绝访问并返回"资源不存在"的错误提示（不泄露跨租户存在性）
