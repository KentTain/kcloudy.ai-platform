# 元数据管理

## 场景

### 定义文档库元数据字段

- **Given** 用户是文档库的 owner 或 admin
- **When** 创建元数据字段定义（字段名、字段类型、是否必填、枚举值列表）
- **Then** 元数据字段定义保存到 library_metadata_fields，可在文档库内复用

### 定义元数据字段——字段类型校验

- **Given** 管理员定义元数据字段
- **When** 设置字段类型为枚举（enum）
- **Then** 必须提供 enum_values 列表，其他类型（text/number/date/boolean）不需要

### 编辑元数据字段定义

- **Given** 用户是文档库的 owner 或 admin
- **When** 修改元数据字段定义（字段名、字段类型、是否必填）
- **Then** 字段定义更新，已设置的元数据值不受影响

### 删除元数据字段定义

- **Given** 用户是文档库的 owner 或 admin
- **When** 删除元数据字段定义
- **Then** 字段定义删除，该字段关联的资源元数据值同时删除

### 为资源设置元数据

- **Given** 用户拥有文档库的编辑权限，且资源（文件/文件夹）存在
- **When** 为资源设置元数据（字段ID + 值）
- **Then** 元数据保存到 resource_metadata，关联资源ID和字段ID

### 为资源设置元数据——枚举值校验

- **Given** 元数据字段类型为枚举
- **When** 设置的值不在 enum_values 列表中
- **Then** 系统拒绝并返回"枚举值不在允许范围内"的错误提示

### 批量设置元数据

- **Given** 用户拥有文档库的编辑权限
- **When** 为多个资源批量设置同一元数据字段
- **Then** 批量创建/更新 resource_metadata 记录

### 查询资源元数据

- **Given** 资源已关联元数据
- **When** 用户查询该资源的元数据
- **Then** 返回该资源的所有元数据键值对（字段名 + 值）

### 按元数据搜索资源

- **Given** 多个资源已关联元数据
- **When** 用户按元数据条件搜索（如 author="张三"）
- **Then** 返回满足条件的资源列表

### 按元数据搜索——组合条件

- **Given** 资源已关联多个元数据字段
- **When** 用户按多个元数据条件组合搜索（如 author="张三" AND version="2.0"）
- **Then** 返回同时满足所有条件的资源列表

### 元数据继承——文件夹

- **Given** 文件夹设置了元数据
- **When** 用户查询该文件夹下文件的元数据
- **Then** 文件可继承父文件夹的元数据（可被文件自身元数据覆盖）

### 多租户隔离

- **Given** 不同租户的元数据定义
- **When** 查询元数据字段定义
- **Then** 仅返回当前租户当前文档库的字段定义
