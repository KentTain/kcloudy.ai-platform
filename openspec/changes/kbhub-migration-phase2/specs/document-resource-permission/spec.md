# 文档库资源权限体系规范

## 场景

### owner/admin 拥有全部资源可编辑权限

- **Given** 用户是文档库的 owner 或 admin
- **When** 用户访问文档库内任意资源（文档库根、目录、文件）
- **Then** 用户对该资源拥有 editable 权限，无需额外配置

### 全员权限默认无直接资源权限

- **Given** 文档库的全员权限（all_members）尚未由管理员配置
- **When** 普通成员（非 owner/admin）访问文档库内资源
- **Then** 该成员对资源无直接权限（none），需通过其他权限组或显式授权获得权限

### 全员权限配置后影响全部普通成员

- **Given** 管理员将全员权限配置为 readonly
- **When** 普通成员访问文档库内资源
- **Then** 该成员对该资源拥有 readonly 权限（除非被更高权限组覆盖）

### 自定义权限组新建后默认复制全员权限

- **Given** 文档库的全员权限已配置为 readonly
- **When** 管理员新建自定义权限组
- **Then** 新权限组的资源权限默认复制全员权限（即 readonly），管理员可后续调整

### 成员加入多个权限组时取最高权限等级

- **Given** 成员同时属于权限组 A（readonly）和权限组 B（editable）
- **When** 该成员访问某资源
- **Then** 该成员对该资源的有效权限为 editable（取最高等级：editable > readonly > none）

### 成员加入多个权限组——均为 deny

- **Given** 成员同时属于权限组 A 和权限组 B，两者对该资源均配置为 deny
- **When** 该成员访问该资源
- **Then** 该成员对该资源的有效权限为 deny（deny 优先于其他权限等级）

### 资源权限继承——文档库根到目录

- **Given** 文档库根目录对某权限组配置为 editable，且子目录未单独配置权限
- **When** 该权限组成员访问子目录
- **Then** 子目录继承根目录的 editable 权限

### 资源权限继承——目录到文件

- **Given** 目录对某权限组配置为 readonly，且目录下文件未单独配置权限
- **When** 该权限组成员访问该文件
- **Then** 文件继承目录的 readonly 权限

### 关闭继承后父级权限截断

- **Given** 目录 A 对某权限组配置为 editable，子目录 B 关闭了权限继承
- **When** 该权限组成员访问子目录 B
- **Then** 子目录 B 不继承目录 A 的 editable 权限，其权限由自身配置或更上层未截断的继承决定

### 关闭继承后子级重新开启继承

- **Given** 目录 A 权限为 editable，子目录 B 关闭继承后自身配置为 readonly，子目录 C 在 B 下且开启继承
- **When** 该权限组成员访问子目录 C
- **Then** 子目录 C 继承子目录 B 的 readonly 权限（而非目录 A 的 editable）

### Policy deny 优先

- **Given** 某权限组对资源同时存在 allow（editable）和 deny 两条策略
- **When** 系统计算该权限组对该资源的有效权限
- **Then** deny 优先，有效权限为 deny

### Policy deny 优先——显式 deny 覆盖继承 allow

- **Given** 父目录对权限组配置为 editable（allow），子目录对该权限组显式配置为 deny
- **When** 系统计算该权限组对子目录的有效权限
- **Then** deny 优先，有效权限为 deny

### 权限变更写入审计日志

- **Given** 管理员修改了某权限组对某资源的权限配置
- **When** 权限变更完成
- **Then** 系统写入审计日志，记录操作人、操作时间、权限组、资源、变更前权限、变更后权限

### 权限变更写入权限变更事件

- **Given** 管理员修改了某权限组对某资源的权限配置
- **When** 权限变更完成
- **Then** 系统发布权限变更事件，下游模块可订阅该事件以同步缓存或触发通知

### 权限排障输出最终允许/拒绝及命中原因

- **Given** 用户对某资源的权限由多条策略共同决定（继承、显式配置、deny 优先等）
- **When** 管理员请求对该用户进行权限排障
- **Then** 系统输出最终结果（允许/拒绝）、有效权限等级、以及每条命中策略的来源和优先级说明

### 权限排障——无任何策略命中

- **Given** 用户不属于任何对该资源有配置的权限组
- **When** 管理员请求对该用户进行权限排障
- **Then** 系统输出最终结果为拒绝，原因为"未命中任何权限策略"

### 多租户隔离——权限配置不可跨租户

- **Given** 租户 A 的管理员尝试为租户 B 的权限组配置资源权限
- **When** 执行配置操作
- **Then** 系统拒绝操作并返回"跨租户操作不允许"的错误提示
