# 标签管理功能规范

## 场景

### 创建标签

- **Given** 用户已登录且具备标签管理权限
- **When** 用户创建标签，指定名称、颜色和可选的分组
- **Then** 标签创建成功，返回标签 ID

### 创建标签——名称重复

- **Given** 同一租户下已存在同名标签
- **When** 用户创建同名标签
- **Then** 系统拒绝创建并返回"标签名称已存在"的错误提示

### 查询标签

- **Given** 租户下存在多个标签
- **When** 用户查询标签列表
- **Then** 返回该租户下所有标签

### 编辑标签

- **Given** 用户具备标签管理权限，且标签存在
- **When** 用户修改标签的名称或颜色
- **Then** 标签更新成功，更新时间刷新

### 删除标签

- **Given** 用户具备标签管理权限，且标签未被任何资源引用
- **When** 用户删除该标签
- **Then** 标签从系统中删除

### 删除标签——已被资源引用

- **Given** 标签已被文档库资源引用
- **When** 用户尝试删除该标签
- **Then** 系统拒绝删除并返回"标签已被引用，不可删除"的错误提示

### 标签分组管理——创建分组

- **Given** 用户具备标签管理权限
- **When** 用户创建标签分组，指定分组名称
- **Then** 分组创建成功，后续创建标签时可归属该分组

### 标签分组管理——编辑分组

- **Given** 用户具备标签管理权限，且分组存在
- **When** 用户修改分组名称
- **Then** 分组名称更新成功，分组下标签不受影响

### 标签分组管理——删除分组

- **Given** 用户具备标签管理权限，且分组下无标签
- **When** 用户删除该分组
- **Then** 分组从系统中删除

### 标签分组管理——删除分组——分组下仍有标签

- **Given** 分组下仍关联有标签
- **When** 用户尝试删除该分组
- **Then** 系统拒绝删除并返回"分组下存在标签，不可删除"的错误提示

### 标签引用人设

- **Given** 标签存在且人设存在
- **When** 用户为标签引用一个人设
- **Then** 标签与人设关联成功，标签可查询到关联的人设信息

### 标签引用人设——替换引用

- **Given** 标签已引用人设 A
- **When** 用户将标签的人设引用替换为人设 B
- **Then** 标签的人设引用更新为 B，与人设 A 的关联解除

### 标签引用人设——取消引用

- **Given** 标签已引用某个人设
- **When** 用户取消标签的人设引用
- **Then** 标签与人设的关联解除

### 标签关键词搜索——匹配名称

- **Given** 租户下存在名称含"重要"的标签
- **When** 用户以关键词"重要"搜索标签
- **Then** 返回名称包含"重要"的标签列表

### 标签关键词搜索——无匹配

- **Given** 租户下不存在名称含"不存在的标签"的标签
- **When** 用户以关键词"不存在的标签"搜索标签
- **Then** 返回空列表

### 多租户隔离——数据不可见

- **Given** 租户 A 和租户 B 各自拥有独立的标签
- **When** 租户 A 的用户查询标签列表
- **Then** 仅返回租户 A 的标签，租户 B 的标签不可见

### 多租户隔离——操作不可达

- **Given** 租户 A 的用户获知租户 B 的标签 ID
- **When** 租户 A 的用户尝试编辑或删除租户 B 的标签
- **Then** 系统拒绝操作并返回"资源不存在"的错误提示（不泄露跨租户存在性）

### 多租户隔离——分组隔离

- **Given** 租户 A 和租户 B 各自拥有独立的标签分组
- **When** 租户 A 的用户查询标签分组列表
- **Then** 仅返回租户 A 的标签分组，租户 B 的分组不可见
