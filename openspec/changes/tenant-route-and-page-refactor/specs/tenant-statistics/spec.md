# tenant-statistics Specification

## 目的

提供租户统计功能，让管理员快速了解租户整体状况，包括租户总数、未激活租户数和过期租户数。

## 需求

### 需求: 租户统计数据必须实时准确

系统必须提供准确的租户统计数据，包括租户总数、未激活租户数和过期租户数。

#### 场景: 获取租户统计数据

- **当** 管理员请求租户列表接口
- **那么** 系统在响应中包含 `stats` 字段，提供实时统计数据

#### 场景: 统计数据准确性

- **当** 系统计算统计数据
- **那么** 租户总数等于所有未删除的租户数量
- **且** 未激活数等于状态为 `inactive` 的租户数量
- **且** 过期数等于 `expired_at` 小于当前时间的租户数量

#### 场景: 统计数据性能

- **当** 系统查询统计数据
- **那么** 使用单次 SQL 查询计算所有统计指标
- **且** 查询时间不超过 100ms（租户数 < 1000）

### 需求: 租户统计必须在列表接口返回

系统必须将统计数据作为租户列表接口的扩展字段返回，避免额外的 HTTP 请求。

#### 场景: 列表接口包含统计

- **当** 管理员请求 `GET /tenant/admin/v1/tenants`
- **那么** 响应包含 `stats` 字段
- **且** `stats` 包含 `totalCount`、`inactiveCount`、`expiredCount`

#### 场景: 统计字段格式

- **当** 系统返回统计数据
- **那么** `totalCount` 为整数类型
- **且** `inactiveCount` 为整数类型
- **且** `expiredCount` 为整数类型
- **且** 所有值大于等于 0

### 需求: 前端必须展示统计卡片

前端必须在租户管理页面展示统计卡片，提供直观的数据可视化。

#### 场景: 统计卡片展示

- **当** 管理员进入租户管理页面
- **那么** 页面顶部显示三个统计卡片
- **且** 第一个卡片显示"租户总数"
- **且** 第二个卡片显示"未激活数"
- **且** 第三个卡片显示"过期数"

#### 场景: 统计卡片样式

- **当** 统计卡片渲染
- **那么** 每个卡片包含图标、标题、数值和描述文字
- **且** 使用一致的卡片样式（参照 ModuleList 页面）

#### 场景: 统计数据加载状态

- **当** 统计数据正在加载
- **那么** 显示骨架屏或加载动画
- **且** 加载完成后显示实际数值
