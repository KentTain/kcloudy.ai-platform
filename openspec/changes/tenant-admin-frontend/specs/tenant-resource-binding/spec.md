# 租户资源绑定功能

## 新增需求

### 需求:管理员可以查看租户资源绑定

系统必须允许管理员查看租户当前的资源配置绑定情况。

#### 场景:查询租户资源绑定
- **当** 管理员请求 GET /admin/v1/tenants/{id}/resources
- **那么** 系统返回租户的资源绑定情况（数据库/存储/缓存/队列/发布订阅）

#### 场景:未绑定资源显示
- **当** 租户某项资源未绑定
- **那么** 系统返回该项资源为 null

### 需求:管理员可以更新租户资源绑定

系统必须允许管理员为租户绑定或解绑资源配置。

#### 场景:绑定资源配置
- **当** 管理员请求 PUT /admin/v1/tenants/{id}/resources 并提供配置 ID
- **那么** 系统更新租户的资源绑定

#### 场景:解绑资源
- **当** 管理员将某个配置设为 null
- **那么** 系统解除该资源的绑定

#### 场景:绑定不存在的配置
- **当** 管理员绑定的配置 ID 不存在
- **那么** 系统返回 404 错误

### 需求:租户详情页提供资源绑定 Tab

系统必须在租户详情页提供资源绑定 Tab，展示租户的资源绑定情况。

#### 场景:显示资源绑定 Tab
- **当** 管理员进入租户详情页并点击「资源绑定」Tab
- **那么** 页面显示租户当前的资源绑定情况

#### 场景:选择资源配置
- **当** 管理员点击资源配置下拉框
- **那么** 系统显示可用的配置列表

#### 场景:测试资源配置连通性
- **当** 管理员点击「测试连接」按钮
- **那么** 系统调用对应的 test-connection API 并显示结果

#### 场景:保存资源绑定
- **当** 管理员修改资源绑定并点击「保存」
- **那么** 系统更新资源绑定并显示成功提示
