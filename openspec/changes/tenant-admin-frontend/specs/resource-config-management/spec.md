# 资源配置管理功能

## 新增需求

### 需求:管理员可以管理数据库配置

系统必须允许管理员对数据库配置进行创建、查询、更新、删除操作，并支持连通性测试。

#### 场景:查询数据库配置列表
- **当** 管理员请求 GET /admin/v1/resource-configs/databases
- **那么** 系统返回数据库配置列表（分页）

#### 场景:创建数据库配置
- **当** 管理员请求 POST /admin/v1/resource-configs/databases 并提供配置信息
- **那么** 系统创建数据库配置并返回配置信息

#### 场景:更新数据库配置
- **当** 管理员请求 PUT /admin/v1/resource-configs/databases/{id} 并提供更新数据
- **那么** 系统更新配置并返回更新后的数据

#### 场景:删除数据库配置
- **当** 管理员请求 DELETE /admin/v1/resource-configs/databases/{id}
- **那么** 系统删除配置

#### 场景:删除已被引用的配置
- **当** 管理员尝试删除已被租户引用的配置
- **那么** 系统返回错误，禁止删除

#### 场景:测试数据库连通性
- **当** 管理员请求 POST /admin/v1/resource-configs/databases/{id}/test-connection
- **那么** 系统测试连接并返回测试结果（成功/失败/延迟）

### 需求:资源配置页面提供统一界面

系统必须提供 Tab 切换的统一页面，支持在数据库、存储、缓存、队列、发布订阅五种类型间切换。

#### 场景:切换资源配置类型
- **当** 管理员点击不同 Tab
- **那么** 页面切换显示对应类型的配置列表

#### 场景:显示统计卡片
- **当** 管理员进入资源配置页面
- **那么** 页面顶部显示配置总数、已被引用数、未被使用数

### 需求:资源配置表单弹窗

系统必须提供表单弹窗用于创建和编辑资源配置。

#### 场景:打开创建表单
- **当** 管理员点击「新增配置」按钮
- **那么** 系统弹出表单弹窗

#### 场景:提交表单验证失败
- **当** 管理员提交表单但必填字段未填写
- **那么** 系统显示验证错误提示

#### 场景:提交表单成功
- **当** 管理员提交有效表单数据
- **那么** 系统创建/更新配置，关闭弹窗，刷新列表
