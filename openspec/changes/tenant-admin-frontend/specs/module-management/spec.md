# 模块管理功能

## 新增需求

### 需求:管理员可以管理模块

系统必须允许管理员对模块进行创建、查询、更新、删除操作。

#### 场景:查询模块列表
- **当** 管理员请求 GET /admin/v1/modules
- **那么** 系统返回模块列表（分页）

#### 场景:创建模块
- **当** 管理员请求 POST /admin/v1/modules 并提供模块信息
- **那么** 系统创建模块并返回模块信息

#### 场景:创建重复编码模块
- **当** 管理员尝试创建已存在编码的模块
- **那么** 系统返回错误 "模块编码已存在"

#### 场景:更新模块
- **当** 管理员请求 PUT /admin/v1/modules/{id} 并提供更新数据
- **那么** 系统更新模块并返回更新后的数据

#### 场景:删除模块
- **当** 管理员请求 DELETE /admin/v1/modules/{id}
- **那么** 系统删除模块

#### 场景:删除已分配模块
- **当** 管理员尝试删除已被租户分配的模块
- **那么** 系统返回错误 "模块已被租户分配，无法删除"

### 需求:管理员可以管理模块菜单

系统必须允许管理员对模块菜单进行创建、查询、更新、删除操作，菜单为树形结构。

#### 场景:查询模块菜单树
- **当** 管理员请求 GET /admin/v1/modules/{id}/menus
- **那么** 系统返回模块的菜单树形结构

#### 场景:创建模块菜单
- **当** 管理员请求 POST /admin/v1/modules/{id}/menus 并提供菜单信息
- **那么** 系统创建菜单并返回菜单信息

#### 场景:创建子菜单
- **当** 管理员创建菜单时指定 parent_id
- **那么** 系统创建子菜单，关联到父菜单

#### 场景:删除有子菜单的菜单
- **当** 管理员尝试删除有子菜单的菜单
- **那么** 系统返回错误 "菜单有子菜单，无法删除"

#### 场景:菜单编辑弹窗
- **当** 管理员点击「新增菜单」或「编辑」按钮
- **那么** 系统弹出菜单表单弹窗，包含名称、编码、路径、父菜单、图标、排序字段

### 需求:管理员可以管理模块权限

系统必须允许管理员对模块权限进行创建、查询、更新、删除操作。

#### 场景:查询模块权限列表
- **当** 管理员请求 GET /admin/v1/modules/{id}/permissions
- **那么** 系统返回模块的权限列表

#### 场景:创建模块权限
- **当** 管理员请求 POST /admin/v1/modules/{id}/permissions 并提供权限信息
- **那么** 系统创建权限并返回权限信息

#### 场景:权限编码格式验证
- **当** 管理员创建权限时编码格式不符合 module:resource:action
- **那么** 系统返回验证错误

#### 场景:操作类型验证
- **当** 管理员创建权限时 action 不是 read/write/delete
- **那么** 系统返回错误 "操作类型必须是 read/write/delete"

### 需求:管理员可以管理模块角色

系统必须允许管理员对模块角色进行创建、查询、更新、删除操作，并支持为角色分配权限。

#### 场景:查询模块角色列表
- **当** 管理员请求 GET /admin/v1/modules/{id}/roles
- **那么** 系统返回模块的角色列表（包含权限信息）

#### 场景:创建模块角色
- **当** 管理员请求 POST /admin/v1/modules/{id}/roles 并提供角色信息
- **那么** 系统创建角色并返回角色信息

#### 场景:更新角色权限
- **当** 管理员请求 PUT /admin/v1/modules/{id}/roles/{roleId}/permissions 并提供权限 ID 列表
- **那么** 系统更新角色权限（整体替换）

#### 场景:修改系统内置角色
- **当** 管理员尝试修改 is_system=true 的角色
- **那么** 系统返回错误 "系统内置角色禁止修改"

#### 场景:删除系统内置角色
- **当** 管理员尝试删除 is_system=true 的角色
- **那么** 系统返回错误 "系统内置角色禁止删除"

### 需求:模块详情页面提供 Tab 切换

系统必须提供模块详情页，包含基本信息、菜单管理、权限管理、角色管理四个 Tab。

#### 场景:查看模块详情
- **当** 管理员访问 /admin/modules/{id}
- **那么** 系统显示模块详情页，默认展示基本信息 Tab

#### 场景:切换 Tab
- **当** 管理员点击不同 Tab
- **那么** 页面切换显示对应的管理界面

### 需求:模块列表页面显示统计卡片

系统必须在模块列表页面顶部显示统计卡片。

#### 场景:显示统计信息
- **当** 管理员进入模块列表页面
- **那么** 页面顶部显示模块总数、启用模块数、必须模块数、已分配次数
