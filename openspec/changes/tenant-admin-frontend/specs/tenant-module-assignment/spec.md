# 租户模块分配功能

## 新增需求

### 需求:管理员可以查看租户已分配模块

系统必须允许管理员查看租户已分配的模块列表。

#### 场景:查询租户模块列表
- **当** 管理员请求 GET /admin/v1/tenants/{id}/modules
- **那么** 系统返回租户已分配的模块列表

#### 场景:显示模块分配状态
- **当** 系统返回模块列表
- **那么** 每个模块包含分配时间、过期时间、启用状态

### 需求:管理员可以为租户分配模块

系统必须允许管理员为租户分配模块。

#### 场景:分配模块
- **当** 管理员请求 POST /admin/v1/tenants/{id}/modules 并提供模块 ID
- **那么** 系统为租户分配模块，创建租户实例层数据

#### 场景:分配已存在的模块
- **当** 管理员尝试分配租户已拥有的模块
- **那么** 系统返回错误（唯一约束冲突）

#### 场景:分配不存在的模块
- **当** 管理员分配的模块 ID 不存在
- **那么** 系统返回 404 错误

#### 场景:分配未启用的模块
- **当** 管理员分配 is_active=false 的模块
- **那么** 系统返回错误 "模块未启用"

#### 场景:必须模块设置过期时间
- **当** 管理员为 is_need=true 的模块设置过期时间
- **那么** 系统返回错误 "必须模块不可设置过期时间"

### 需求:管理员可以取消租户模块分配

系统必须允许管理员取消租户的模块分配。

#### 场景:取消模块分配
- **当** 管理员请求 DELETE /admin/v1/tenants/{id}/modules/{moduleId}
- **那么** 系统取消模块分配，删除租户实例层数据

#### 场景:取消必须模块分配
- **当** 管理员尝试取消 is_need=true 的模块
- **那么** 系统返回错误 "必须模块禁止取消"

#### 场景:取消有用户使用的模块
- **当** 管理员取消有用户使用该模块角色的模块
- **那么** 系统返回错误，提示先移除相关用户角色

### 需求:租户详情页提供模块分配 Tab

系统必须在租户详情页提供模块分配 Tab，展示租户已分配的模块列表。

#### 场景:显示模块分配 Tab
- **当** 管理员进入租户详情页并点击「模块分配」Tab
- **那么** 页面显示租户已分配的模块列表

#### 场景:分配模块弹窗
- **当** 管理员点击「分配模块」按钮
- **那么** 系统弹出模块分配表单，显示可分配的模块列表

#### 场景:取消分配确认
- **当** 管理员点击「取消分配」按钮
- **那么** 系统弹出确认对话框，提示取消分配的影响

#### 场景:查看模块详情
- **当** 管理员点击模块名称
- **那么** 系统跳转到模块详情页
