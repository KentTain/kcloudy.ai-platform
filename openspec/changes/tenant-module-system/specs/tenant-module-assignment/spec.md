## ADDED Requirements

### 需求:为租户分配模块

系统必须支持为租户分配功能模块，分配时自动同步菜单、权限、角色到租户实例层。

#### 场景:分配模块成功
- **当** 管理员请求 `POST /api/v1/tenant/tenants/{id}/modules` 并提供 module_id
- **那么** 创建 TenantModule 记录；发布 ModuleAssigned 事件；IAM 模块自动创建租户实例层菜单、权限、角色

#### 场景:分配已分配的模块
- **当** 管理员尝试分配租户已拥有的模块
- **那么** 返回 HTTP 400，错误消息为"模块已分配"

#### 场景:分配不存在的模块
- **当** 管理员尝试分配不存在的模块
- **那么** 返回 HTTP 404

#### 场景:分配未启用的模块
- **当** 管理员尝试分配 is_active=false 的模块
- **那么** 返回 HTTP 400，错误消息为"模块未启用"

#### 场景:分配必须模块设置过期时间
- **当** 管理员为 is_need=true 的模块设置 expired_at
- **那么** 返回 HTTP 400，错误消息为"必须模块不可设置过期时间"

### 需求:取消租户模块分配

系统必须支持取消租户的模块分配，取消前进行安全检查。

#### 场景:取消非必须模块成功
- **当** 管理员请求 `DELETE /api/v1/tenant/tenants/{id}/modules/{moduleId}` 且模块 is_need=false 且无用户使用该模块角色
- **那么** 删除 TenantModule 记录；发布 ModuleUnassigned 事件；级联删除租户实例层关联数据

#### 场景:取消必须模块
- **当** 管理员尝试取消 is_need=true 的模块
- **那么** 返回 HTTP 400，错误消息为"必须模块禁止取消"

#### 场景:有用户使用模块角色时取消
- **当** 管理员尝试取消模块，但租户实例层有 UserRole 关联该模块角色
- **那么** 返回 HTTP 400，错误消息为"有用户正在使用该模块的角色，请先移除相关用户角色"

### 需求:查询租户已分配模块

系统必须支持查询租户已分配的模块列表。

#### 场景:获取租户模块列表
- **当** 管理员请求 `GET /api/v1/tenant/tenants/{id}/modules`
- **那么** 返回租户已分配的模块列表，包含 started_at、expired_at、is_active 等信息
